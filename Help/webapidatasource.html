Overview

The WebAPIDataSource is a comprehensive implementation of the IDataSource interface specifically designed for REST API integration within the BeepDM framework. It provides a unified way to interact with web APIs using familiar database-like operations while handling the complexities of HTTP communication, authentication, caching, and error recovery.

🔄 Database-like Operations

Use familiar CRUD operations (Create, Read, Update, Delete) with any REST API

Configuration

Connection Properties

Configure your WebAPIDataSource using the WebAPIConnectionProperties class with built-in validation:

// Create WebAPI connection with comprehensive settings
var webApiConnection = new WebAPIConnectionProperties
{
    // Core connection settings
    ConnectionName = "GitHub API",
    ConnectionString = "https://api.github.com",
    AuthType = AuthTypeEnum.Bearer,
    ApiKey = "ghp_your-github-token",
    
    // Request configuration
    TimeoutMs = 30000,
    MaxRetries = 3,
    RetryIntervalMs = 1000,
    
    // Performance settings
    EnableCaching = true,
    CacheExpiryMinutes = 15,
    MaxConcurrentRequests = 10,
    
    // Rate limiting
    EnableRateLimit = true,
    RateLimitRequestsPerMinute = 60,
    
    // Response handling
    ResponseFormat = "json",
    UserAgent = "MyApp/1.0"
};

// Add custom headers
webApiConnection.Headers.Add(new WebApiHeader 
{ 
    Headername = "Accept", 
    Headervalue = "application/vnd.github.v3+json" 
});

// Set additional parameters
webApiConnection.SetParameterValue("DefaultPageSize", "50");
webApiConnection.SetParameterValue("MaxPageSize", "100");

// Validate configuration
if (webApiConnection.ValidateConfiguration())
{
    Console.WriteLine("Configuration is valid");
}
else
{
    Console.WriteLine($"Configuration errors: {webApiConnection.GetValidationErrors()}");
}

Configuration Validation and Error Handling

// Comprehensive validation example
public static WebAPIConnectionProperties CreateValidatedConnection(string configName)
{
    var connection = new WebAPIConnectionProperties();
    
    try
    {
        // Load configuration from various sources
        connection.ConnectionName = configName;
        connection.ConnectionString = Configuration.GetConnectionString(configName);
        connection.AuthType = Enum.Parse<AuthTypeEnum>(Configuration[$"{configName}:AuthType"]);
        
        // Set auth-specific properties
        switch (connection.AuthType)
        {
            case AuthTypeEnum.ApiKey:
                connection.ApiKey = Configuration[$"{configName}:ApiKey"];
                connection.ApiKeyHeader = Configuration[$"{configName}:ApiKeyHeader"] ?? "X-API-Key";
                break;
                
            case AuthTypeEnum.Basic:
                connection.UserID = Configuration[$"{configName}:Username"];
                connection.Password = Configuration[$"{configName}:Password"];
                break;
                
            case AuthTypeEnum.OAuth2:
                connection.ClientId = Configuration[$"{configName}:ClientId"];
                connection.ClientSecret = Configuration[$"{configName}:ClientSecret"];
                connection.TokenUrl = Configuration[$"{configName}:TokenUrl"];
                connection.Scope = Configuration[$"{configName}:Scope"];
                break;
        }
        
        // Apply environment-specific settings
        var environment = Configuration["Environment"];
        if (environment == "Production")
        {
            connection.TimeoutMs = 60000;
            connection.MaxRetries = 3;
            connection.ValidateServerCertificate = true;
        }
        else if (environment == "Development")
        {
            connection.TimeoutMs = 30000;
            connection.MaxRetries = 1;
            connection.IgnoreSSLErrors = true; // Only for dev!
        }
        
        // Validate the configuration
        if (!connection.ValidateConfiguration())
        {
            var errors = connection.GetValidationErrors();
            throw new ConfigurationException($"Invalid {configName} configuration: {errors}");
        }
        
        Logger.Information($"Successfully configured {configName} connection");
        return connection;
    }
    catch (Exception ex)
    {
        Logger.Error($"Failed to create {configName} connection: {ex.Message}");
        throw;
    }
}

// Usage
var githubConnection = CreateValidatedConnection("GitHub");
var stripeConnection = CreateValidatedConnection("Stripe");

HTTP Convenience Methods

In addition to the IDataSource CRUD interface, WebAPIDataSource exposes low-level HTTP helpers defined in WebAPIDataSource.Http.cs. These are the preferred way to call API-specific endpoints that don't map cleanly to entity operations, and they are the building blocks used by connector subclasses.

GET — raw response or deserialized

// Raw HttpResponseMessage
HttpResponseMessage resp = await ds.GetAsync(
    "users/me",
    query: new Dictionary<string, string> { ["fields"] = "id,name,email" },
    headers: new Dictionary<string, string> { ["Accept-Language"] = "en" }
);

// Deserialize directly to T
var me = await ds.GetAsync<UserProfile>("users/me");
// Returns default(T) on failure — never throws

POST / PUT / DELETE

// POST with a body
await ds.PostAsync("tweets", body: new { text = "Hello from Beep!" });

// PUT (full replace)
await ds.PutAsync($"users/{userId}", body: updatedUser);

// DELETE
await ds.DeleteAsync($"tweets/{tweetId}");
Authentication and headers are applied automatically

All four methods call _authHelper.EnsureAuthenticatedAsync() and _authHelper.AddAuthenticationHeaders(request) before sending, and apply the default headers from WebAPIConnectionProperties. Retry logic is handled by WebAPIRequestHelper.SendWithRetryAsync.

Absolute vs relative URLs

If endpointOrUrl starts with http it is used as-is. Otherwise it is combined with _configHelper.BaseUrl (the ConnectionString value).

Building a Custom API Connector

The recommended way to integrate a specific REST API — Twitter, GitHub, Stripe, any social or SaaS platform — is to subclass WebAPIDataSource and override the methods that need API-specific URL or response handling. The TwitterDataSource is the canonical reference implementation.

Connector Pattern

Minimum viable connector skeleton

// 1. Annotate with [AddinAttribute] so AssemblyHandler registers it automatically
[AddinAttribute(
    Category = DatasourceCategory.Connector,
    DatasourceType = DataSourceType.Twitter  // use the correct enum value for your API
)]
public class MyApiDataSource : WebAPIDataSource
{
    // 2. Map logical entity names to API endpoint paths
    private static readonly Dictionary<string, string> EntityEndpoints
        = new(StringComparer.OrdinalIgnoreCase)
    {
        ["items.list"]   = "items",
        ["items.by_id"]  = "items/{id}",
        ["items.search"] = "items/search"
    };

    // 3. Declare which AppFilter keys are required for each entity
    private static readonly Dictionary<string, string[]> RequiredFilters
        = new(StringComparer.OrdinalIgnoreCase)
    {
        ["items.by_id"]  = new[] { "id" },
        ["items.search"] = new[] { "query" }
    };

    public MyApiDataSource(
        string datasourcename, IDMLogger logger,
        IDMEEditor dmeEditor, DataSourceType databasetype, IErrorsInfo errorObject)
        : base(datasourcename, logger, dmeEditor, databasetype, errorObject)
    {
        EntitiesNames = EntityEndpoints.Keys.ToList();
        Entities = EntitiesNames
            .Select(n => new EntityStructure { EntityName = n, DatasourceEntityName = n })
            .ToList();
    }

    // 4. Override GetEntityAsync to drive URL resolution and response extraction
    public override async Task<IEnumerable<object>> GetEntityAsync(
        string EntityName, List<AppFilter> Filter)
    {
        if (!EntityEndpoints.TryGetValue(EntityName, out var endpoint))
            throw new InvalidOperationException($"Unknown entity '{EntityName}'");

        var q = FiltersToQuery(Filter);
        RequireFilters(EntityName, q,
            RequiredFilters.GetValueOrDefault(EntityName, Array.Empty<string>()));

        var resolvedEndpoint = ResolveEndpoint(endpoint, q);
        using var resp = await GetAsync(resolvedEndpoint, q);
        if (resp is null || !resp.IsSuccessStatusCode) return Array.Empty<object>();

        return ExtractArray(resp, "data");  // key in JSON root where items live
    }

    // 5. Expose sync overload
    public override IEnumerable<object> GetEntity(string EntityName, List<AppFilter> filter)
        => GetEntityAsync(EntityName, filter).GetAwaiter().GetResult() ?? Array.Empty<object>();
}

TwitterDataSource targets the Twitter API v2 (https://api.twitter.com/2) with Bearer token auth. Set up the connection via WebAPIConnectionProperties or through ConfigEditor.AddDataConnection.

Register via ConfigEditor (recommended)

var conn = new WebAPIConnectionProperties
{
    ConnectionName   = "Twitter",
    ConnectionString = "https://api.twitter.com/2",
    DatabaseType     = DataSourceType.Twitter,
    Category         = DatasourceCategory.Connector,
    DriverName       = "TwitterDataSource",       // matches [AddinAttribute]

    // Authentication
    ApiKey           = "AAAA...",                 // Bearer token
    // Store auth type in Parameters (read by WebAPIAuthenticationHelper)
};
// Set bearer auth type in extra Parameters
conn.Parameters = "AuthType=Bearer";

// Add a default Authorization header directly (simplest approach)
conn.Headers.Add(new WebApiHeader
{
    Key   = "Authorization",
    Value = $"Bearer {bearerToken}"
});

dmeEditor.ConfigEditor.AddDataConnection(conn);

// Get the data source (AssemblyHandler resolves TwitterDataSource via [AddinAttribute])
var twitter = (TwitterDataSource)dmeEditor.GetDataSource("Twitter");
twitter.Openconnection();
Do not hardcode secrets

Load your Bearer token from environment variables or a secrets manager. Never commit tokens to version control.

AppFilter → query-string mapping

FiltersToQuery converts each AppFilter to a key=value query-string entry using AppFilter.FieldName as the key and AppFilter.FilterValue as the value. Endpoint path parameters (e.g., {id}) are consumed by ResolveEndpoint and removed from the query string.

Twitter: Cursor Pagination

Twitter API v2 uses cursor-based pagination via meta.next_token. TwitterDataSource.GetEntity(name, filter, pageNumber, pageSize) emulates page-number semantics by walking the cursor chain up to pageNumber pages, then returns the requested page.

Paged retrieval (IDataSource PagedResult)

var filters = new List<AppFilter>
{
    new AppFilter { FieldName = "query", FilterValue = "#csharp", Operator = "=" }
};

// Page 1 (first 20 results)
PagedResult page1 = twitter.GetEntity("tweets.search", filters, pageNumber: 1, pageSize: 20);
Console.WriteLine($"Page 1: {page1.Data.Count} items");
Console.WriteLine($"Has next: {page1.HasNextPage}");

// Page 2
PagedResult page2 = twitter.GetEntity("tweets.search", filters, pageNumber: 2, pageSize: 20);

Explicit cursor pass-through (typed method)

string? cursor = null;
do
{
    var batch = await twitter.GetTweets("#dotnet", maxResults: 100, nextToken: cursor);
    ProcessBatch(batch);

    // Extract next_token from the last raw page for the next call
    // (use typed method overload or inspect PagedResult.TotalPages instead)
    cursor = GetNextCursor(batch); // your own helper to read meta.next_token
}
while (cursor != null);
Rate limits

Twitter API v2 Free tier: 500,000 tweet reads / month; 1 app / 1 user. Basic tier: 10,000 tweet reads / month with higher rate ceilings. Use WebAPIConnectionProperties.EnableRateLimit = true and RateLimitRequestsPerMinute to throttle automatically.

Connector Checklist

Follow these steps when building any new WebAPIDataSource subclass:

  1. Add [AddinAttribute(Category = DatasourceCategory.Connector, DatasourceType = DataSourceType.YourType)].
  2. Define EntityEndpoints (entity name → URL template dictionary).
  3. Define RequiredFilters (entity name → required AppFilter.FieldName array).
  4. Populate EntitiesNames and Entities in the constructor.
  5. Override GetEntityAsync: call FiltersToQuery → RequireFilters → ResolveEndpoint → GetAsync → ExtractArray.
  6. Add sync GetEntity override that delegates to GetEntityAsync(...).GetAwaiter().GetResult().
  7. Optionally override GetEntity (paged) for cursor or offset pagination.
  8. Add typed [CommandAttribute] methods for Beep tooling discoverability.
  9. Create a Models.cs with JSON-annotated POCOs for response deserialization.
  10. Store the connector in BeepDataSources/Connectors/<Category>/<Name>/.
