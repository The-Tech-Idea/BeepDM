FileManager Reader Host

How to discover, select, and switch file readers in FileDataSource.

Overview

FileDataSource supports runtime reader selection through IFileDataSourceReaderHost. You can list available readers, inspect metadata, and switch by type, implementation identity, or stable reader ID.

What this solves
  • Discover all registered readers from FileReaderRegistry.
  • Switch readers without changing connection object construction code.
  • Select between multiple readers for the same DataSourceType (for example, two CSV readers).

Reader Host API

IFileDataSourceReaderHost

public interface IFileDataSourceReaderHost
{
    IReadOnlyList<FileReaderDescriptorInfo> GetAvailableReaderDescriptors(bool discover = false);
    IReadOnlyList<DataSourceType> GetAvailableReaderTypes();
    FileReaderDescriptorInfo? GetCurrentReaderDescriptor();
    object? GetCurrentReader();

    bool TrySwitchReader(DataSourceType targetType, bool reconfigure, out string? reason);
    bool TrySwitchReader(string implementationTypeName, bool reconfigure, out string? reason);
    bool TrySwitchReader(Guid readerId, bool reconfigure, out string? reason);
    bool TrySwitchReaderByExtension(string extension, bool reconfigure, out string? reason);

    void ResetReaderSelection();
}

How Reader Selection Works

During Openconnection(), FileDataSource resolves the reader using:

  1. Selected override (if a switch API was called), otherwise
  2. DatasourceType default behavior.

On successful switch, DatasourceType is updated and the reader is optionally reconfigured using connection properties.

Safety behavior
  • Switch calls return false with a reason string for expected failures.
  • Switch is blocked while active transaction/ingestion state is in progress.

Multiple Readers for Same Format

Descriptor listing does not collapse by FormatType, so multiple CSV readers are shown independently.

Use one of these explicit selectors to choose the exact reader:

  • TrySwitchReader(string implementationTypeName, ...)
  • TrySwitchReader(Guid readerId, ...)
Important

TrySwitchReader(DataSourceType.CSV, ...) is still valid, but it may resolve whichever CSV reader is currently mapped in the factory for that type. Use implementation identity or reader ID when deterministic selection is required.

Create a New FileReader

To add a custom reader, implement IFileFormatReader, decorate it with [FileReader(...)], and ensure the assembly is loaded so registry discovery can find it.

1) Implement IFileFormatReader + Attribute

using System.Collections.Generic;
using TheTechIdea.Beep.FileManager.Attributes;
using TheTechIdea.Beep.FileManager.Readers;
using TheTechIdea.Beep.ConfigUtil;
using TheTechIdea.Beep.Utilities;

[FileReader(DataSourceType.CSV, "My Fast CSV Reader", "csv")]
public sealed class MyFastCsvReader : IFileFormatReader
{
    public DataSourceType SupportedType => DataSourceType.CSV;
    public bool HasHeader { get; set; } = true;
    public ParseMode ParseMode { get; set; } = ParseMode.Lenient;
    public IReadOnlyList<RowDiagnostic> LastDiagnostics => new List<RowDiagnostic>();

    public string GetDefaultExtension() => "csv";
    public void Configure(IConnectionProperties props) { /* delimiter/encoding setup */ }
    public void ClearDiagnostics() { }

    public string[] ReadHeaders(string filePath) => new[] { "Id", "Name" };
    public EntityStructure GetEntityStructure(string filePath) => null;
    public IEnumerable<string[]> ReadRows(string filePath) { yield break; }
    public string InferFieldType(string current, string rawValue) => typeof(string).FullName;

    public bool CreateFile(string filePath, IReadOnlyList<string> headers) => true;
    public bool AppendRow(string filePath, IReadOnlyList<string> headers, IReadOnlyList<string> values) => true;
    public bool RewriteFile(string filePath, IReadOnlyList<string> headers, IEnumerable<IReadOnlyList<string>> rows) => true;
}

2) Discover and List Readers

var ds = dmeEditor.GetDataSource("MyFileConnection") as FileDataSource;
var host = (IFileDataSourceReaderHost)ds;

// discover=true triggers FileReaderRegistry discovery
var readers = host.GetAvailableReaderDescriptors(discover: true);
foreach (var r in readers)
{
    Console.WriteLine($"{r.DisplayName} | {r.FormatType} | {r.ImplementationTypeName}");
}

3) Select the New Reader Explicitly

var custom = host.GetAvailableReaderDescriptors(true)
    .First(r => r.ImplementationTypeName == "MyNamespace.MyFastCsvReader");

if (host.TrySwitchReader(custom.ReaderId, reconfigure: true, out var reason))
{
    ds.Openconnection();
}
else
{
    Console.WriteLine(reason);
}
Best practice for duplicate formats

If multiple readers target the same format (for example CSV), prefer switching by ImplementationTypeName or ReaderId instead of only DataSourceType to avoid ambiguity.

Usage Examples

List and Inspect Readers

var ds = dmeEditor.GetDataSource("MyFileConnection") as FileDataSource;
var host = ds as IFileDataSourceReaderHost;

var readers = host.GetAvailableReaderDescriptors(discover: true);
foreach (var r in readers)
{
    Console.WriteLine($"{r.ReaderId} | {r.DisplayName} | {r.FormatType} | {r.DefaultExtension}");
    Console.WriteLine($"  Impl: {r.ImplementationTypeName}");
}

Switch by Implementation Name

var ok = host.TrySwitchReader(
    "DuckDBDataSourceCore.FileReaders.DuckDbCsvFileReader",
    reconfigure: true,
    out var reason);

Console.WriteLine(reason);
if (ok) ds.Openconnection();

Switch by ReaderId

var csvCandidates = host.GetAvailableReaderDescriptors(true)
    .Where(r => r.FormatType == DataSourceType.CSV)
    .ToList();

var selected = csvCandidates.First(); // your UI/command selects one
if (host.TrySwitchReader(selected.ReaderId, true, out var reason))
{
    Console.WriteLine($"Switched to {selected.DisplayName}");
    ds.Openconnection();
}
else
{
    Console.WriteLine(reason);
}
