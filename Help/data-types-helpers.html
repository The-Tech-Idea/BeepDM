Architecture Overview

The DataTypesHelpers subsystem answers the question: "Given a database column type like NVARCHAR(50) in SQL Server, what is the equivalent .NET type, and how do I generate DDL for it?"

Originally a single 2,000-line monolith, it was refactored into a layered architecture with a thin facade, a caching lookup engine, a repository aggregator, and 26 partial-class files covering 100+ database systems.

Layered design

// Layer 1: Public Facade (DataTypeFieldMappingHelper)
// All methods delegate to specialized helpers below
DataTypeFieldMappingHelper.GetDataType("MyDb", field, editor);    // → DataTypeMappingLookup
DataTypeFieldMappingHelper.GetMappings();                          // → DataTypeMappingRepository
DataTypeFieldMappingHelper.GetSqlServerDataTypesMapping();        // → DatabaseTypeMappingRepository

// Layer 2: Core Engine (3 helpers)
DataTypeMappingLookup      // Lookup with thread-safe caching, size/precision substitution
DataTypeMappingRepository  // Routes DataSourceType/DataSourceName → filtered mapping lists
DataTypeBasicOperations    // .NET type lists, validation, custom converters

// Layer 3: Per-DB Factories (26 partial-class files)
// Each file contains type mapping lists for one category of databases
DatabaseTypeMappingRepository.GetSqlServerDataTypesMapping();
// → [ { NetDataType="System.String", DataType="nvarchar(N)", Fav=true }, ... ]

How Type Mappings Get Loaded

The bootstrap chain

// Stage 1: LoadConfigurations() seeds the in-memory mappings
EnvironmentService.AddAllDataSourceMappings(editor);
// → editor.ConfigEditor.DataTypesMap becomes a List<DatatypeMapping>
//   containing 1000+ entries from all 100+ database systems
//   Each entry: DataSourceName, NetDataType, DataType, Fav flag

// Stage 2: Persisted to disk as DataTypeMapping.json
//   Loaded/saved via ConfigEditor.ReadDataTypeFile() / SaveDataTypeFile()

// Stage 3: At runtime, DataTypeMappingLookup uses them:
//   "Given data source 'SqlServerDataSource', convert System.String → nvarchar(N)"
//   → Searches DataTypesMap for DataSourceName="SqlServerDataSource",
//     NetDataType="System.String" → returns DataType="nvarchar(N)"
The DatatypeMapping entity
PropertyTypeDescription
DataSourceNamestringMatches AssemblyClassDefinition.className (e.g. "SqlServerDataSource")
NetDataTypestringThe .NET/C# type (e.g. "System.String", "System.Int32")
DataTypestringThe database-native type. May contain placeholders: (N)=length, (P,S)=precision,scale
FavboolPreferred mapping — lookup prioritizes these when multiple matches exist

DataTypeBasicOperations

Provides the canonical list of .NET data types and utility methods for type validation and custom conversion.

Available .NET types

// Short list (16 primitive types)
string[] types = DataTypeBasicOperations.GetNetDataTypes();
// byte, sbyte, int, uint, short, ushort, long, ulong,
// float, double, char, bool, object, string, decimal, DateTime

// Full list (with System.* prefix, includes arrays and special types)
string[] fullTypes = DataTypeBasicOperations.GetNetDataTypes2();
// System.Byte[], System.SByte[], System.Byte, System.SByte,
// System.Int32, System.UInt32, System.Int16, System.UInt16,
// System.Int64, System.UInt64, System.Single, System.Double,
// System.Char, System.Boolean, System.Object, System.String,
// System.Decimal, System.DateTime, System.TimeSpan,
// System.DateTimeOffset, System.Guid, System.Xml

// Validate that a field's type can be mapped
bool valid = DataTypeBasicOperations.IsValidFieldMapping("MySqlDb", field, editor);

// Apply a custom converter on top of standard mapping
string result = DataTypeBasicOperations.GetCustomDataType("MyDb", field, editor,
    standardType => standardType switch {
        "System.String" => "TEXT",
        "System.Int32"  => "INTEGER",
        _ => standardType
    });

DataTypeMappingLookup — Core Engine

The heart of type resolution. Converts between database-native types and .NET types using the DataTypesMap collection with thread-safe caching, placeholder substitution for sizes/precision, and intelligent fallbacks.

Three concurrent caches for performance

// Cache 1: Mapping cache — caches full DatatypeMapping objects
private static readonly ConcurrentDictionary<string, DatatypeMapping> _mappingCache;

// Cache 2: Data type cache — caches string results (the most common lookup)
private static readonly ConcurrentDictionary<string, string> _dataTypeCache;

// Cache 3: Class-level cache — caches filtered lists per className
private static readonly ConcurrentDictionary<string, List<DatatypeMapping>> _classTypeMappingsCache;

// Clear all caches when mappings are reloaded
DataTypeMappingLookup.ClearCache();

// Get cache statistics for diagnostics
var stats = DataTypeMappingLookup.GetCacheStatistics();
// { MappingCacheCount: 34, DataTypeCacheCount: 152, ClassMappingsCacheCount: 8, ... }

Placeholder system — how sizes and precision work

// Mappings use placeholders that get substituted at lookup time:
//
//   (N)    → string length       e.g. nvarchar(N)  → nvarchar(50)
//   (N,S)  → precision & scale   e.g. decimal(N,S) → decimal(18,2)
//   (P,S)  → alt precision/scale e.g. number(P,S)  → number(10,4)

// Example: mapping System.String to SQL Server
// Stored mapping:  DataType = "nvarchar(N)", NetDataType = "System.String"
// When field.Size1 = 50:
//   → result = "nvarchar(50)"

// Example: mapping System.Decimal to SQL Server
// Stored mapping:  DataType = "decimal(P,S)", NetDataType = "System.Decimal"
// When field.NumericPrecision = 18, field.NumericScale = 2:
//   → result = "decimal(18,2)"

// Default String size when Size1 == 0:
//   → Falls back to editor.typesHelper.DefaultStringSize (usually 255)

Intelligent fallbacks

// When a mapping can't be found, fallbacks analyze the raw type name:

private static string GetFallbackDataType(EntityField fld)
{
    var FieldtypeLower = fld.Fieldtype.ToLowerInvariant();
    return FieldtypeLower switch
    {
        var type when type.Contains("int") =>
            fld.Size1 > 10 ? "System.Int64" : "System.Int32",
        var type when type.Contains("decimal") || type.Contains("numeric") =>
            "System.Decimal",
        var type when type.Contains("float") || type.Contains("double") =>
            "System.Double",
        var type when type.Contains("bool") =>
            "System.Boolean",
        var type when type.Contains("date") || type.Contains("time") =>
            "System.DateTime",
        var type when type.Contains("guid") =>
            "System.Guid",
        var type when type.Contains("byte") =>
            "System.Byte[]",
        _ => "System.String"   // ultimate default
    };
}

// Reverse fallback: .NET type → default SQL type
// System.Int32 → "int", System.String → "varchar(255)", etc.

Lookup algorithm — GetDataType(string DSname, EntityField fld, ...)

// Step 1: Check if field type is a System type (contains "System.")
if (!IsSystemType(fld.Fieldtype))
    // Non-System type → use GetFieldtypeWoConversion (direct match)
else
    // System type → resolve via class handler:
    //   a. Get classHandler from DataSourceClass → className
    //   b. Filter DataTypesMap to that className's entries
    //   c. For System.String: try (N) placeholder with Fav first
    //   d. For numeric types: try (P,S) or (N) placeholder with Fav first
    //   e. Fall back: first match regardless of Fav

// Step 2: Apply placeholder replacements
//   "(N)"   → $"({fld.NumericPrecision})"     // or default 255
//   "(P,S)" → $"({fld.NumericPrecision},{fld.NumericScale})"

// Step 3: Cache result by composite key
//   $"{DSname}|{fld.EntityName}|{fld.FieldName}|..."

DataTypeMappingRepository — Aggregator

Routes lookups and aggregates all 100+ database mapping lists into a single unified collection.

Two lookup paths

// Path A: by DataSource name string
// Used when you have an open data source
var mappings = DataTypeMappingRepository.GetDataTypes("MySqlDb", editor);
// → Looks up classHandler for "MySqlDb"
// → Filters DataTypesMap to entries where DataSourceName == className

// Path B: by DataSourceType enum
// Used when you know the DB type but haven't opened it yet
var mappings = DataTypeMappingRepository.GetDataTypes(DataSourceType.SqlServer, editor);
// → Looks up DataDriversClasses for DataSourceType.SqlServer
// → Gets classHandler string → filters DataTypesMap

// Path C: Get ALL mappings from every database
var all = DataTypeMappingRepository.GetAllMappings();
// → Returns 1000+ entries, used by LoadConfigurations() to seed the initial map

TypeHelper — Dynamic Type Activation

Uses compiled LINQ expression trees to create fast, reflection-free object activators. Supports up to 7 constructor arguments with generic type safety.

Usage

// Create an instance with no arguments
object obj = typeof(MyClass).GetInstance();

// With 1-2 typed constructor arguments
object obj = typeof(DbConnection).GetInstance(connectionString);
object obj = typeof(MyDataSource).GetInstance(name, logger);

// Get a compiled activator function (fastest path, reusable)
ObjectActivator<MyDataSource> activator = TypeHelper.GetActivator<MyDataSource>(ctor);
MyDataSource ds = activator("MyDb", logger, editor, dbType, errorObj);

Example: What a Mapping Looks Like

SQL Server mappings (DatabaseTypeMappingRepository.SqlServer.cs)

public static List<DatatypeMapping> GenerateSqlServerDataTypesMapping()
{
    var mappings = new List<DatatypeMapping>
    {
        // String mappings
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.String", DataType = "nvarchar(N)", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.String", DataType = "varchar(N)", Fav = false },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.String", DataType = "text", Fav = false },

        // Integer mappings
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Int32", DataType = "int", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Int16", DataType = "smallint", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Int64", DataType = "bigint", Fav = true },

        // Floating point
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Double", DataType = "float", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Decimal", DataType = "decimal(P,S)", Fav = true },

        // Binary / Boolean / DateTime
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Byte[]", DataType = "varbinary(max)", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Boolean", DataType = "bit", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.DateTime", DataType = "datetime2", Fav = true },
        new DatatypeMapping { DataSourceName = "SqlServerDataSource",
            NetDataType = "System.Guid", DataType = "uniqueidentifier", Fav = true },
    };
    return mappings;
}
Key: DataSourceName matches AssemblyClassDefinition.className

DataSourceName = "SqlServerDataSource" matches the className of the AssemblyClassDefinition discovered by AssemblyHandler when scanning DLLs for IDataSource implementations.

End-to-End Usage Scenarios

Scenario A: .NET type → database column type (for DDL generation)

// You have a C# entity field and need to create a SQL column
var field = new EntityField
{
    FieldName = "ProductName",
    Fieldtype = "System.String",
    Size1     = 100    // nvarchar(100)
};

string sqlType = DataTypeFieldMappingHelper.GetDataType(
    "MySqlServerDb", field, dmeEditor);
// → "nvarchar(100)"

// Generate CREATE TABLE statement
string ddl = $"CREATE TABLE Products ({field.FieldName} {sqlType})";
// → "CREATE TABLE Products (ProductName nvarchar(100))"

Scenario B: Database column type → .NET type (when reading schema)

// Reading a database schema, you have the raw column type
string rawDbType = "decimal(18,2)";  // from INFORMATION_SCHEMA.COLUMNS

// Convert to .NET type
string netType = DataTypeFieldMappingHelper.GetDataType(
    "MySqlServerDb", rawDbType, dmeEditor);
// → "System.Decimal"

// Use Type.GetType() to get the actual .NET Type
Type type = Type.GetType(netType);
// → typeof(decimal)

Scenario C: Getting all type mappings for a data source

// Show the type mapping UI for a connection wizard
var mappings = DataTypeFieldMappingHelper.GetDataTypes(
    "MySqlServerDb", dmeEditor);

foreach (var m in mappings)
{
    Console.WriteLine($"{m.NetDataType,-25} → {m.DataType,-20} {(m.Fav ? "★" : "")}");
}
// System.String            → nvarchar(N)          ★
// System.String            → varchar(N)
// System.Int32             → int                  ★
// System.Decimal           → decimal(P,S)         ★
// System.DateTime          → datetime2            ★
// ...

Scenario D: Schema generation across database types

// Generate DDL for the same entity on different database backends
var field = new EntityField
{
    FieldName = "Price",
    Fieldtype = "System.Decimal",
    NumericPrecision = 18,
    NumericScale = 2
};

// SQL Server
string sql = DataTypeFieldMappingHelper.GetDataType("MySqlServer", field, editor);
// → "decimal(18,2)"

// PostgreSQL
string pg = DataTypeFieldMappingHelper.GetDataType("MyPostgre", field, editor);
// → "numeric(18,2)"

// MySQL
string mysql = DataTypeFieldMappingHelper.GetDataType("MyMySql", field, editor);
// → "decimal(18,2)"

// SQLite (no decimal type, uses REAL)
string sqlite = DataTypeFieldMappingHelper.GetDataType("MySqlite", field, editor);
// → "real"

Scenario E: Custom type converter for specialized mapping

// Override the standard mapping for a specific use case
string result = DataTypeFieldMappingHelper.GetCustomDataType(
    "MyDb", field, editor,
    customTypeConverter: standardType =>
    {
        // Force all strings to use "TEXT" regardless of size
        if (standardType == "System.String") return "TEXT";
        // Force all decimals to use "REAL" for SQLite compatibility
        if (standardType == "System.Decimal") return "REAL";
        return standardType;
    });

Adding a New Database Type Mapping

Step-by-step

// 1. Choose or create the right partial file in DatabaseTypeMappingRepositories/
//    e.g., DatabaseTypeMappingRepository.SqlServer.cs for a new RDBMS

// 2. Add a Get*DataTypeMappings() method
public static List<DatatypeMapping> GetMyNewDbDataTypeMappings()
{
    return new List<DatatypeMapping>
    {
        // Use "MyNewDbDataSource" as DataSourceName (matches classHandler)
        new DatatypeMapping { DataSourceName = "MyNewDbDataSource",
            NetDataType = "System.String", DataType = "varchar(N)", Fav = true },
        new DatatypeMapping { DataSourceName = "MyNewDbDataSource",
            NetDataType = "System.Int32",  DataType = "int",  Fav = true },
        new DatatypeMapping { DataSourceName = "MyNewDbDataSource",
            NetDataType = "System.Decimal", DataType = "decimal(P,S)", Fav = true },
        // ... add all needed types
    };
}

// 3. Register it in GetAllMappings() (DataTypeMappingRepository.cs)
ls.AddRange(DatabaseTypeMappingRepository.GetMyNewDbDataTypeMappings());

// 4. Add a facade method in DataTypeFieldMappingHelper.cs
public static List<DatatypeMapping> GetMyNewDbDataTypeMappings()
{
    return DatabaseTypeMappingRepository.GetMyNewDbDataTypeMappings();
}

// The new mappings will be discoverable immediately after LoadConfigurations()
