Overview & File Structure

ConnectionHelper (namespace TheTechIdea.Beep.Helpers) is a static facade split across many partial classes. It bundles four orthogonal concerns — driver discovery, connection-string building, validation, security — together with a library of per-category ConnectionDriversConfig factory methods that pre-populate every field a driver needs (class names, NuGet package, connection-string template, icon, etc.).

Driver Linking

ConnectionDriverLinkingHelper matches an IConnectionProperties instance to the best available ConnectionDriversConfig registered in IConfigEditor.DataDriversClasses. The lookup chain is:

  1. Exact DriverName + DriverVersion
  2. DriverName only (latest available version)
  3. DatabaseType enum match
  4. File extension match (for Category == FILE)

Signatures

// In ConnectionHelper (facade) and ConnectionDriverLinkingHelper (implementation)
public static ConnectionDriversConfig LinkConnection2Drivers(
    IConnectionProperties cn, IConfigEditor configEditor);

public static List<ConnectionDriversConfig> GetDriversForDataSourceType(
    DataSourceType dataSourceType, IConfigEditor configEditor);

public static List<ConnectionDriversConfig> GetDriversForFileExtension(
    string fileExtension, IConfigEditor configEditor);

public static ConnectionDriversConfig GetBestMatchingDriver(
    IConnectionProperties connectionProperties, IConfigEditor configEditor);

Usage

// Resolve the driver config for an existing connection
var driver = ConnectionHelper.LinkConnection2Drivers(myConn, dmeEditor.ConfigEditor);
if (driver != null)
    Console.WriteLine($"Driver: {driver.PackageName} v{driver.version}");

// Find all drivers that handle SQLite
var sqliteDrivers = ConnectionHelper.GetDriversForDataSourceType(
    DataSourceType.SqlLite, dmeEditor.ConfigEditor);

// Resolve by file extension (e.g. "csv")
var csvDrivers = ConnectionHelper.GetDriversForFileExtension("csv", dmeEditor.ConfigEditor);

// Automatic best-match (tries all strategies)
var best = ConnectionHelper.GetBestMatchingDriver(myConn, dmeEditor.ConfigEditor);

Connection String Processing

ConnectionStringProcessingHelper fills placeholder tokens ({Host}, {Database}, {UserID}, {Password}, etc.) in a driver's connection-string template at connection-open time. It also normalises relative paths (./, ../) to absolute paths relative to IConfigEditor.ExePath.

Signatures

// Fills all {placeholders} in the driver's ConnectionString template
public static string ReplaceValueFromConnectionString(
    ConnectionDriversConfig DataSourceDriver,
    IConnectionProperties ConnectionProp,
    IDMEEditor DMEEditor);

// Converts a relative path to an absolute path under basePath
public static string NormalizePath(string relativePath, string basePath);

// Normalizes FilePath in-place on the connection properties object
public static void NormalizeFilePath(IConnectionProperties connectionProp, string basePath);

// Returns a list of {placeholder} names that have no matching value
public static List<string> ValidateRequiredPlaceholders(
    string connectionStringTemplate, IConnectionProperties connectionProp);

How placeholder substitution works

// Template in driver config:
// "Server={Host};Database={Database};User Id={UserID};Password={Password}"

var conn = new ConnectionProperties
{
    Host     = "db.example.com",
    DataBase = "sales",
    UserID   = "app",
    Password = "s3cr3t"
};

var driver = ConnectionHelper.LinkConnection2Drivers(conn, dmeEditor.ConfigEditor);
string cs = ConnectionHelper.ReplaceValueFromConnectionString(driver, conn, dmeEditor);
// "Server=db.example.com;Database=sales;User Id=app;Password=s3cr3t"

// Validate nothing is missing before opening
var missing = ConnectionHelper.ValidateRequiredPlaceholders(driver.ConnectionString, conn);
if (missing.Any())
    throw new InvalidOperationException($"Missing fields: {string.Join(", ", missing)}");
Priority order for input string

The helper resolves the base string in this order: ConnectionString → Url → FilePath + FileName. Relative paths that start with ./ are automatically expanded to absolute paths.

Connection String Security

ConnectionStringSecurityHelper masks secret values before connection strings are logged, displayed in UIs, or transmitted in error reports. It uses regex to locate and replace well-known secret keywords.

Signatures

// Replace passwords, API keys, access keys, tokens, and secrets with ********
public static string SecureConnectionString(string connectionString);

// Returns true if the string contains any of the masked keyword patterns
public static bool ContainsSensitiveInformation(string connectionString);

Usage

string raw = "Server=db;Database=sales;User Id=app;Password=MyP@ss!;ApiKey=key-abc123";

string safe = ConnectionHelper.SecureConnectionString(raw);
// "Server=db;Database=sales;User Id=app;Password=********;ApiKey=********"

// Guard before logging
if (ConnectionHelper.ContainsSensitiveInformation(raw))
    logger.WriteLog(ConnectionHelper.SecureConnectionString(raw));
else
    logger.WriteLog(raw);

Parameter Utilities

Defined in ConnectionHelper_GetParameterList.cs (partial class of ConnectionHelper). These methods parse semi-colon delimited connection strings as key=value dictionaries without opening any database connection.

Signatures

Usage

string cs = "Server=localhost;Database=Northwind;User Id=sa;Password=P@ssw0rd";

// Inspect

// ["Server", "Database", "User Id", "Password"]

// Remove a key
cs = ConnectionHelper.RemoveParameter(cs, "Password");

Category Config Factories

Each partial class file exposes a Get*Configs() method that returns a List<ConnectionDriversConfig> pre-populated with all known drivers for that category. Individual Create*Config() factory methods produce a single ConnectionDriversConfig with every field set (GUID, package name, DLL name, class handler, connection-string template, icon, flags, NuGet metadata).

These factories are used at startup to seed IConfigEditor.DataDriversClasses so that drivers are available for LinkConnection2Drivers even before NuGet packages are installed.

Liveness & Test Connection

Two classes handle connection health checks at different lifecycle stages:

  • ProxyLivenessHelper — tests a live, already-open IDataSource using the cheapest strategy for its category (ping query, metadata call, or connection-state check). Used by the proxy/watchdog subsystem.
  • TestConnectionHelper — async setup-time test against a ConnectionDriversConfig object before a connection is registered (e.g., during the connection wizard UI).

Probe strategy by category

// ProxyLivenessHelper — called by proxy and watchdog internally
bool alive = ProxyLivenessHelper.IsAlive(ds, timeoutMs: 3_000);
// Strategy selected by ds.Category:
//  RDBMS      → cheapest engine-specific ping query (SELECT 1, etc.)
//  NoSQL      → GetEntitesList (metadata)
//  FILE       → GetEntitesList
//  WEBAPI     → GetEntitesList (HEAD request equivalent)
//  CLOUD      → GetEntitesList
//  VectorDB   → GetEntitesList
//  IoT/Stream/Queue/Blockchain/Connector → ConnectionState signal
//  INMEMORY   → alive unless ConnectionState == Broken
//  default    → ConnectionState signal

TestConnectionHelper — async liveness via IDataSource

// Tests a live DMEEditor-managed datasource
var ds = dmeEditor.GetDataSource("MyDb");
ds.Openconnection();

var (ok, msg) = await TestConnectionHelper.TestConnectionAsync(ds);
Console.WriteLine($"{ok}: {msg}");
// "True: MyDb (RDBMS) is alive"

TestConnectionHelper — setup-time config test

// Test a NEW connection config before registering it
var config = new ConnectionDriversConfig
{
    DatasourceCategory = DatasourceCategory.RDBMS,
    DatasourceType     = DataSourceType.SqlServer,
    ConnectionString   = "Server=localhost;Database=test;Trusted_Connection=True"
};

var (ok, msg) = await TestConnectionHelper.TestConnectionAsync(config, timeout: 10);
if (!ok)
    MessageBox.Show($"Cannot connect: {msg}");
else
    dmeEditor.ConfigEditor.AddDataConnection(myConnectionProps);
Never throws

Both helpers catch all exceptions internally and map them to (false, message). It is safe to call them without a try-catch.

The Analysis/ sub-folder contains two static helpers for inspecting POCO type graphs and inferring relational structure — a prerequisite when converting C# object models to EntityStructure instances.

RelationshipInferencer

Determines RelationshipCardinality from POCO property graphs using a four-tier priority: explicit attributes → naming conventions → type analysis → bidirectional detection.

MethodReturnsDescription
InferCardinality(PropertyInfo sourceNavProperty, Type sourceType, Type targetType) RelationshipCardinality Infers the cardinality between source and target types via the given navigation property. Returns: OneToOne, OneToMany, ManyToOne, ManyToMany, SelfReference, or Unknown.

Inference Priority

  1. Explicit attributes — [ForeignKey], [InverseProperty]
  2. Naming conventions — properties ending in Id / ID
  3. Type analysis — ICollection<T> on one side implies One-to-Many
  4. Bidirectional detection — matching navigation property on the target type
// Check whether Order.Customer is a collection or single ref
var prop = typeof(Order).GetProperty("Customer");
bool isCollection = NavigationPropertyDetector.IsCollectionNavigation(prop);
// false — single reference

// Infer cardinality between Order and Customer
var card = RelationshipInferencer.InferCardinality(
    typeof(Order).GetProperty("Customer"),
    typeof(Order),
    typeof(Customer));
// → RelationshipCardinality.ManyToOne

POCO-to-Entity Conversion

PocoToEntityConverter (in Conversion/) uses reflection and DataAnnotations to automatically build a fully-configured EntityStructure from any C# class — eliminating the need to hand-craft entity definitions.

Key Detection Strategy

StrategyBehaviour
AttributeOnlyOnly properties decorated with [Key] are treated as primary keys.
ConventionOnlyProperties named Id or {ClassName}Id are auto-detected as keys.
AttributeThenConvention (default)Tries [Key] first; falls back to naming convention if no attribute is found.

Supported DataAnnotations

AttributeEffect on EntityField
[Key]Sets IsKey = true
[Required]Sets AllowDBNull = false
[StringLength(n)]Sets FieldSize = n
[MaxLength(n)]Sets FieldSize = n
[Range(min, max)]Stores range metadata on the field
[Column(name)]Overrides the field name to the column alias

Circular Reference Guard

The converter tracks visited types and halts recursion when a cycle is detected, preventing infinite loops in deeply nested object graphs. 

public class Product
{
    [Key]
    public int ProductId { get; set; }

    [Required, StringLength(100)]
    public string Name { get; set; }

    [Range(0, double.MaxValue)]
    public decimal Price { get; set; }

    public string Description { get; set; }
}

// Convert to EntityStructure in one call
EntityStructure entity = PocoToEntityConverter.ConvertPocoToEntity<Product>(
    strategy: PocoToEntityConverter.KeyDetectionStrategy.AttributeThenConvention,
    entityName: "Products");

// entity.EntityName  → "Products"
// entity.Fields[0]   → ProductId  (IsKey=true, AllowDBNull=false)
// entity.Fields[1]   → Name       (AllowDBNull=false, FieldSize=100)
// entity.Fields[2]   → Price      (range metadata attached)
// entity.Fields[3]   → Description (nullable)

// Then feed directly into any helper
IDataSourceHelper helper = new GeneralDataSourceHelper(
    DataSourceType.Postgre, dmeEditor);
var (createSql, ok, err) = helper.GenerateCreateTableSql(entity);
