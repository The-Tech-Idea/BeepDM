Overview

EnvironmentService is a static utility class providing cross-platform folder management and application initialization. It is the foundation for all BeepDM application data storage.

  • Cross-platform paths — Windows, macOS, and Linux folder locations via RuntimeInformation
  • Folder hierarchy — Main Beep folder → App Repo folder → Application folders
  • Initialization — Bootstraps ConfigEditor with default drivers, type mappings, and queries
  • Sanitization — Cross-platform safe folder names (invalid chars → underscores)
  • Idempotency — All methods track whether they've been called and skip duplicates

Folder Creation Flow

  1. GetBaseFolderPath() — Determines OS-appropriate base directory
  2. CreateDirectorySafely() — Creates folders with error handling
  3. SanitizeFolderName() — Cross-platform safe naming
  4. Auto-creates parent directories as needed

Folder Name Sanitization

private static string SanitizeFolderName(string folderName)
{
    // Replace invalid chars with underscores
    char[] invalidChars = Path.GetInvalidFileNameChars();
    foreach (char c in invalidChars)
        folderName = folderName.Replace(c, '_');

    // Trim whitespace
    folderName = folderName.Trim();

    // Enforce max length
    if (folderName.Length > 100)
        folderName = folderName.Substring(0, 100);

    return folderName;
}

This ensures folders work across Windows, macOS, and Linux without filesystem errors.

Usage Examples

Basic Setup

// Create folder hierarchy
var beepPath = EnvironmentService.CreateMainFolder();
Console.WriteLine($"Beep data: {beepPath}");

var repoPath = EnvironmentService.CreateAppRepofolder("MyApp");
Console.WriteLine($"App repo: {repoPath}");

var configPath = EnvironmentService.CreateAppfolder("MyApp", "Config");
Console.WriteLine($"Config: {configPath}");

Full Initialization with Editor

var editor = new DMEEditor();

// Create folders
EnvironmentService.CreateMainFolder();
EnvironmentService.CreateAppRepofolder("MyApp");

// Initialize configurations (all three at once)
editor.CreateBeepMapping();

// Or individually for more control:
editor.AddAllConnectionConfigurations();
editor.AddAllDataSourceMappings();
editor.AddAllDataSourceQueryConfigurations();

Checking Paths

Console.WriteLine($"Beep root: {EnvironmentService.BeepDataPath}");
Console.WriteLine($"App repo: {EnvironmentService.AppRepoDataPath}");

// On Windows:  C:\ProgramData\TheTechIdea\Beep
//             C:\ProgramData\TheTechIdea\Beep\MyApp
// On macOS:    ~/Library/Application Support/TheTechIdea/Beep
//             ~/Library/Application Support/TheTechIdea/Beep/MyApp
// On Linux:    ~/.config/TheTechIdea/Beep
//             ~/.config/TheTechIdea/Beep/MyApp

Cross-Platform Safety

// Unsafe name gets sanitized
var path = EnvironmentService.CreateAppfolder("My App: Data*");
// Result: creates folder named "My App_ Data_"

// Full automated setup (used by BeepService.Configure())
editor.CreateBeepMapping();
Console.WriteLine($"Drivers: {editor.ConfigEditor.DataDriversClasses.Count}");
Console.WriteLine($"Mappings: {editor.ConfigEditor.DataTypesMap.Count}");
Console.WriteLine($"Queries: {editor.ConfigEditor.QueryList.Count}");
