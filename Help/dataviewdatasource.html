DataViewDataSource

DataViewDataSource is a federated virtual data source that unifies entities from multiple underlying datasources into a single logical view. It implements IDataSource, IDataViewOperations, and IDisposable, and persists its definition as a JSON file via DMDataView.

The view can operate in two execution modes: Cached (data is materialised into a local in-memory engine and refreshed on a TTL) or DirectQuery (always fetched live from the backing sources). Cross-source joins are expressed as FederatedJoinDefinition records and are either detected automatically from foreign-key metadata or defined manually.

Architecture

A DataViewDataSource instance owns one IDMDataView (concrete type DMDataView). The view holds the list of EntityStructure objects (each referring back to a real datasource via DataSourceID) and a list of FederatedJoinDefinition records describing how entities are joined cross-source. 

DataViewDataSource (IDataSource + IDataViewOperations)
├── IDMDataView (DMDataView) — persisted as JSON
│   ├── List<EntityStructure>       — entities from 1..N datasources
│   ├── List<FederatedJoinDefinition> — cross-source joins
│   ├── ExecutionMode               — Cached | DirectQuery
│   ├── CacheTTLSeconds             — default 300 s
│   ├── LocalEngineConnectionName   — in-memory engine name
│   └── ViewName / ViewID / VID
│
├── DataViewConnection (IDataConnection, IsLogical=true)
│   └── No physical file; OpenConnection() always returns Open
│
└── Partial classes

Execution flow — Cached mode

  1. GetEntity() detects cross-source entities (multiple distinct DataSourceID values).
  2. RefreshCacheIfExpired() checks CacheLastRefresh + CacheTTLSeconds.
  3. If stale, PrepareMergedQueryDB() materialises all entities into the LocalEngineConnectionName in-memory store.
  4. The local engine then satisfies the query — including joins — as a native SQL operation.

Execution flow — DirectQuery mode / single-source

  1. Single-source entities are delegated directly to the backing IDataSource.
  2. File, URL, and query view types are passed through unchanged.

DataViewConnection

DataViewConnection implements IDataConnection for view-oriented sources. Because views are logical constructs, IsLogical = true means OpenConnection() immediately returns Open without touching the filesystem.

PropertyTypeNotes
InMemoryboolWhen true the view is backed only by in-memory data. Default false.
IsLogicalboolWhen true, OpenConnection() always succeeds without filesystem I/O. Set to true by the constructor.
ConnectionPropIConnectionPropertiesStandard connection properties (FileName, FilePath, DriverName, …).
ConnectionStatusConnectionStateTracks current state (Open / Closed).
DbConnIDbConnectionUnused for logical connections; reserved for future use.
 
// DataViewConnection.OpenConnection()
public ConnectionState OpenConnection()
{
    return OpenConn(); // fast-path when IsLogical=true → sets status=Open immediately
}

Constructor

public DataViewDataSource(
    string        datasourcename,
    IDMLogger     logger,
    IDMEEditor    pDMEEditor,
    DataSourceType pDatasourceType,
    IErrorsInfo   per);

The constructor performs the following steps:

  1. Assigns Logger, ErrorObject, DMEEditor.
  2. Sets DatasourceType = DataSourceType.Text, Category = DatasourceCategory.VIEWS.
  3. Creates a DataViewConnection with IsLogical = true.
  4. Resolves the file path from the configured FolderFileTypes.DataView folder, or from the path embedded in datasourcename.
  5. Looks up an existing ConnectionProperties entry for the file name. If none exists, creates a new entry with DriverName = "DataViewReader" and saves the connections config.
  6. Sets DataViewFile to the full absolute path of the JSON view definition.
Note: The view JSON is not loaded in the constructor. It is loaded lazily on first access via LoadView(), which is triggered by GetEntitesList() and other data methods when DataView.Entities.Count <= 2.

Smart routing in GetEntity

// Cross-source path (entities have different DataSourceIDs at root level)
bool isCrossSource = DataView.Entities
    .Where(e => e.ParentId == 0)
    .Select(e => e.DataSourceID)
    .Distinct(StringComparer.OrdinalIgnoreCase)
    .Count() > 1;

if (isCrossSource && ent.Viewtype == ViewType.Table)
{
    RefreshCacheIfExpired();
    var db = PrepareMergedQueryDB();
    return db?.GetEntity(EntityName, filter);
}

// Single-source fallback
IDataSource srcDs = GetDataSourceObject(EntityName);
return srcDs?.GetEntity(EntityName, filter);

Join Graph Intelligence

Defined in DataViewDataSource.JoinGraph.cs. Provides automatic FK-based join detection, BFS path-finding, and a graph adjacency export.

MethodSignatureDescription
AutoDetectJoins IErrorsInfo AutoDetectJoins() Iterates all entities, opens their backing datasource, reads EntityStructure.Relations, and adds matching FederatedJoinDefinition records for any FK relationships where both entities are present in the view. Skips duplicates and manually-defined joins. Calls InvalidateCache() on success.
FindJoinPath List<string> FindJoinPath(string fromEntityName, string toEntityName) BFS over the undirected join graph. Returns the shortest sequence of entity names connecting from to to, or null if no path exists.
GetJoinGraph Dictionary<string, List<string>> GetJoinGraph() Returns an adjacency list (undirected) keyed by uppercase entity name. All entities appear even if they have no edges (isolated nodes).

// Auto-detect joins from FK metadata
view.AutoDetectJoins();

// Find shortest join path between two entities
var path = view.FindJoinPath("CUSTOMERS", "ORDERITEMS");
// → ["CUSTOMERS", "ORDERS", "ORDERITEMS"]

// Inspect the graph
var graph = view.GetJoinGraph();
foreach (var kvp in graph)
    Console.WriteLine($"{kvp.Key} → [{string.Join(", ", kvp.Value)}]");

// Manually define a cross-source join
view.JoinDefinitions.Add(new FederatedJoinDefinition
{
    GuidID            = Guid.NewGuid().ToString(),
    LeftEntityName    = "CUSTOMERS",
    LeftColumn        = "CustomerID",
    LeftDataSourceID  = "SqliteDB",
    RightEntityName   = "CRMENTRIES",
    RightColumn       = "CustID",
    RightDataSourceID = "CrmRestApi",
    JoinType          = FederatedJoinType.Left,
    IsManuallyDefined = true
});

// INNER JOIN [ORDERITEMS] ON [ORDERS].[OrderID] = [ORDERITEMS].[OrderID]

Cache invalidation

InvalidateCache() is called automatically by any mutation that changes the view structure: adding/removing entities, changing filters, moving entities, and modifying joins. It resets CacheLastRefresh to DateTime.MinValue so that the next GetEntity() call triggers a fresh materialise.

Validation & Health

Defined in DataViewDataSource.Validation.cs. All methods are non-destructive — they open connections only to test reachability, never modify data.

MethodReturnDescription
ValidateView List<string> Top-level check: runs ValidateEntities + ValidateJoins and returns all error messages.
ValidateEntities List<string> For each entity: checks DataSourceID is non-empty, datasource is registered, and connection can be opened.
GetBrokenEntities List<string> Returns names of entities whose backing datasource cannot be opened.
GetUnconnectedEntities List<string> Returns entity names that are isolated in the join graph — no join edge to any other entity in the view.
DetectSchemaChanges List<string> Compares the stored EntityStructure.Fields against the live schema from the backing datasource. Returns a list of change descriptions (added / removed / type-changed fields). 
// Full view validation
var errors = view.ValidateView();
if (errors.Any())
{
    foreach (var e in errors) Console.WriteLine("ERROR: " + e);
}

// Identify entities that can't be reached
var broken = view.GetBrokenEntities();
Console.WriteLine($"Broken: {string.Join(", ", broken)}");

// Find isolated entities (no joins to rest of the graph)
var lone = view.GetUnconnectedEntities();
Console.WriteLine($"Unconnected: {string.Join(", ", lone)}");

// Check for upstream schema drift on one entity
var drift = view.DetectSchemaChanges("PRODUCTS");
if (drift.Any())
    Console.WriteLine("Schema changed: " + string.Join("; ", drift));

Examples

Example 1 — Create a federated view from two sources

// 1. Resolve the DataViewDataSource from DMEEditor
var view = (DataViewDataSource)dme.GetDataSource("SalesView.json");

// 2. Load the view definition
view.GetEntitesList();

// 3. Verify entities and auto-detect joins from FK metadata
view.AssignDataSourceToEntities(); // ensure DataSourceIDs are set
view.AutoDetectJoins();

// 4. Show the execution plan
Console.WriteLine(view.GetExecutionPlan());

// 5. Execute a cross-source query
var orders = view.GetEntity("ORDERS", new List<AppFilter>
{
    new AppFilter { FieldName = "Status", Operator = "=", FilterValue = "OPEN" }
});

Example 2 — Apply filters and check cache

// Narrow each participating entity before materialising
view.SetEntityFilter("ORDERS",    "OrderDate > '2025-01-01'");
view.SetEntityFilter("CUSTOMERS", "Region = 'EMEA'");

// Cache was just invalidated. Force async materialise.
await view.MaterializeAsync();

// Check that the cache is now warm
var (expired, _, refreshed, ttlLeft) = view.GetCacheStatus();
Console.WriteLine($"Cache fresh — {ttlLeft}s remaining (refreshed {refreshed:u})");

// Query now routes through the local engine
var rows = view.GetEntity("ORDERS", null);
Console.WriteLine($"Rows: {rows?.Count()}");

Example 3 — View health check and schema sync

var summary = view.GetViewSummary();

if (!summary.IsValid)
{
    Console.WriteLine("Validation errors:");
    summary.Warnings.ForEach(w => Console.WriteLine("  " + w));
}

if (summary.BrokenEntities.Any())
{
    Console.WriteLine("Broken entities (check datasource connections):");
    summary.BrokenEntities.ForEach(b => Console.WriteLine("  " + b));
}

if (summary.UnconnectedEntities.Any())
{
    Console.WriteLine("Isolated entities (no join path):");
    summary.UnconnectedEntities.ForEach(u => Console.WriteLine("  " + u));
}

// Sync schema for any entity whose upstream definition may have changed
foreach (string entityName in view.EntitiesNames)
{
    var drift = view.DetectSchemaChanges(entityName);
    if (drift.Any())
    {
        Console.WriteLine($"Schema drift on {entityName} — refreshing...");
        var changes = view.RefreshEntitySchema(entityName);
        changes.ForEach(c => Console.WriteLine("  Applied: " + c));
    }
}

Example 4 — Clone and merge views

// Create a test copy of the production view
var devView = view.CloneView("SalesView_Test");
// devView is a fully independent DMDataView — saving it will not affect the original

// Merge a partner view (e.g. Inventory entities) into the current view
var inventoryDef = JsonSerializer.Deserialize<DMDataView>(
    File.ReadAllText("InventoryView.json"));

view.MergeView(inventoryDef);

// Add a manual cross-datasource join for the newly merged entities
view.JoinDefinitions.Add(new FederatedJoinDefinition
{
    GuidID            = Guid.NewGuid().ToString(),
    LeftEntityName    = "ORDERS",
    LeftColumn        = "ProductID",
    LeftDataSourceID  = "SalesDB",
    RightEntityName   = "PRODUCTS",
    RightColumn       = "ProductID",
    RightDataSourceID = "InventoryDB",
    JoinType          = FederatedJoinType.Left,
    IsManuallyDefined = true,
    Description       = "Sales ↔ Inventory product link"
});

Console.WriteLine($"View now has {view.JoinDefinitions.Count} joins across {view.GetAllDataSourceIDs().Count} sources.");
