Architecture Overview

The Universal DataSource Helpers framework provides a single abstraction over 40+ datasource types. Instead of writing if (type == SqlServer) ... else if (type == MongoDB) ..., you get one IDataSourceHelper instance from the DataSourceHelperFactory and call the same methods regardless of the underlying database.

Before vs After

// BEFORE (fragmented, datasource-specific)
if (type == DataSourceType.SqlServer)
    sql = RDBMSHelper.GenerateInsertQuery(type, table, data);
else if (type == DataSourceType.MongoDB)
    cmd = MongoDBHelper.GenerateInsertCommand(table, data);
else if (type == DataSourceType.Redis)
    key = RedisHelper.BuildHashSet(table, data);

// AFTER (universal, one interface)
IDataSourceHelper helper = DataSourceHelperFactory.GetHelper(type, dmeEditor);
var (sql, params, success, error) = helper.GenerateInsertSql(table, data);
// Works for all 40+ datasources

IDataSourceHelper — The Contract

Source: DataManagementModelsStandard/Editor/IDataSourceHelper.cs

Every datasource helper implements this interface. The 12+ methods cover the full lifecycle of schema operations, DDL, DML, and type mapping.

Complete interface

public interface IDataSourceHelper
{
    DataSourceType SupportedType { get; set; }
    string Name { get; }
    DataSourceCapabilities Capabilities { get; }

    // === Schema Operations ===
    (string Query, bool Success) GetSchemaQuery(string userName);
    (string Query, bool Success) GetTableExistsQuery(string tableName);
    (string Query, bool Success) GetColumnInfoQuery(string tableName);

    // === DDL Operations ===
    (string Sql, bool Success, string ErrorMessage) GenerateCreateTableSql(
        EntityStructure entity, string schemaName = null, DataSourceType? dataSourceType = null);
    (string Sql, bool Success, string ErrorMessage) GenerateDropTableSql(
        string tableName, string schemaName = null);
    (string Sql, bool Success, string ErrorMessage) GenerateTruncateTableSql(
        string tableName, string schemaName = null);
    (string Sql, bool Success, string ErrorMessage) GenerateCreateIndexSql(
        string tableName, string[] columns, string indexName = null, bool unique = false);

    // === DML Operations ===
    (string Sql, Dictionary<string,object> Params, bool Success, string ErrorMessage)
        GenerateInsertSql(string tableName, Dictionary<string,object> data);
    (string Sql, Dictionary<string,object> Params, bool Success, string ErrorMessage)
        GenerateUpdateSql(string tableName, Dictionary data, Dictionary conditions);
    (string Sql, Dictionary<string,object> Params, bool Success, string ErrorMessage)
        GenerateDeleteSql(string tableName, Dictionary conditions);
    (string Sql, Dictionary<string,object> Params, bool Success, string ErrorMessage)
        GenerateSelectSql(string tableName, List<string> columns = null,
                          Dictionary<string,object> conditions = null);

    // === Utilities ===
    string QuoteIdentifier(string identifier);
    string MapClrTypeToDatasourceType(Type clrType);
    Type MapDatasourceTypeToClrType(string datasourceType);
    (bool Valid, List<string> Errors) ValidateEntity(EntityStructure entity);
}
Design principles
  • Every method returns tuples with Success and ErrorMessage — no exceptions thrown
  • Params dictionaries for DML enable proper parameterized queries
  • Optional dataSourceType on GenerateCreateTableSql enables cross-dialect DDL from one helper
  • QuoteIdentifier handles each datasource's quoting style ([brackets], "quotes", or `backticks`)

DataSourceHelperFactory — Resolution

Source: Core/DataSourceHelperFactory.cs

Maps any DataSourceType to the correct IDataSourceHelper implementation. 18 RDBMS types share one RdbmsHelper (with SupportedType set per dialect), while each NoSQL/API type has its own dedicated helper.

Usage

var factory = new DataSourceHelperFactory(dmeEditor);

// Get the helper for any datasource type
IDataSourceHelper helper = factory.GetHelper(DataSourceType.MongoDB);

// All 40+ types work — same API
IDataSourceHelper sqlHelper = factory.GetHelper(DataSourceType.SqlServer);
IDataSourceHelper redisHelper = factory.GetHelper(DataSourceType.Redis);
IDataSourceHelper cassHelper = factory.GetHelper(DataSourceType.Cassandra);
IDataSourceHelper restHelper = factory.GetHelper(DataSourceType.WebAPI);

// Each produces datasource-specific output from the same method call:
var (sql, params, ok, err) = redisHelper.GenerateInsertSql("users", data);
// → Redis HSET command, not SQL

DataSourceCapabilities & Capability Matrix

Before generating SQL for a datasource, check if it even supports the operation. The capability matrix maps 40+ datasources × 20 features for graceful degradation.

20 capability flags per datasource

public class DataSourceCapabilities
{
    public bool SupportsTransactions { get; set; }
    public bool SupportsJoins { get; set; }
    public bool SupportsAggregations { get; set; }
    public bool SupportsPaging { get; set; }
    public bool SupportsOrderBy { get; set; }
    public bool SupportsGroupBy { get; set; }
    public bool SupportsWindowFunctions { get; set; }
    public bool SupportsFullTextSearch { get; set; }
    public bool SupportsIndexes { get; set; }
    public bool SupportsForeignKeys { get; set; }
    public bool SupportsStoredProcedures { get; set; }
    public bool SupportsViews { get; set; }
    public bool SupportsTriggers { get; set; }
    public bool SupportsSequences { get; set; }
    public bool SupportsJson { get; set; }
    public bool SupportsXml { get; set; }
    public bool SupportsTtl { get; set; }
    public bool SupportsBulkOperations { get; set; }
    public bool SupportsPrimaryKeys { get; set; }
    public bool IsSchemaEnforced { get; set; }

    // Dynamic lookup
    public bool IsCapable(string capabilityName);
}

Usage — graceful degradation

// Get capabilities from helper
var caps = helper.Capabilities;

// Step 1: Check before doing something unsupported
if (!caps.SupportsJoins)
{
    // Fall back to multiple queries + client-side join
    LogWarning("JOIN not supported on this datasource");
}

// Step 2: Check from the static matrix directly
bool hasTransactions = DataSourceCapabilityMatrix.Supports(
    DataSourceType.Redis, "SupportsTransactions");
// → true (Redis Lua scripts provide atomicity via MULTI/EXEC)

bool hasJoins = DataSourceCapabilityMatrix.Supports(
    DataSourceType.Cassandra, "SupportsJoins");
// → false (Cassandra requires denormalization)

// Step 3: Get all datasources that support a feature
var ftDatasources = DataSourceCapabilityMatrix
    .GetDatasourcesSupportingCapability("SupportsFullTextSearch");
// → [Elasticsearch, MongoDB, ClickHouse, PostgreSQL, SQL Server, ...]

// Step 4: Get a summary of all capabilities
var summary = DataSourceCapabilityMatrix.GetCapabilitySummary();
// → Dictionary mapping datasource → supported capabilities

PocoToEntityConverter — POCO → EntityStructure

Automatically generates EntityStructure objects from C# POCO classes using reflection. Detects keys, extracts DataAnnotations, and prevents circular references.

Three key detection strategies

public enum KeyDetectionStrategy
{
    AttributeOnly,             // Only [Key] attribute
    ConventionOnly,            // "Id" / "{ClassName}Id" naming convention
    AttributeThenConvention    // Try attribute first, fall back to convention
}

Usage

public class Product
{
    [Key]
    public int ProductId { get; set; }

    [Required]
    [StringLength(100)]
    public string Name { get; set; }

    [Range(0, double.MaxValue)]
    public decimal Price { get; set; }

    public string Description { get; set; }
}

// One call to convert
var entity = PocoToEntityConverter.ConvertPocoToEntity<Product>(
    strategy: PocoToEntityConverter.KeyDetectionStrategy.AttributeThenConvention,
    entityName: "Products");

// Result: EntityStructure with:
//  EntityName = "Products"
//  Fields = [
//    { FieldName="ProductId", IsKey=true, Fieldtype="System.Int32" },
//    { FieldName="Name", IsKey=false, Fieldtype="System.String", Size1=100, IsRequired=true },
//    { FieldName="Price", Fieldtype="System.Decimal", NumericPrecision=18, NumericScale=2 },
//    { FieldName="Description", Fieldtype="System.String" }
//  ]

// Then feed the entity to any helper for DDL:
IDataSourceHelper helper = factory.GetHelper(DataSourceType.Postgre);
var (ddl, ok, err) = helper.GenerateCreateTableSql(entity);

Circular reference detection

// Detects and reports circular dependencies (e.g. Parent → Child → Parent)
var diagnostics = PocoToEntityConverter.GetCircularReferenceDiagnostics<MyClass>();
if (diagnostics.HasCircularReferences)
    foreach (var cycle in diagnostics.CircularPaths)
        Console.WriteLine($"Circular: {cycle}");

RdbmsHelper — 18 SQL Dialects

Source: RdbmsHelpers/RdbmsHelper.cs + .Constraints.cs + .Transactions.cs

Wraps the legacy static RDBMSHelper behind the IDataSourceHelper interface. Supports 18 SQL dialects via the SupportedType property.

var helper = new RdbmsHelper(dmeEditor);
helper.SupportedType = DataSourceType.Postgre;

var (ddl, ok, err) = helper.GenerateCreateTableSql(entity);
// → PostgreSQL DDL with SERIAL, TEXT, NUMERIC types

var (insert, params, ok, err) = helper.GenerateInsertSql("users", data);
// → "INSERT INTO users (...) VALUES (...) RETURNING id"

MongoDBHelper — Document Database

Source: MongoDBHelpers/MongoDBHelper.cs

Generates MongoDB aggregation pipelines and BSON-type documents instead of SQL.

var helper = factory.GetHelper(DataSourceType.MongoDB);

var (pipeline, params, ok, err) = helper.GenerateInsertSql("users", data);
// → Generates MongoDB insert document with ObjectId, BSON types

var (query, params, ok, err) = helper.GenerateSelectSql("users",
    columns: new List<string> { "name", "email" },
    conditions: new Dictionary<string,object> { ["age"] = 30 });
// → db.users.find({ age: 30 }, { name: 1, email: 1 })

RedisHelper — Key-Value Store

Source: RedisHelpers/RedisHelper.cs

Simulates table operations using Redis Hashes, Lua scripts for atomicity, and SCAN-based pagination.

var helper = factory.GetHelper(DataSourceType.Redis);

var (cmd, params, ok, err) = helper.GenerateInsertSql("user:1001", data);
// → HSET user:1001 name "John" email "john@example.com"

var (cmd, params, ok, err) = helper.GenerateSelectSql("user:*", columns, conditions);
// → SCAN-based lookup with pattern matching

CassandraHelper — Wide-Column Store

Source: CassandraHelpers/CassandraHelper.cs

Generates CQL (Cassandra Query Language) with composite primary keys and token-based pagination.

var helper = factory.GetHelper(DataSourceType.Cassandra);

var (cql, params, ok, err) = helper.GenerateCreateTableSql(entity);
// → CREATE TABLE keyspace.products (
//     id UUID, name TEXT, price DECIMAL,
//     PRIMARY KEY ((partition_key), clustering_column))

var (cql, params, ok, err) = helper.GenerateSelectSql("products",
    conditions: new Dictionary<string,object> { ["id"] = someUuid });
// → SELECT * FROM products WHERE id = ?

RestApiHelper — HTTP APIs

Source: RestApiHelpers/RestApiHelper.cs

Generates HTTP method operations (GET/POST/PUT/PATCH/DELETE) with query parameter building and JSON body generation.

var helper = factory.GetHelper(DataSourceType.WebAPI);

var (url, body, ok, err) = helper.GenerateInsertSql("https://api.example.com/products", data);
// → POST https://api.example.com/products
//    Body: {"name": "Widget", "price": 19.99}

var (url, params, ok, err) = helper.GenerateSelectSql(
    "https://api.example.com/products",
    conditions: new Dictionary<string,object> { ["category"] = "tools" });
// → GET https://api.example.com/products?category=tools

End-to-End Usage Scenarios

A: POCO → DDL for any database

// 1. Define your model
public class Customer { [Key] public int Id { get; set; }
    [StringLength(200)] public string Name { get; set; }
    public DateTime CreatedAt { get; set; } }

// 2. Convert to EntityStructure
var entity = PocoToEntityConverter.ConvertPocoToEntity<Customer>(
    KeyDetectionStrategy.AttributeThenConvention, "Customers");

// 3. Pick a target database and generate DDL
var factory = new DataSourceHelperFactory(dmeEditor);
var helper = factory.GetHelper(DataSourceType.Postgre);
var (ddl, ok, err) = helper.GenerateCreateTableSql(entity);

// 4. Execute
ds.ExecuteSql(ddl);

B: Cross-datasource capability-aware query

string BuildSafeQuery(DataSourceType type, IDMEEditor editor)
{
    var helper = new DataSourceHelperFactory(editor).GetHelper(type);
    var caps = helper.Capabilities;
    string query = "SELECT * FROM products";

    if (caps.SupportsFullTextSearch)
        query += " WHERE MATCH(description) AGAINST('search term')";
    else if (caps.IsSchemaEnforced)
        query += " WHERE description LIKE '%search term%'";
    else
        return "Full-text search not available for " + type;

    if (caps.SupportsPaging)
        query += " LIMIT 10";

    return query;
}

C: Entity validation across databases

var entity = /* from user input */;
var helper = factory.GetHelper(targetDbType);

var (valid, errors) = helper.ValidateEntity(entity);
if (!valid)
{
    foreach (var err in errors)
        Console.WriteLine($"Validation: {err}");
    return;
}

// Safe to proceed
var (ddl, ok, err) = helper.GenerateCreateTableSql(entity);

Implementing a New Datasource Helper

Step-by-step

// 1. Implement IDataSourceHelper
public class Neo4jHelper : IDataSourceHelper
{
    private readonly IDMEEditor _editor;
    public DataSourceType SupportedType { get; set; } = DataSourceType.Neo4j;
    public string Name => "Neo4j";
    public DataSourceCapabilities Capabilities =>
        DataSourceCapabilityMatrix.GetCapabilities(SupportedType);

    public Neo4jHelper(IDMEEditor editor) { _editor = editor; }

    // Implement all 12 interface methods with Cypher syntax:
    public (string, Dictionary<string,object>, bool, string)
        GenerateInsertSql(string table, Dictionary<string,object> data)
    {
        // Build: CREATE (n:Label {prop1: $p1, prop2: $p2})
        var labels = string.Join(", ", data.Keys.Select(k => $"{k}: ${k}"));
        var sql = $"CREATE (n:{table} {{{labels}}})";
        return (sql, data, true, null);
    }
    // ... implement remaining methods
}

// 2. Register in DataSourceHelperFactory
_helpers[DataSourceType.Neo4j] = (e) => new Neo4jHelper(e);

// 3. Configure capabilities in DataSourceCapabilityMatrix
// Neo4j: SupportsJoins=true (via MATCH), SupportsAggregations=true, etc.
