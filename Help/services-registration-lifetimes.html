Services Registration & Lifetimes

How to configure and register BeepService with Microsoft Dependency Injection for desktop, web, Blazor, and CLI applications.

Location: TheTechIdea.Beep.Container.Services — DataManagementEngineStandard/Services/.

Overview

BeepService is the top-level container that wires together all core BeepDM components (IDMEEditor, IConfigEditor, IAssemblyHandler, etc.) and exposes them through a single IBeepService interface suitable for dependency injection.

The services layer provides three registration styles:

StyleAPIBest for
Platform shortcutsAddBeepForDesktop, AddBeepForWeb, AddBeepForBlazorServer, AddBeepForBlazorWasmMost applications — sensible platform defaults
Options patternAddBeepServices(Action<BeepServiceOptions>)Fine-grained cross-platform control
Fluent builderAddBeepServices().WithDirectory(…).AsSingleton().Build()IntelliSense-driven, method-chained setup
Legacy simpleRegister(services, path, name, configType)Backward compatibility only

Configure()

Initialises all components. Called automatically by the DI registration helpers or manually for direct instantiation.

// Direct instantiation (no DI):
var beep = new BeepService();
beep.Configure(
    directorypath: AppContext.BaseDirectory,
    containername: "MyApp",
    configType: BeepConfigType.DataConnector,
    AddasSingleton: false
);

Initialisation order inside Configure():

  1. Sets AppRepoName, ConfigureationType, BeepDirectory
  2. Creates ErrorsInfo, DMLogger, JsonLoader
  3. Creates ConfigEditor (reads JSON configs from {BeepDirectory}/Beep)
  4. Creates Util, AssemblyHandler, DMEEditor
  5. Optionally registers all instances into IServiceCollection (singleton or scoped)
  6. Calls LoadConfigurations() — loads data-source configs, mappings

LoadServicesScoped() / LoadServicesSingleton()

// Called automatically by Configure(); exposed for advanced scenarios
beep.LoadServicesSingleton();  // registers IDMLogger, IConfigEditor, IDMEEditor, IUtil, IJsonLoader, IAssemblyHandler
beep.LoadServicesScoped();     // registers same interfaces using AddKeyedScoped

Desktop Registration

Use AddBeepForDesktop for WinForms, WPF, and any long-running desktop process. Always registers as Singleton.

DesktopBeepOptions

PropertyDefaultDescription
DirectoryPathAppContext.BaseDirectoryRoot data directory
AppRepoName"DesktopApp"Container name
ConfigTypeApplicationConfig mode
EnableAutoMappingtrueMaps at startup
EnableAssemblyLoadingtrueScans plugins
EnableProgressReportingtrueEnables UI progress hooks
EnableDesignTimeSupporttrueSupports VS designers
AutoInitializeFormsfalseAuto-init forms on startup
InitializationTimeout5 minMax startup wait

Registration

// In Program.cs (WinForms / WPF with IHost):
var host = Host.CreateDefaultBuilder(args)
    .ConfigureServices((context, services) =>
    {
        services.AddBeepForDesktop(opts =>
        {
            opts.DirectoryPath = AppContext.BaseDirectory;
            opts.AppRepoName = "MyDesktopApp";
            opts.EnableProgressReporting = true;
            opts.EnableAutoMapping = true;
            opts.EnableAssemblyLoading = true;
        });
    })
    .Build();

// Resolve after host is built:
var beepService = host.Services.GetRequiredService<IBeepService>();

With Progress Reporting

var progress = new Progress<PassedArgs>(args =>
{
    Console.WriteLine($"[{args.EventType}] {args.Messege}");
});

// If your desktop extension surfaces a UseBeepForDesktop method:
var beepService = host.UseBeepForDesktop(progress);
Desktop always forces ServiceLifetime.Singleton — the lifetime in BeepServiceOptions is overridden automatically by DesktopBeepOptions.ToBeepServiceOptions().

Web / ASP.NET Core Registration

Use AddBeepForWeb for ASP.NET Core Web API and MVC applications. Always registers as Scoped (one instance per HTTP request).

Registration & Middleware

// Program.cs — options pattern
builder.Services.AddBeepForWeb(opts =>
{
    opts.DirectoryPath = Path.Combine(builder.Environment.ContentRootPath, "Beep");
    opts.AppRepoName = "MyWebApi";
    opts.EnableConnectionPooling = true;
    opts.EnableRequestIsolation = true;
});

var app = builder.Build();
app.UseBeepForWeb();   // Registers connection cleanup middleware
app.MapControllers();
app.Run();

Fluent Builder (Web)

builder.Services.AddBeepForWeb()
    .InDirectory(Path.Combine(basePath, "Beep"))
    .WithAppRepo("MyWebApi")
    .WithConnectionPooling()
    .WithRequestIsolation()
    .WithApiDiscovery()
    .WithMapping()
    .Build();

Controller Injection Example

[ApiController]
[Route("api/[controller]")]
public class DataSourceController : ControllerBase
{
    private readonly IBeepService _beep;
    public DataSourceController(IBeepService beep) => _beep = beep;

    [HttpGet("list")]
    public IActionResult GetDataSources()
    {
        var sources = _beep.DMEEditor.DataSources
            .Select(ds => new { ds.DatasourceName, Type = ds.DatasourceType.ToString() });
        return Ok(sources);
    }
}

Blazor Registration

Two separate extension methods cover the two Blazor hosting models.

BlazorBeepOptions

PropertyDefaultDescription
DirectoryPath{BaseDir}/BeepRoot data directory
AppRepoName"BlazorApp"Container name
ServiceLifetimeSet per methodScoped (Server) or Singleton (WASM)
EnableSignalRProgressfalseReal-time progress via SignalR (Server)
UseBrowserStoragefalseBrowser localStorage/sessionStorage (WASM)
EnableCircuitHandlersfalseCircuit lifecycle hooks (Server)
EnableAutoMappingtrueMaps at startup
EnableAssemblyLoadingtrueScans plugins

Blazor Server

// Program.cs — Blazor Server
builder.Services.AddBeepForBlazorServer(opts =>
{
    opts.DirectoryPath = Path.Combine(builder.Environment.ContentRootPath, "Beep");
    opts.AppRepoName = "MyBlazorServerApp";
    opts.EnableSignalRProgress = true;    // real-time progress via SignalR
    opts.EnableCircuitHandlers = true;    // cleanup on circuit disconnect
});
// Lifetime = Scoped (one per Blazor circuit)

Blazor WebAssembly

// Program.cs — Blazor WASM
builder.Services.AddBeepForBlazorWasm(opts =>
{
    opts.DirectoryPath = "Beep";          // browser virtual path
    opts.AppRepoName = "MyBlazorWasmApp";
    opts.UseBrowserStorage = true;        // persist config in localStorage
});
// Lifetime = Singleton (WASM has single user per process)

Blazor Component Usage

@inject IBeepService BeepService
@inject IJSRuntime JSRuntime

@code {
    private List<string> dataSources = new();

    protected override async Task OnInitializedAsync()
    {
        var ds = BeepService.DMEEditor.GetDataSource("mydb");
        if (ds != null)
        {
            var entities = await Task.Run(() => ds.GetEntities("", new List<ReportFilter>()));
            dataSources = entities.Select(e => e.EntityName).ToList();
        }
    }
}

Fluent Builder API

AddBeepServices() (no arguments) returns an IBeepServiceBuilder for method-chained configuration. This provides the best IntelliSense experience and is platform-agnostic.

IBeepServiceBuilder Methods

MethodDescription
WithDirectory(path)Sets DirectoryPath
WithAppRepo(name)Sets AppRepoName
WithConfigType(type)Sets ConfigType
WithMapping(enable)Enables/disables auto mapping
WithAssemblyLoading(enable)Enables/disables plugin scanning
WithTimeout(timeSpan)Sets initialization timeout
WithValidation(enable)Enables/disables startup validation
WithProperty(key, value)Adds an entry to AdditionalProperties
AsSingleton()Registers as Singleton lifecycle
AsScoped()Registers as Scoped lifecycle
AsTransient()Registers as Transient lifecycle
Build()Completes registration and returns IBeepService

Usage

// Desktop (singleton)
services.AddBeepServices()
    .WithDirectory(AppContext.BaseDirectory)
    .WithAppRepo("MyApp")
    .WithConfigType(BeepConfigType.Application)
    .WithMapping()
    .WithAssemblyLoading()
    .AsSingleton()
    .Build();

// Web (scoped)
services.AddBeepServices()
    .WithDirectory(Path.Combine(basePath, "Beep"))
    .WithAppRepo("MyWebApi")
    .WithProperty("EnableConnectionPooling", true)
    .AsScoped()
    .Build();

Options Pattern (alternative)

IBeepService beep = services.AddBeepServices(opts =>
{
    opts.DirectoryPath = AppContext.BaseDirectory;
    opts.AppRepoName = "MyApp";
    opts.ConfigType = BeepConfigType.DataConnector;
    opts.ServiceLifetime = ServiceLifetime.Singleton;
    opts.EnableAutoMapping = true;
    opts.EnableAssemblyLoading = true;
});

EnvironmentService

EnvironmentService is a static helper that resolves platform-appropriate data folder paths and populates BeepDM mapping configurations. It is called internally by BeepService.Configure() but can also be used directly.

Folder Creation

// Create the root Beep folder in a platform-appropriate location:
// Windows: %APPDATA%\TheTechIdea\Beep  (or AppContext.BaseDirectory\TheTechIdea\Beep)
// Linux/macOS: ~/.local/share/TheTechIdea/Beep
string mainFolder = EnvironmentService.CreateMainFolder();

// Create a named application repository sub-folder:
string appFolder = EnvironmentService.CreateAppRepofolder("MyApp");

// Extension form — callable on an IDMEEditor instance:
string path = dmeEditor.CreateMainFolder();

Beep Mapping Setup

// Registers all connection configs, datasource mappings, and query configs for an editor:
dmeEditor.CreateBeepMapping();
EnvironmentService.BeepDataPath and AppRepoDataPath are static read-only properties that cache the last created folder paths. Useful for diagnostics.

ServiceCollectionExtensions

Low-level IServiceCollection extensions for registering individual BeepDM components.

JsonLoader

// Singleton (default for long-lived apps)
services.AddJsonLoader();

// Scoped (per-request in web apps)
services.AddJsonLoaderScoped();

// Transient (new instance each time)
services.AddJsonLoaderTransient();

DMEEditor

// Register as singleton with existing handler + logger instances
services.AddDMEEditor(assemblyHandler, logger);
Prefer AddBeepForDesktop / AddBeepForWeb over these low-level methods. These extensions are intended for scenarios where you need granular control of individual component lifetimes.

ServiceHelper

ServiceHelper is a static service-locator that wraps a cached IServiceProvider. Use it sparingly — prefer constructor injection wherever possible. 

// Typically called once in Program.cs / App.xaml.cs after building the host:
ServiceHelper.Initialize(host.Services);

// Resolve a service anywhere, without constructor injection:
var beep = ServiceHelper.GetService<IBeepService>();
var editor = ServiceHelper.GetService<IDMEEditor>();
Do not call GetService before Initialize — it will throw a NullReferenceException. Initialize as early as possible in the application lifecycle.

Lifetime Selection Guide

Application typeRecommended lifetimeRegistration method
WinForms / WPF desktopSingletonAddBeepForDesktop
Console / CLISingletonAddBeepServices().AsSingleton().Build()
ASP.NET Core Web API / MVCScopedAddBeepForWeb
Blazor ServerScopedAddBeepForBlazorServer
Blazor WebAssemblySingletonAddBeepForBlazorWasm
Azure Functions (isolated)ScopedAddBeepServices().AsScoped().Build()
Background / Worker servicesSingletonAddBeepServices().AsSingleton().Build()
Never inject a Scoped IBeepService into a Singleton — this creates a captive dependency. If a singleton component needs BeepService, register BeepService as Singleton or use IServiceScopeFactory.

End-to-End Examples

Example 1 — WinForms with IHost

// Program.cs
[STAThread]
static void Main()
{
    var host = Host.CreateDefaultBuilder()
        .ConfigureServices((ctx, services) =>
        {
            services.AddBeepForDesktop(opts =>
            {
                opts.DirectoryPath = AppContext.BaseDirectory;
                opts.AppRepoName = "MyDesktopApp";
                opts.EnableProgressReporting = true;
            });
            services.AddSingleton<MainForm>();
        })
        .Build();

    ServiceHelper.Initialize(host.Services);
    var beepService = host.Services.GetRequiredService<IBeepService>();

    Application.Run(host.Services.GetRequiredService<MainForm>());
}

Example 2 — ASP.NET Core Web API

// Program.cs
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddBeepForWeb(opts =>
{
    opts.DirectoryPath = Path.Combine(builder.Environment.ContentRootPath, "Beep");
    opts.AppRepoName = "MyWebApi";
    opts.EnableConnectionPooling = true;
});

builder.Services.AddControllers();

var app = builder.Build();
app.UseBeepForWeb();
app.MapControllers();
app.Run();

Example 3 — Blazor Server

// Program.cs
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddRazorComponents().AddInteractiveServerComponents();
builder.Services.AddBeepForBlazorServer(opts =>
{
    opts.AppRepoName = "MyBlazorApp";
    opts.EnableSignalRProgress = true;
    opts.EnableCircuitHandlers = true;
});

var app = builder.Build();
app.MapRazorComponents<App>().AddInteractiveServerRenderMode();
app.Run();

Example 4 — Blazor WebAssembly

// Program.cs
var builder = WebAssemblyHostBuilder.CreateDefault(args);
builder.RootComponents.Add<App>("#app");

builder.Services.AddBeepForBlazorWasm(opts =>
{
    opts.AppRepoName = "MyBlazorWasmApp";
    opts.UseBrowserStorage = true;
});

await builder.Build().RunAsync();
