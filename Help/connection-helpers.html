Architecture Overview

The Connection Helpers subsystem is the metadata catalog for BeepDM. It defines everything the engine needs to know about every supported database, file format, cloud service, API protocol, and SaaS connector before any actual driver code is loaded.

Two-layer design

Bootstrapping & How Drivers Get Loaded

The connection helpers don't operate in isolation — they are the first step in a multi-stage bootstrap process that ultimately produces running IDataSource instances.

The complete bootstrap chain

// Stage 1: LoadConfigurations() seeds Metadata (ConnectionDriversConfig entries)
EnvironmentService.AddAllConnectionConfigurations(editor);
// → editor.ConfigEditor.DataDriversClasses now has 200+ entries
//   Each entry has: classHandler, PackageName, dllname, AdapterType,
//   ConnectionString template, DatasourceType, iconname, flags

// Stage 2: LoadAssemblies() discovers Runtime Code (AssemblyClassDefinition entries)
LLoader.LoadAllAssembly(progress, token);
// → Scans DLLs for [AddinAttribute] classes implementing IDataSource
// → editor.ConfigEditor.DataSourcesClasses now has matching Type entries

// Stage 3: classHandler → className bridge
//   ConnectionDriversConfig.classHandler == "SQLiteDataSource"
//   AssemblyClassDefinition.className == "SQLiteDataSource"
//   → DMEEditor resolves: string match → Type → IDataSource instance

The relationship between the two collections is documented in detail in the README Deep Dive section.

Driver Linking — ConnectionDriverLinkingHelper

Source: DataManagementEngineStandard/Helpers/ConnectionHelpers/ConnectionDriverLinkingHelper.cs

This is the resolver that answers the question: "Given these connection properties (host, database type, file extension, etc.), which driver configuration should I use?"

It searches IConfigEditor.DataDriversClasses — the 200+ entry catalog seeded by LoadConfigurations() — using a cascade of four matching strategies. Only LinkConnection2Drivers() and GetBestMatchingDriver() are the two public entry points; all other methods are public query helpers.

The Four-Strategy Cascade — How LinkConnection2Drivers() Works

When called, it tries each strategy in order. The first non-null result wins. This means more specific matches (exact package) take priority over generic ones (enum type).

Strategy 1: Exact Package Name + Version

// Source: FindDriverByPackageNameAndVersion() — line 55
// Matches when cn.DriverName AND cn.DriverVersion are both set
// e.g. cn.DriverName = "System.Data.SqlClient", cn.DriverVersion = "4.6.1.1"

private static ConnectionDriversConfig FindDriverByPackageNameAndVersion(
    IConnectionProperties cn, IConfigEditor configEditor)
{
    if (string.IsNullOrEmpty(cn.DriverName) || string.IsNullOrEmpty(cn.DriverVersion))
        return null;

    return configEditor.DataDriversClasses
        .FirstOrDefault(c =>
            c.PackageName.Equals(cn.DriverName, StringComparison.InvariantCultureIgnoreCase)
            && c.version == cn.DriverVersion);
}
// Used when the user explicitly picked a driver from a list
// and we know both the package name and version.

Strategy 2: Package Name Only

// Source: FindDriverByPackageName() — line 71
// Matches when only cn.DriverName is set (version unknown or "latest")

private static ConnectionDriversConfig FindDriverByPackageName(
    IConnectionProperties cn, IConfigEditor configEditor)
{
    if (string.IsNullOrEmpty(cn.DriverName))
        return null;

    return configEditor.DataDriversClasses
        .FirstOrDefault(c =>
            c.PackageName.Equals(cn.DriverName, StringComparison.InvariantCultureIgnoreCase));
}
// Typical for user-typed connections where the driver name is known
// but the version was left as default.

Strategy 3: DataSourceType Enum

// Source: FindDriverByDataSourceType() — line 91
// The fallback used 90% of the time. Matches by DatabaseType enum value.
// e.g. cn.DatabaseType = DataSourceType.SqlServer

private static ConnectionDriversConfig FindDriverByDataSourceType(
    IConnectionProperties cn, IConfigEditor configEditor)
{
    return configEditor.DataDriversClasses
        .FirstOrDefault(c => c.DatasourceType == cn.DatabaseType);
}
// Works because every Create*Config() factory sets DatasourceType explicitly.
// This is the simplest match: "I want SQLite" → finds the SQLite driver entry.

Strategy 4: File Extension (File category only)

// Source: FindDriverByFileExtension() — line 103
// Only attempted when cn.Category == DatasourceCategory.FILE
// Parses the extension from cn.FileName (e.g. "data.parquet" → "parquet")

private static ConnectionDriversConfig FindDriverByFileExtension(
    IConnectionProperties cn, IConfigEditor configEditor)
{
    if (string.IsNullOrEmpty(cn.FileName))
        return null;

    var driversWithExtensions = configEditor.DataDriversClasses
        .Where(p => !string.IsNullOrEmpty(p.extensionstoHandle))
        .ToList();

    if (!driversWithExtensions.Any())
        return null;

    string fileExtension = Path.GetExtension(cn.FileName)
        .Replace(".", "").ToLowerInvariant();
    if (string.IsNullOrEmpty(fileExtension))
        return null;

    return driversWithExtensions
        .FirstOrDefault(c =>
            c.extensionstoHandle.ToLowerInvariant().Contains(fileExtension));
}
// Example: FileName = "sales.parquet"
// → extension = "parquet"
// → finds the driver whose extensionstoHandle contains "parquet"

Strategy X (internal): Local-Only Driver

// Source: FindDriverByIsLocal() — line 80
// Finds a driver marked CreateLocal = true (e.g. SQLite, in-memory DBs)
// Not part of the public cascade but available internally

private static ConnectionDriversConfig FindDriverByIsLocal(
    IConnectionProperties cn, IConfigEditor configEditor)
{
    return configEditor.DataDriversClasses
        .FirstOrDefault(c => c.CreateLocal);
}

GetBestMatchingDriver() — Compatibility-First Variant

Unlike LinkConnection2Drivers() which returns any first match, GetBestMatchingDriver() first narrows the field to only compatible drivers (those whose DatasourceType, DatasourceCategory, and file extension all match the connection properties). This is the method used by the setup wizard and connection-test dialog.

Source

public static ConnectionDriversConfig GetBestMatchingDriver(
    IConnectionProperties connectionProperties, IConfigEditor configEditor)
{
    if (connectionProperties == null || configEditor == null)
        return null;

    // Step 1: Filter to compatible drivers only
    var allDrivers = configEditor.DataDriversClasses.ToList();
    var compatibleDrivers = allDrivers
        .Where(d => IsDriverCompatible(d, connectionProperties))
        .ToList();

    if (!compatibleDrivers.Any())
        return null;

    // Step 2: Within compatible set, prioritize exact package+version
    if (!string.IsNullOrEmpty(connectionProperties.DriverName)
        && !string.IsNullOrEmpty(connectionProperties.DriverVersion))
    {
        var exactMatch = compatibleDrivers.FirstOrDefault(d =>
            d.PackageName.Equals(connectionProperties.DriverName,
                StringComparison.InvariantCultureIgnoreCase)
            && d.version == connectionProperties.DriverVersion);
        if (exactMatch != null) return exactMatch;
    }

    // Step 3: Then package name only
    if (!string.IsNullOrEmpty(connectionProperties.DriverName))
    {
        var packageMatch = compatibleDrivers.FirstOrDefault(d =>
            d.PackageName.Equals(connectionProperties.DriverName,
                StringComparison.InvariantCultureIgnoreCase));
        if (packageMatch != null) return packageMatch;
    }

    // Step 4: Fall back to first compatible driver
    return compatibleDrivers.First();
}

IsDriverCompatible() — Three-Point Compatibility Check

public static bool IsDriverCompatible(
    ConnectionDriversConfig driverConfig,
    IConnectionProperties connectionProperties)
{
    if (driverConfig == null || connectionProperties == null)
        return false;

    // Check 1: DataSourceType must match
    if (driverConfig.DatasourceType != connectionProperties.DatabaseType)
        return false;

    // Check 2: DatasourceCategory must match
    if (driverConfig.DatasourceCategory != connectionProperties.Category)
        return false;

    // Check 3: For file connections, file extension must be supported
    if (connectionProperties.Category == DatasourceCategory.FILE
        && !string.IsNullOrEmpty(connectionProperties.FileName)
        && !string.IsNullOrEmpty(driverConfig.extensionstoHandle))
    {
        string fileExtension = Path.GetExtension(connectionProperties.FileName)
            .Replace(".", "").ToLowerInvariant();
        if (!string.IsNullOrEmpty(fileExtension)
            && !driverConfig.extensionstoHandle
                .ToLowerInvariant().Contains(fileExtension))
        {
            return false;
        }
    }

    return true;
}

End-to-End Usage Scenarios

Scenario A: Opening a known database by name

// User has a connection named "northwind.db" already configured.
// DMEEditor.GetDataSourceClass() calls LinkConnection2Drivers() internally:

// 1. Look up the connection from user config
var cn = dmEditor.ConfigEditor.DataConnections
    .FirstOrDefault(c => c.ConnectionName == "northwind.db");

// 2. Resolve which driver handles it
var driver = ConnectionHelper.LinkConnection2Drivers(cn, dmEditor.ConfigEditor);
// → cn.DatabaseType = DataSourceType.SqlLite
// → Strategy 3 kicks in → finds SQLite entry in DataDriversClasses
// → driver.classHandler = "SQLiteDataSource"
// → driver.ConnectionString template is now available

// 3. Then DMEEditor uses classHandler to find the IDataSource Type:
var asDef = dmEditor.ConfigEditor.DataSourcesClasses
    .FirstOrDefault(x => x.className == driver.classHandler);

// 4. Instantiate and open
Type t = dmEditor.assemblyHandler.GetType(asDef.type.AssemblyQualifiedName);
IDataSource ds = (IDataSource)Activator.CreateInstance(t, "northwind.db",
    dmEditor.Logger, dmEditor, dmEditor.ErrorObject, dmEditor);
ds.Openconnection();

Scenario B: Connection wizard — finding compatible drivers for a dropdown

// The user selected "SQLite" from a database type picker.
// Populate the driver version dropdown:

DataSourceType selectedType = DataSourceType.SqlLite;

var drivers = ConnectionHelper.GetDriversForDataSourceType(
    selectedType, dmEditor.ConfigEditor);

foreach (var d in drivers)
{
    Console.WriteLine($"{d.PackageName} v{d.version} " +
                      $"[{(d.IsMissing ? "Not installed" : "Ready")}]");
}
// Output:
//   System.Data.SQLite v4.6.1.1 [Ready]
//   Microsoft.Data.Sqlite v8.0.0 [Not installed]

Scenario C: Opening a file — auto-detecting format by extension

// User selects "C:\data\sales_2024.parquet"
string filePath = @"C:\data\sales_2024.parquet";

var conn = new ConnectionProperties
{
    ConnectionName = "Sales Data",
    FileName       = filePath,
    Category       = DatasourceCategory.FILE
};

// GetBestMatchingDriver will filter to FILE category,
// then find which driver handles ".parquet"
var driver = ConnectionHelper.GetBestMatchingDriver(conn, dmEditor.ConfigEditor);
// → Finds the Parquet driver entry in DataDriversClasses
// → driver.classHandler = "ParquetDataSource"

// Or get ALL drivers that can handle parquet:
var parquetDrivers = ConnectionHelper.GetDriversForFileExtension(
    "parquet", dmEditor.ConfigEditor);
// → Returns all FILE-category drivers whose extensionstoHandle contains "parquet"

Scenario D: Checking if a driver is compatible before use

// Before instantiating a data source, verify the driver is right

var myConn = new ConnectionProperties
{
    DatabaseType = DataSourceType.Mysql,
    Category     = DatasourceCategory.RDBMS
};

var candidateDriver = dmEditor.ConfigEditor.DataDriversClasses
    .FirstOrDefault(d => d.PackageName == "MySql.Data");

if (ConnectionHelper.IsDriverCompatible(candidateDriver, myConn))
{
    // Safe to use — DatasourceType and Category both match
    string cs = ConnectionHelper.ReplaceValueFromConnectionString(
        candidateDriver, myConn, dmEditor);
    // cs = "Server={host};Database={database};Uid={userid};Pwd={password}"
}
else
{
    Console.WriteLine("Driver mismatch — check DatabaseType and Category");
}

Connection String Processing

ConnectionStringProcessingHelper fills placeholder tokens ({Host}, {Database}, {UserID}, {Password}, etc.) in a driver's connection-string template at connection-open time.

How placeholder substitution works

// Template in driver config:
// "Server={Host};Database={Database};User Id={UserID};Password={Password}"

var conn = new ConnectionProperties
{
    Host     = "db.example.com",
    DataBase = "sales",
    UserID   = "app",
    Password = "s3cr3t"
};

var driver = ConnectionHelper.LinkConnection2Drivers(conn, dmeEditor.ConfigEditor);
string cs = ConnectionHelper.ReplaceValueFromConnectionString(driver, conn, dmeEditor);
// "Server=db.example.com;Database=sales;User Id=app;Password=s3cr3t"

// Validate nothing is missing before opening
var missing = ConnectionHelper.ValidateRequiredPlaceholders(driver.ConnectionString, conn);
if (missing.Any())
    throw new InvalidOperationException($"Missing fields: {string.Join(", ", missing)}");
Priority order for input string

ConnectionString → Url → FilePath + FileName. Relative paths starting with ./ are automatically expanded to absolute paths.

Connection String Security

ConnectionStringSecurityHelper masks secret values before connection strings are logged, displayed in UIs, or transmitted in error reports.

Usage

string raw = "Server=db;Database=sales;User Id=app;Password=MyP@ss!;ApiKey=key-abc123";

string safe = ConnectionHelper.SecureConnectionString(raw);
// "Server=db;Database=sales;User Id=app;Password=********;ApiKey=********"

// Guard before logging
if (ConnectionHelper.ContainsSensitiveInformation(raw))
    logger.WriteLog(ConnectionHelper.SecureConnectionString(raw));
else
    logger.WriteLog(raw);

// Selective mask — shows first/last 2 chars
string partial = ConnectionHelper.SelectiveMask(raw);
// "Server=db;Database=sales;User Id=ap**;Password=My****!;ApiKey=ke*c123"

Parameter Utilities

Parse and mutate semi-colon delimited connection strings as key=value dictionaries without opening any database connection.

Usage

string cs = "Server=localhost;Database=Northwind;User Id=sa;Password=P@ssw0rd";

// Inspect
var keys = ConnectionHelper.GetParameterList(cs);      // ["Server", "Database", "User Id", "Password"]
string db = ConnectionHelper.GetParameterValue(cs, "Database");  // "Northwind"

// Mutate — switch database without rebuilding the whole string
cs = ConnectionHelper.SetParameterValue(cs, "Database", "Sales");
cs = ConnectionHelper.SetParameterValue(cs, "Encrypt", "True");  // add new
cs = ConnectionHelper.RemoveParameter(cs, "Password");           // remove

Category Factory Overview

Each partial class file exposes a Get*Configs() method that returns a List<ConnectionDriversConfig> pre-populated with all known drivers for that category. Individual Create*Config() factory methods produce a single ConnectionDriversConfig with every field set (GUID, package name, DLL name, class handler, connection-string template, icon, flags, NuGet metadata).

These factories are used at startup to seed IConfigEditor.DataDriversClasses so that drivers are available for LinkConnection2Drivers even before NuGet packages or plugin DLLs are downloaded.

Liveness & Test Connection

Two classes handle connection health checks at different lifecycle stages:

  • ProxyLivenessHelper — tests a live, already-open IDataSource using the cheapest strategy for its category. Used by the proxy/watchdog subsystem.
  • TestConnectionHelper — async setup-time test against a ConnectionDriversConfig object before a connection is registered, or against an open IDataSource for runtime health checks.

Probe strategy by category (ProxyLivenessHelper)

bool alive = ProxyLivenessHelper.IsAlive(ds, timeoutMs: 3_000);
// Strategy selected by ds.Category:
//  RDBMS      → cheapest engine-specific ping query (SELECT 1, etc.)
//  NoSQL      → GetEntitesList (metadata)
//  FILE       → GetEntitesList
//  WEBAPI     → GetEntitesList (HEAD request equivalent)
//  CLOUD      → GetEntitesList
//  VectorDB   → GetEntitesList
//  IoT/Stream/Queue/Blockchain/Connector → ConnectionState signal
//  INMEMORY   → alive unless ConnectionState == Broken
//  default    → ConnectionState signal

TestConnectionHelper — runtime liveness via IDataSource

var ds = dmeEditor.GetDataSource("MyDb");
ds.Openconnection();

var (ok, msg) = await TestConnectionHelper.TestConnectionAsync(ds);
Console.WriteLine($"{ok}: {msg}");
// "True: MyDb (RDBMS) is alive"

TestConnectionHelper — setup-time config test

// Test a NEW connection config before registering it
var config = new ConnectionDriversConfig
{
    DatasourceCategory = DatasourceCategory.RDBMS,
    DatasourceType     = DataSourceType.SqlServer,
    ConnectionString   = "Server=localhost;Database=test;Trusted_Connection=True"
};

var (ok, msg) = await TestConnectionHelper.TestConnectionAsync(config, timeout: 10);
if (!ok)
    MessageBox.Show($"Cannot connect: {msg}");
else
    dmeEditor.ConfigEditor.AddDataConnection(myConnectionProps);
Never throws

Both helpers catch all exceptions internally and map them to (false, message). It is safe to call them without a try-catch.

Extending — Adding a New Driver

To add a new data source driver to the catalog:

Step-by-step

// 1. Choose the right partial file for your category
//    e.g., ConnectionHelper_RDBMS.cs for a new relational database

// 2. Add a Create*Config() factory method
public static ConnectionDriversConfig CreateMyNewDbConfig()
{
    return new ConnectionDriversConfig
    {
        GuidID            = Guid.NewGuid().ToString(),
        PackageName       = "MyNewDb.Driver",
        DriverClass       = "MyNewDb.Driver",
        version           = "1.0.0",
        dllname           = "MyNewDb.Driver.dll",
        AdapterType       = "MyNewDb.Driver.MyDbDataAdapter",
        CommandBuilderType= "MyNewDb.Driver.MyDbCommandBuilder",
        DbConnectionType  = "MyNewDb.Driver.MyDbConnection",
        DbTransactionType = "MyNewDb.Driver.MyDbTransaction",
        ConnectionString  = "Server={Host};Database={Database};User Id={UserID};Password={Password}",
        iconname          = "mydb.svg",
        classHandler      = "MyNewDbDataSource",    // ← Must match the IDataSource className
        ADOType           = true,
        DatasourceCategory= DatasourceCategory.RDBMS,
        DatasourceType    = DataSourceType.MyNewDb,
        IsMissing         = true,   // true until NuGet/download is done
        NuggetMissing     = true
    };
}

// 3. Register it in the category's Get*Configs() collector
public static List<ConnectionDriversConfig> GetRDBMSConfigs()
{
    var configs = new List<ConnectionDriversConfig>
    {
        CreateSqlServerConfig(),
        // ... existing drivers ...
        CreateMyNewDbConfig()           // ← Add here
    };
    return configs;
}

// 4. If it's a new DataSourceType, add the enum value
//    in DataSourceType.cs

// 5. The driver will now appear in DataDriversClasses
//    after LoadConfigurations() runs
classHandler is critical

The classHandler string must match the className of the AssemblyClassDefinition discovered by AssemblyHandler. If they don't match, dm.GetDataSource(...) will return null even though the driver appears in the catalog.
