Overview

InMemoryDataSource stores data in memory without external database dependencies, implementing both IDataSource and IInMemoryDB interfaces.

Use Cases

  • Unit testing — No external database setup needed
  • Prototyping — Rapid development without schema migrations
  • Runtime caches — Temporary data for workflows and lookups
  • Development — Local dev without persistent storage overhead
  • ETL staging — Intermediate data before transformation

Characteristics

Lifecycle

  1. Construction — Calls Createfolder(), sets up directory structure at %ProgramData%/TheTechIdea/Beep/InMemory/{name}/
  2. Open — Openconnection() calls OpenDatabaseInMemory()
  3. Load — LoadStructure() reads createscripts.json and InMemoryStructures.json
  4. Create — CreateStructure() runs ETL create script
  5. Load data — LoadData() runs ETL import script
  6. Save — SaveStructure() writes entities to JSON files
  7. Close — Closeconnection() sets ConnectionStatus to Closed
var props = new ConnectionProperties
{
    ConnectionName = "TempData",
    DatabaseType = DataSourceType.InMemory,
    Category = DatasourceCategory.INMEMORY
};
editor.ConfigEditor.AddDataConnection(props);
var ds = editor.GetDataSource("TempData");
ds.Openconnection(); // Calls OpenDatabaseInMemory → ConnectionState.Open

Structure Management

// Load from persisted JSON files
var result = ds.LoadStructure(progress, token);
// Reads crateescripts.json → CreateScript
// Reads InMemoryStructures.json → InMemoryStructures

// OnLoadStructure event fires after successful load

// Create entities from script
var result = ds.CreateStructure(progress, token);
// Runs DMEEditor.ETL.RunCreateScript() if structure not yet created
// Marks IsStructureCreated = true

// Save current state to files
var result = ds.SaveStructure();
// Copies Entities → InMemoryStructures
// Generates create scripts via DMEEditor.ETL.GetCreateEntityScript()
// Generates copy data scripts via DMEEditor.ETL.GetCopyDataEntityScript()
// Serializes to createscripts.json and InMemoryStructures.json
// OnSaveStructure event fires

Data Operations

// Load data from ETL import script
ds.CreateScript = importScript;
var result = ds.LoadData(progress, token);
// Runs DMEEditor.ETL.RunImportScript() if folder exists
// Marks IsLoaded = true

// Sync (placeholder, throws NotImplementedException in current version)
var result = ds.SyncData(progress, token);

// Refresh (placeholder)

// IDataSource CRUD (throwing stubs by default)
// These throw NotImplementedException unless overridden:

var result = ds.UpdateEntity("Customers", data);

CRUD stubs

Core CRUD methods are throwing stubs. Concrete implementations override these in datasource-specific subclasses (e.g., SQLiteDataSource).

Usage Examples

Basic Setup

var props = new ConnectionProperties
{
    ConnectionName = "TestData",
    DatabaseType = DataSourceType.InMemory,
    Category = DatasourceCategory.INMEMORY
};
editor.ConfigEditor.AddDataConnection(props);
var ds = editor.GetDataSource("TestData");
ds.Openconnection();

// Define schema
ds.Entities.Add(new EntityStructure
{
    EntityName = "TestTable",
    Fields = new List<EntityField>
    {
        new EntityField { FieldName = "Id", FieldType = "int" },
        new EntityField { FieldName = "Name", FieldType = "string" }
    }
});

With Persistence

ds.Entities.Add(entityDef);

// Save structure to disk
ds.SaveStructure();
// Files created at:
// %ProgramData%/TheTechIdea/Beep/InMemory/TestData/
//   createscripts.json
//   InMemoryStructures.json

// Later: reload from disk
ds.Openconnection();
ds.LoadStructure(progress, token);
// Entities restored from InMemoryStructures.json

ETL Staging

// Create ETL script for schema
ds.CreateScript = editor.ETL.GetCreateEntityScript(entities);
ds.CreateStructure(progress, token);

// Create import script and load data
ds.CreateScript = importScript;
ds.LoadData(progress, token);

    {

        var result = service.AddCustomer(new Customer { Name = "John" });
        Assert.IsTrue(result.Success);
    }

    [TestCleanup]
    public void Cleanup()
    {
        _ds.Closeconnection();
    }
}
