Overview

BeepService is the central bootstrap service that creates and wires together all core BeepDM components:

  • DMEEditor - Main data management orchestrator
  • ConfigEditor - Configuration persistence manager
  • AssemblyHandler - Plugin discovery and loading
  • DMLogger - Logging subsystem
  • JsonLoader - Configuration file I/O
  • Util/ErrorsInfo - Utilities and error handling

It implements IBeepService and IDisposable, with thread-safe initialization.

BeepService Class

public class BeepService : IBeepService, IDisposable
{
    public BeepService();
    public BeepService(IServiceCollection services);

    // Core Components
    public IDMEEditor DMEEditor { get; set; }
    public IConfigEditor Config_editor { get; set; }
    public IDMLogger lg { get; set; }
    public IUtil util { get; set; }
    public IErrorsInfo Erinfo { get; set; }
    public IJsonLoader jsonLoader { get; set; }
    public IAssemblyHandler LLoader { get; set; }
    public IServiceCollection Services { get; }

    // Configuration
    public string AppRepoName { get; }
    public BeepConfigType ConfigureationType { get; }
    public string BeepDirectory { get; }
    public AssemblyHandlerType AssemblyHandlerType { get; set; }

    // Legacy (deprecated)
    [Obsolete("Use AppRepoName instead")]
    public string Containername { get; }
}

Internal Initialization Flow

  1. Store AppRepoName and configuration type
  2. Create main folder via EnvironmentService.CreateMainFolder()
  3. Initialize ErrorsInfo, DMLogger, JsonLoader
  4. Create ConfigEditor with root path
  5. Create Util with logger, errors, config
  6. Create AssemblyHandler (Default or SharedContext)
  7. Create DMEEditor wired to all above
  8. Register DI services (singleton or scoped)
  9. Load configurations (drivers, mappings, queries, folders)

Assembly Loading

LoadAssemblies

// Synchronous
public void LoadAssemblies();
public void LoadAssemblies(Progress<PassedArgs> progress);

// Async wrapper
public async Task LoadAssembliesAsync(Progress<PassedArgs> progress);

Loading is idempotent - subsequent calls are no-ops after first load.

Internal Flow

  1. Thread-safe via _assembliesLock
  2. Calls AssemblyHandler.LoadAllAssembly(progress, token)
  3. Syncs loaded assemblies to ConfigEditor.LoadedAssemblies
  4. Calls RestoreDriverConfigAndAutoLoad()

RestoreDriverConfigAndAutoLoad

After assembly loading, automatically restores driver configurations and loads driver DLLs from local packages:

private void RestoreDriverConfigAndAutoLoad()
{
    // 1. Load connection driver configs
    Config_editor.LoadConnectionDriversConfigValues();

    // 2. For each driver with auto-load flag:
    foreach (var driver in Config_editor.DataDriversClasses
        .Where(ShouldAutoLoadDriver))
    {
        // Try loading from local package first
        if (LLoader.LoadDriverFromLocalPackage(driver, out _))
            continue;

        // Check if NuGet package is missing
        driver.NuggetMissing = !LLoader.HasLocalPackage(driver);
    }

    // 3. Update loaded assemblies list
    Config_editor.LoadedAssemblies = LLoader.Assemblies.Select(c => c.DllLib).ToList();
    Config_editor.SaveConnectionDriversConfigValues();
}

DI Service Registration

BeepService can register core components in the DI container:

Singleton Registration

public void LoadServicesSingleton()
{
    Services.AddSingleton<IDMLogger>(lg);
    Services.AddSingleton<IConfigEditor>(Config_editor);
    Services.AddSingleton<IDMEEditor>(DMEEditor);
    Services.AddSingleton<IUtil>(util);
    Services.AddSingleton<IJsonLoader>(jsonLoader);
    Services.AddSingleton<IAssemblyHandler>(LLoader);
}

Scoped Registration

public void LoadServicesScoped()
{
    Services.AddKeyedScoped<IDMLogger, DMLogger>("Logger");
    Services.AddKeyedScoped<IConfigEditor, ConfigEditor>("ConfigEditor");
    Services.AddKeyedScoped<IDMEEditor, DMEEditor>("Editor");
    Services.AddKeyedScoped<IUtil, Util>("Util");
    Services.AddKeyedScoped<IJsonLoader, JsonLoader>("JsonLoader");
    Services.AddKeyedScoped<IAssemblyHandler, AssemblyHandler>("AssemblyHandler");
}

Note: Scoped registration uses keyed services for resolution by name.

Assembly Handler Types

TypeClassWhen to Use
Default AssemblyHandler Standard applications. Each instance manages its own assembly context.
SharedContext SharedContextAssemblyHandler Multi-tenant or plugin-heavy apps. Assemblies are shared across instances, reducing memory. 
var service = new BeepService();
service.AssemblyHandlerType = AssemblyHandlerType.SharedContext;
service.Configure(path, "MyApp", BeepConfigType.Application);

Lifecycle

Thread Safety

  • _configLock - Guards configuration loading
  • _assembliesLock - Guards assembly loading
  • Configuration loading is idempotent (single call only)
  • Assembly loading is idempotent (single call only)

Disposal

public void Dispose()
{
    // Cancels assembly loading
    tokenSource?.Cancel();
    tokenSource?.Dispose();
    // Disposes DMEEditor and Logger
}

Usage Examples

Basic Bootstrap

var service = new BeepService();
service.Configure(
    directorypath: AppContext.BaseDirectory,
    containername: "MyApp",
    configType: BeepConfigType.DataConnector);

// Load plugins
service.LoadAssemblies();

// Use
var editor = service.DMEEditor;
var ds = editor.GetDataSource("MyDB");

With DI Container

var services = new ServiceCollection();
var service = new BeepService(services);
service.Configure(path, "MyApp", BeepConfigType.Application,
    AddasSingleton: true);

var provider = services.BuildServiceProvider();
var editor = provider.GetRequiredService<IDMEEditor>();

Desktop App with Progress

var service = new BeepService();
service.Configure(baseDir, "DesktopApp", BeepConfigType.Application);

var progress = new Progress<PassedArgs>(args =>
{
    Console.WriteLine($"Loading: {args.Messege}");
});

await service.LoadAssembliesAsync(progress);

Console.WriteLine("BeepDM initialized successfully");

Design-Time Support

// When in Visual Studio designer, BeepService auto-configures:
var service = new BeepService();
// Internally calls ConfigureForDesignTime() if design-time
// This enables drag-drop datasource browsing in WinForms/WPF designer
