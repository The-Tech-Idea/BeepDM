ClassCreator

The ClassCreator component provides dynamic class generation capabilities for creating entity classes from database schemas.

IClassCreator Interface

public interface IClassCreator
{
    IDMEEditor DMEEditor { get; set; }
    string outputFileName { get; set; }
    string outputpath { get; set; }

    string CreateClass(string classname, EntityStructure entity, string poutputpath,
        string NameSpacestring = "TheTechIdea.ProjectClasses", bool GenerateCSharpCodeFiles = true);

    string CreatePOCOClass(string classname, EntityStructure entity, string usingheader, string implementations,
        string extracode, string outputpath, string nameSpacestring = "TheTechIdea.ProjectClasses",
        bool GenerateCSharpCodeFiles = true);

    string CreateDLL(string dllname, List<EntityStructure> entities, string outputpath,
        IProgress<PassedArgs> progress, CancellationToken token,
        string NameSpacestring = "TheTechIdea.ProjectClasses", bool GenerateCSharpCodeFiles = true);

    Type CreateEntityTypeAtRuntime(EntityStructure entity,
        string namespaceString = "TheTechIdea.ProjectClasses");

    // Plus additional batch, EF Core conversion, and code-generation methods.
}

Dynamic Class Generation

// Access the class creator
var classCreator = dmeEditor.classCreator;

// Get entity structure from data source
var dataSource = dmeEditor.GetDataSource("CustomerDB");
var customerEntity = dataSource.GetEntityStructure("Customers", true);

// Generate POCO class code
string classCode = classCreator.CreatePOCOClass(customerEntity, "MyCompany.DataModels");

Console.WriteLine("Generated Class:");
Console.WriteLine(classCode);

// Output:
// namespace MyCompany.DataModels
// {
//     public class Customer
//     {
//         public int CustomerID { get; set; }
//         public string Name { get; set; }
//         public string Email { get; set; }
//         public DateTime CreatedDate { get; set; }
//     }
// }

// Create runtime type
Type customerType = classCreator.CreatePOCOType(customerEntity, "MyCompany.DataModels");

// Create instance dynamically
var customerInstance = Activator.CreateInstance(customerType);

// Set properties using reflection
customerType.GetProperty("Name")?.SetValue(customerInstance, "John Doe");
customerType.GetProperty("Email")?.SetValue(customerInstance, "john@example.com");

Batch Class Generation

// Generate classes for multiple entities
var entities = new List<EntityStructure>();

// Get all entities from a data source
var dataSource = dmeEditor.GetDataSource("NorthwindDB");
var entityNames = dataSource.GetEntitesList();

foreach (var entityName in entityNames)
{
    var entity = dataSource.GetEntityStructure(entityName, true);
    entities.Add(entity);
}

// Compile to assembly
Assembly generatedAssembly = classCreator.CompileTOAssembly(entities, "NorthwindModels");


// Or create DLL file
string dllPath = classCreator.CreateDLL(entities, @"C:\Output", "NorthwindModels");

Console.WriteLine($"Generated assembly with {entities.Count} entity classes");
Console.WriteLine($"DLL created at: {dllPath}");
💡 ClassCreator Features
  • POCO Generation: Create Plain Old CLR Objects from database schemas
  • Runtime Compilation: Compile classes to assemblies at runtime
  • Type Creation: Dynamically create .NET types for use in applications
  • Batch Processing: Generate multiple classes in a single operation
  • Database Layers: Generate DAL, EF Core DbContext, EntityConfiguration, and Repository classes
  • Web API: Scaffold ASP.NET Core controllers and Minimal API endpoints
  • Testing: Generate unit test templates and FluentValidation validators
  • UI & GraphQL: Generate Blazor components and GraphQL schema types
  • Reverse Mapping: Convert CLR types, DataTables, and POCO lists back to EntityStructure

ClassCreator — Modular Architecture & Helpers

ClassCreator is implemented as a set of C# partial classes, each delegating to a focused generator helper. All helpers accept an IDMEEditor and are initialized in the main constructor, so every partial has access to the full DME context.

Constructor & Helpers (ClassCreator.cs)

public ClassCreator(IDMEEditor dmeEditor)
{
    DMEEditor = dmeEditor ?? throw new ArgumentNullException(nameof(dmeEditor));

    // Core helpers (always initialized)
    _generationHelper   = new ClassGenerationHelper(dmeEditor);    // naming, type mapping, validation
    _pocoHelper         = new PocoClassGeneratorHelper(dmeEditor);  // POCO / domain classes
    _modernHelper       = new ModernClassGeneratorHelper(dmeEditor); // record / init-only classes

    // Specialized helpers
    _validationTestingHelper = new ValidationAndTestingGeneratorHelper(dmeEditor);
    _documentationHelper     = new DocumentationGeneratorHelper(dmeEditor);
    _uiComponentHelper       = new UiComponentGeneratorHelper(dmeEditor);
    _serverlessHelper        = new ServerlessGeneratorHelper(dmeEditor);
    _databaseHelper          = new DatabaseClassGeneratorHelper(dmeEditor);
    _webApiHelper            = new WebApiGeneratorHelper(dmeEditor);
    _pocoToEntityHelper      = new PocoToEntityGeneratorHelper(dmeEditor);
    _efCoreToEntityHelper    = new EfCoreToEntityGeneratorHelper(dmeEditor);
}

ClassCreator — Database & EF Core Generation

The ClassCreator.Database.cs partial (backed by DatabaseClassGeneratorHelper) generates data-access artifacts directly from EntityStructure metadata.

Data Access Layer

// Generate a repository-style DAL class for a single entity
var entity = dataSource.GetEntityStructure("Orders", true);
string dalPath = classCreator.GenerateDataAccessLayer(entity, @"C:\Output\DAL");
// Writes Orders.DAL.cs with CRUD methods wired to the Beep IDataSource API

EF Core DbContext

// Generate an EF Core DbContext for all entities in a data source
var entities = entityNames
    .Select(n => dataSource.GetEntityStructure(n, true))
    .ToList();

string ctxPath = classCreator.GenerateDbContext(
    entities,
    namespaceString: "MyApp.Data",
    outputPath: @"C:\Output\EFCore"
);
// Writes MyAppDbContext.cs with DbSet<T> properties and OnModelCreating overrides

EF Core Entity Configuration

// Generate IEntityTypeConfiguration<T> from an EntityStructure
string cfgPath = classCreator.GenerateEntityConfiguration(
    entity,
    namespaceString: "MyApp.Data.Configurations",
    outputPath: @"C:\Output\EFCore\Config"
);
// Writes OrdersConfiguration.cs with column mappings, key, and index declarations

Repository Pattern

// Generate interface + implementation
string repoPath = classCreator.GenerateRepositoryImplementation(
    entity,
    outputPath: @"C:\Output\Repositories",
    namespaceName: "MyApp.Repositories",
    interfaceOnly: false   // set true to emit only IOrdersRepository.cs
);
// Writes IOrdersRepository.cs and OrdersRepository.cs with async CRUD + filtering
When to use each artifact
  • DAL: Thin wrapper over IDataSource for non-EF scenarios or legacy migrations.
  • DbContext + EntityConfiguration: Full EF Core integration with explicit column/key mapping.
  • Repository: Clean separation layer for domain logic consuming either EF or direct Beep data sources.

ClassCreator — Testing & Validation Generation

The ClassCreator.Testing.cs partial (backed by ValidationAndTestingGeneratorHelper) generates xUnit/NUnit test class templates and FluentValidation validator classes from entity metadata.

Unit Test Class

// Generate an xUnit test scaffold for CRUD operations against an entity
string testPath = classCreator.GenerateUnitTestClass(entity, @"C:\Output\Tests");

// Generated class includes:
// - Arrange/Act/Assert patterns for Create, Read, Update, Delete
// - Mocked IDataSource via Moq
// - Constructor and field assertions for all EntityField entries

FluentValidation Validators

// Generate a FluentValidation AbstractValidator<T> from EntityStructure field metadata
string validatorPath = classCreator.GenerateFluentValidators(
    entity,
    outputPath: @"C:\Output\Validators",
    namespaceName: "MyApp.Validators"
);

// Fields with IsRequired=true emit .NotEmpty()
// Fields with Size1 > 0 emit .MaximumLength(n)
// Fields typed as email convention emit .EmailAddress()
// Nullable numeric fields emit .InclusiveBetween(min, max) if range metadata present
Entity validation is a precondition

Both methods call ValidateEntityStructure(entity) internally before delegating to the helper. If the entity has no fields, duplicate field names, or an empty EntityName, an ArgumentException is thrown describing all validation failures.

ClassCreator — POCO-to-Entity & Type Conversion

Two partial files handle reverse mapping — going from existing CLR types or data structures back to EntityStructure metadata that the rest of Beep can consume: ClassCreator.PocoToEntity.cs (namespace scanning) and ClassCreator.EntityExtensions.cs (EntitiesExtensions integration).

Namespace Scanning & POCO → EntityStructure

// Discover all POCO types in a namespace
List<Type> pocoTypes = classCreator.ScanNamespaceForPocos(
    "MyApp.Domain",
    assembly: Assembly.GetExecutingAssembly()
);

// Convert a single POCO type (with relationship detection)
EntityStructure entity = classCreator.ConvertPocoToEntity(
    typeof(Order),
    detectRelationships: true  // scans navigation properties for FK relationships
);

// Generate entity class code from the POCO (optionally write to file)
string code = classCreator.GenerateEntityClassFromPoco(
    typeof(Order),
    outputPath: @"C:\Output",
    namespaceString: "MyApp.Entities"
);

// Bulk conversion: every POCO in the namespace
List<string> codes = classCreator.GenerateEntityClassesFromNamespace(
    "MyApp.Domain",
    outputPath: @"C:\Output",
    targetNamespace: "MyApp.Entities"
);

From .NET Type / DataTable / List (EntityExtensions)

// From a CLR type
EntityStructure fromType = classCreator.CreateEntityStructureFromType(
    typeof(Customer), entityName: "Customer");

// Generic overload
EntityStructure fromGeneric = classCreator.CreateEntityStructureFromType<Customer>();

// From an ADO.NET DataTable
EntityStructure fromTable = classCreator.CreateEntityStructureFromDataTable(myDataTable);

// From a typed list (infers fields from item type)
EntityStructure fromList = classCreator.CreateEntityStructureFromList(customers, "Customer");

// Batch: multiple types at once
List<EntityStructure> all = classCreator.CreateEntityStructuresFromTypes(
    typeof(Customer), typeof(Order), typeof(Product));
Field category mapping

CreateEntityStructureFromType uses EntitiesExtensions.GetFieldCategory(Type) via DataTypesHelper to assign the correct FieldCategory (Numeric, String, Date, Boolean, etc.) to each generated field, ensuring downstream type-mapping and query generation work correctly.

The refactored ConfigEditor uses specialized managers for different configuration responsibilities.

ConfigEditor Structure

public class ConfigEditor : IConfigEditor
{
    // Specialized managers
    private readonly ConfigPathManager _pathManager;
    private readonly DataConnectionManager _connectionManager;
    private readonly QueryManager _queryManager;
    private readonly EntityMappingManager _entityManager;
    private readonly ComponentConfigManager _componentManager;
    private readonly MigrationHistoryManager _migrationHistoryManager;
    
    // Configuration properties
    public BeepConfigType ConfigType { get; set; } = BeepConfigType.Application;
    public string ConfigPath { get; set; }
    public List<ConnectionProperties> DataConnections
    {
        get => _connectionManager.DataConnections;
        set => _connectionManager.DataConnections = value;
    }
    public List<QuerySqlRepo> QueryList
    {
        get => _queryManager.QueryList;
        set => _queryManager.QueryList = value;
    }
    public List<ConnectionDriversConfig> DataDriversClasses

        get => _componentManager.DataDriversClasses;

Database-Specific Type Mapping

// Map .NET types to different database systems
var fieldMappings = new Dictionary<string, string>();

fieldMappings["SqlServer"] = typesHelper.GetDataType("SqlServer", typeof(decimal), 18, 2);
// Returns: "DECIMAL(18,2)"

fieldMappings["MySQL"] = typesHelper.GetDataType("MySQL", typeof(decimal), 18, 2);
// Returns: "DECIMAL(18,2)"

fieldMappings["Oracle"] = typesHelper.GetDataType("Oracle", typeof(decimal), 18, 2);
// Returns: "NUMBER(18,2)"

fieldMappings["PostgreSQL"] = typesHelper.GetDataType("PostgreSQL", typeof(decimal), 18, 2);
// Returns: "NUMERIC(18,2)"

// Validate type mappings
foreach (var mapping in fieldMappings)
{
    bool isValid = typesHelper.IsValidDataType(mapping.Key, mapping.Value);
    Console.WriteLine($"{mapping.Key}: {mapping.Value} - Valid: {isValid}");
}

File Utilities

Advanced file processing utilities for various file formats and operations.

FileHelper Utilities

// File operation utilities
var fileHelper = new FileHelper(dmeEditor);

// Detect file encoding
Encoding encoding = fileHelper.DetectFileEncoding(@"C:\Data\customers.csv");
Console.WriteLine($"File encoding: {encoding.EncodingName}");

// Get file metadata
var fileInfo = fileHelper.GetFileMetadata(@"C:\Data\customers.csv");
Console.WriteLine($"Size: {fileInfo.Size} bytes");
Console.WriteLine($"Created: {fileInfo.CreatedDate}");
Console.WriteLine($"Modified: {fileInfo.ModifiedDate}");

// Validate file format
bool isValidCsv = fileHelper.ValidateFileFormat(@"C:\Data\customers.csv", "CSV");
bool isValidExcel = fileHelper.ValidateFileFormat(@"C:\Data\customers.xlsx", "Excel");

// Copy file with progress
var progress = new Progress<FileProgressArgs>(args =>
{
    Console.WriteLine($"Copying: {args.BytesCopied}/{args.TotalBytes} ({args.PercentComplete:F1}%)");
});

await fileHelper.CopyFileAsync(@"C:\Source\largefile.dat", @"C:\Destination\largefile.dat", progress);

// Split large file into chunks
var chunks = await fileHelper.SplitFileAsync(@"C:\Data\hugefile.csv", 1024 * 1024 * 100); // 100MB chunks
Console.WriteLine($"File split into {chunks.Count} chunks");

// Merge chunks back
await fileHelper.MergeChunksAsync(chunks, @"C:\Data\merged_hugefile.csv");

CircuitBreaker Usage

// Create circuit breaker with configuration
var circuitBreaker = new CircuitBreaker(
    name: "DatabaseCircuitBreaker",
    failureThreshold: 5,           // Fail after 5 consecutive failures
    timeout: TimeSpan.FromMinutes(2), // Stay open for 2 minutes
    retryTimeout: TimeSpan.FromSeconds(30) // Try again after 30 seconds
);

// Execute operation with circuit breaker protection
async Task<List<Customer>> GetCustomersWithProtection()
{
    return await circuitBreaker.ExecuteAsync(async () =>
    {
        var dataSource = dmeEditor.GetDataSource("CustomerDB");
        
        if (dataSource.ConnectionStatus != ConnectionState.Open)
        {
            dataSource.Openconnection();
        }
        
        var customers = await dataSource.GetEntityAsync("Customers", null);
        return customers.Cast<Customer>().ToList();
    });
}

// Handle circuit breaker events
circuitBreaker.OnCircuitOpened += (sender, args) =>
{
    Console.WriteLine($"Circuit breaker opened: {args.Reason}");
    // Send alert, switch to cache, etc.
};

circuitBreaker.OnCircuitClosed += (sender, args) =>
{
    Console.WriteLine("Circuit breaker closed - service recovered");
};

circuitBreaker.OnCircuitHalfOpened += (sender, args) =>
{
    Console.WriteLine("Circuit breaker half-open - testing service");
};

// Use the protected method
try
{
    var customers = await GetCustomersWithProtection();
    Console.WriteLine($"Retrieved {customers.Count} customers");
}
catch (CircuitBreakerOpenException)
{
    Console.WriteLine("Service is currently unavailable - circuit breaker is open");
    // Fallback to cached data or alternative service
}
catch (Exception ex)
{
    Console.WriteLine($"Operation failed: {ex.Message}");
}
Circuit Breaker Best Practices
  • Appropriate Thresholds: Set failure thresholds based on service characteristics
  • Fallback Strategies: Always have fallback mechanisms when circuit is open
  • Monitoring: Monitor circuit breaker state and performance
  • Testing: Test circuit breaker behavior under various failure scenarios

Batch Processing Extensions

// Batch insert with progress reporting
var customers = new List<Customer>
{
    new Customer { Name = "John Doe", Email = "john@example.com" },
    new Customer { Name = "Jane Smith", Email = "jane@example.com" },
    // ... thousands more records
};

var dataSource = dmeEditor.GetDataSource("CustomerDB");

// Configure batch settings
var batchConfig = new BatchConfiguration
{
    BatchSize = 1000,
    MaxConcurrency = 4,
    ContinueOnError = true,
    EnableProgressReporting = true
}

// Progress reporting
var progress = new Progress<BatchProgressArgs>(args =>
{
    Console.WriteLine($"Batch Progress: {args.ProcessedCount}/{args.TotalCount} " +
                     $"({args.ProgressPercentage:F1}%) - Errors: {args.ErrorCount}");
});

// Execute batch insert
var result = await dataSource.BatchInsertAsync("Customers", customers, batchConfig, progress);

Console.WriteLine($"Batch insert completed:");
Console.WriteLine($"  Processed: {result.ProcessedCount}");
Console.WriteLine($"  Successful: {result.SuccessCount}");
Console.WriteLine($"  Errors: {result.ErrorCount}");
Console.WriteLine($"  Duration: {result.Duration}");

// Review errors if any
if (result.Errors.Any())
{
    Console.WriteLine("Errors encountered:");
    foreach (var error in result.Errors.Take(10)) // Show first 10 errors
    {
        Console.WriteLine($"  Record {error.RecordIndex}: {error.ErrorMessage}");
    }
}

DataConversionHelper

DataConversionHelper is an internal static class used internally by the engine whenever a DataTable needs to be materialized into a typed binding or list structure. It handles the standard edge-cases: DBNull, single-character strings, enums from strings, and safe numeric conversions.

Conversion methods

// Runtime-typed ObservableBindingList (type known only at runtime)
IBindingList ConvertDataTableToObservableList(DataTable dataTable, Type type);

// Strongly-typed ObservableBindingList<T> (compile-time type)
ObservableBindingList<T> ConvertDataTableToObservableBindingList<T>(DataTable dt)
    where T : Entity, new();

// Simple List<T>
List<T> ConvertDataTable<T>(DataTable dt);

// Row-level conversion (supports DBNull, char, enum, numeric ChangeType)
T GetItem<T>(DataRow dr);

// Guard helper
bool IsNumericType(Type type); // covers byte → decimal
DBNull & type-coercion rules
  • DBNull — value types get Activator.CreateInstance(propType) (i.e., default); reference types get null.
  • char — single-character strings are unwrapped to char automatically.
  • enum — string values are parsed with Enum.Parse.
  • numeric — Convert.ChangeType is attempted; falls back to default on failure.

FilterHelper

Translates a List<AppFilter> into a DataView row filter (fast, index-aware) or a LINQ predicate chain (flexible, streaming). Both paths support the same set of comparison operators.

API

// Returns DataView with RowFilter built from filters (empty filters = full view)
DataView ApplyFilters(DataTable records, List<AppFilter> filters);

// Returns IEnumerable<DataRow> filtered via LINQ predicates
IEnumerable<DataRow> ApplyFiltersEnumerable(DataTable records, List<AppFilter> filters);
Supported operators
Operator stringBehaviour
"equals" / "="Exact match
"contains"LIKE %value% (RowFilter) / Contains OrdinalIgnoreCase (LINQ)
">" / "<" / ">=" / "<="Numeric / date comparison
"<>" / "!="Not-equal
"between"Inclusive range using FilterValue (min) and FilterValue1 (max)

Example

var filters = new List<AppFilter>
{
    new AppFilter { FieldName = "Status",   Operator = "equals",  FilterValue = "Active" },
    new AppFilter { FieldName = "Total",    Operator = ">=",     FilterValue = 100 },
    new AppFilter { FieldName = "OrderDate",Operator = "between", FilterValue = new DateTime(2025,1,1),
                                                                   FilterValue1 = new DateTime(2025,12,31) }
};

// DataView path (preferred for DataGridView binding)
DataView view = FilterHelper.ApplyFilters(ordersTable, filters);

// LINQ path (preferred for server-side pagination)
var rows = FilterHelper.ApplyFiltersEnumerable(ordersTable, filters)
    .Skip(page * pageSize).Take(pageSize);

FileDataSourceHelper

Converts file-system paths into ConnectionProperties objects by auto-resolving the correct ConnectionDriversConfig for each file extension. Supports all DataSourceType values registered in ConfigEditor.DataDriversClasses.

API

// Create ConnectionProperties for multiple files (skips missing files and already-registered connections)
List<ConnectionProperties> CreateFileConnections(IDMEEditor dme, string[] filenames);

// Create a single ConnectionProperties from one file path
ConnectionProperties CreateFileDataConnection(IDMEEditor dme, string file);

"xml"      → DataSourceType.XML

Usage

// Register all CSV files in a folder as data connections
var files = Directory.GetFiles(@"C:\Data", "*.csv");
var connections = FileDataSourceHelper.CreateFileConnections(dmeEditor, files);
foreach (var cn in connections)
    dmeEditor.ConfigEditor.AddDataConnection(cn);

// Register a single file
var conn = FileDataSourceHelper.CreateFileDataConnection(dmeEditor, @"C:\Data\sales.parquet");
if (conn != null)
    dmeEditor.ConfigEditor.AddDataConnection(conn);
Driver selection precedence
  1. First driver whose extensionstoHandle contains the file extension and is marked Favourite = true.
  2. Highest-version driver matching the extension.
  3. Returns null (logged) if no matching driver is found.

ReflectionHelper

Provides high-performance compiled-Expression activators, runtime numeric checks, generic ExpandoObject projection, and field-mapping via reflection. Used internally by ETL, data conversion, and object-mapping pipelines.

API surface

Compiled activator pattern (hot-path object creation)

// One-time: compile the activator for Order's parameterless constructor
var ctor = typeof(Order).GetConstructor(Type.EmptyTypes);
var activator = ReflectionHelper.GetActivator<Order>(ctor);

// Hot-path reuse (avoids Activator.CreateInstance overhead in tight loops)
for (int i = 0; i < 100_000; i++)
{
    var order = activator(); // compiled IL — ~3x faster than Activator.CreateInstance
    // populate from DataRow ...
}
