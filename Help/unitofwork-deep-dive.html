UnitofWork<T> — Deep Dive

UnitofWork<T> is the primary transactional data-access component in BeepDM. It wraps an ObservableBindingList<T> (OBL) and a backing IDataSource to provide change-tracking, events, validation, undo/redo, export, and commit/rollback — all in a single, generic unit pattern.

It works in two modes:

  • Connected mode — backed by a live IDataSource (SQL, file, REST, DataView, …). Changes are persisted on Commit().
  • List mode — works purely in-memory on a pre-loaded ObservableBindingList<T>. No datasource I/O.
14 Lifecycle Events
Pre/Post hooks for Create, Insert, Update, Delete, Query, Commit, Edit, and Revert. Every hook supports Cancel.
Undo / Redo
Granular per-action undo/redo stack via OBL. Configurable depth, enable/disable at runtime.
Validation
Data Annotations + EntityStructure-based custom rules. Can block commit when errors exist.
Commit Order
FK-safe ordered commit: DeletesFirst, InsertsFirst, or UpdatesFirst.
Export / Import
DataTable, JSON stream, CSV stream — async, streaming, with optional header row.
Soft Delete & Concurrency
Configurable soft-delete field; optimistic concurrency with LastWriteWins or ThrowOnConflict.

Entity States

State is tracked automatically by ObservableBindingList<T>. The UnitofWorkStateHelper provides a facade over OBL's internal GetTrackingItem(entity).

StateWhen set
EntityState.UnchangedFreshly loaded or after AcceptChanges().
EntityState.AddedAfter New() or Add(entity).
EntityState.ModifiedAfter any property change on a tracked item.
EntityState.DeletedAfter Delete(entity).
 
// Read the state of a specific item
var state = uow._stateHelper.GetEntityState(customer);

// Force-mark as modified (useful when bypassing property change notification)
uow._stateHelper.MarkAsModified(customer);

// Accept all changes (clears dirty flag, resets all states to Unchanged)
uow.Units.AcceptChanges();

Revert a single item

// Revert one item to its original values (before the last AcceptChanges)
bool reverted = uow.RevertItem(customer);

// Async overload
bool reverted = await uow.RevertItemAsync(customer, cts.Token);

Subscribing to events

using var uow = new UnitofWork<Customer>(dme, "SalesDB", "Customers");

// ── Pre-events: inspect the record and optionally cancel ────────────────────

uow.PreCreate += (sender, args) =>
{
    var c = (Customer)args.Record;
    // Validate before the entity is added to Units
    if (string.IsNullOrWhiteSpace(c?.Name))
    {
        args.Cancel = true;
        args.Messege = "Customer name is required.";
    }
};

uow.PreInsert += (sender, args) =>
{
    var c = (Customer)args.Record;
    Console.WriteLine($"About to insert {c.Name} into datasource…");
    // Set args.Cancel = true to abort the INSERT to the datasource
};

uow.PreUpdate += (sender, args) =>
{
    var c = (Customer)args.Record;
    // Example: block updates for archived customers
    if (c.Status == "ARCHIVED")
    {
        args.Cancel = true;
        args.Messege = "Archived records cannot be modified.";
    }
};

uow.PreDelete += (sender, args) =>
{
    var c = (Customer)args.Record;
    Console.WriteLine($"Pre-Delete: {c.Name} (ID={c.ID})");
    // Example: require confirmation for premium customers
};

uow.PreQuery += (sender, args) =>
{
    Console.WriteLine("Loading data…");
    // args.Cancel = true would suppress the datasource round-trip
};

uow.PreCommit += (sender, args) =>
{
    // Final validation gate before any DB write
    if (!IsNetworkAvailable())
    {
        args.Cancel = true;
        args.Messege = "No database connection.";
    }
};

// ── Post-events: receive completed results ──────────────────────────────────

uow.PostCreate += (sender, args) =>
{
    var c = (Customer)args.Record;
    Console.WriteLine($"Entity created in memory: {c.Name}");
};

uow.PostInsert += (sender, args) =>
{
    var c = (Customer)args.Record;
    Console.WriteLine($"Inserted to DB: {c.Name}, new PK = {c.ID}");
};

uow.PostUpdate += (sender, args) =>
{
    var c = (Customer)args.Record;
    AuditLog.Record("UPDATE", c);
};

uow.PostDelete += (sender, args) =>
{
    var c = (Customer)args.Record;
    Console.WriteLine($"Deleted: {c.Name}");
};

uow.PostQuery += (sender, args) =>
{
    // sender is the Units collection after load
    Console.WriteLine($"Loaded {uow.Units.Count} rows.");
};

uow.PostCommit += (sender, args) =>
{
    Console.WriteLine("All changes committed — refreshing UI.");
    RefreshGrid();
};

// ── Property change ─────────────────────────────────────────────────────────
uow.PostEdit += (sender, args) =>
{
    Console.WriteLine($"Field '{args.PropertyName}' changed to '{args.PropertyValue}'");
    // Useful for live audit trails or conditional UI updates
};

// ── Revert notification ─────────────────────────────────────────────────────
uow.OnItemReverted += (sender, args) =>
{
    var c = (Customer)args.Record;
    Console.WriteLine($"Item reverted: {c.Name}");
};

Commit & Rollback

MethodSignatureDescription
Commit() Task<IErrorsInfo> Commit() Persists all pending changes: raises PreCommit, wraps datasource in a transaction, calls CommitAllAsync via OBL (insert→update→delete ordered by CommitOrder), calls PostCommit on success.
Commit(progress, token) Task<IErrorsInfo> Commit(IProgress<PassedArgs>, CancellationToken) Same with progress reporting and cancellation support.
CommitBatchAsync Task<CommitBatchResult> CommitBatchAsync(batchSize, progress, ct) Commits in fixed-size batches. Useful for very large collections to avoid single-transaction timeouts.
Rollback() Task<IErrorsInfo> Rollback() Calls DataSource.EndTransaction() then Units.RejectChanges(). Restores added/deleted items and original property values.

Commit internals

During Commit(), OBL's CommitAllAsync is called with three async lambdas (insert, update, delete). The _defaultsHelper applies audit defaults just before each DB call: 

// Simplified internal Commit flow:
// 1. Raise PreCommit — abort if Cancel = true
// 2. DataSource.BeginTransaction()
// 3. OBL.CommitAllAsync(
//        insertAsync: item => { ApplyInsertDefaults(item); InsertDoc(item); },
//        updateAsync: item => { ApplyUpdateDefaults(item); UpdateDoc(item); },
//        deleteAsync: item => { DataSource.DeleteEntity(entityName, item); },
//        CommitOrder)
// 4. If AllSucceeded → DataSource.Commit() → AcceptChanges per item
//    Else            → DataSource.EndTransaction() (rollback)
// 5. Clear InsertedKeys/UpdatedKeys/DeletedKeys
// 6. Raise PostCommit
// Typical commit with cancellable progress
var cts = new CancellationTokenSource();
var progress = new Progress<PassedArgs>(p =>
    Console.WriteLine($"Progress: {p.ParameterString1}"));

var result = await uow.Commit(progress, cts.Token);
if (result.Flag != Errors.Ok)
{
    Console.WriteLine("Commit failed: " + result.Message);
    await uow.Rollback();
}

Validation

Validation is powered by OBL's built-in validation framework, which combines Data Annotations ([Required], [MaxLength], …) with EntityStructure-based rules described in the schema metadata.

Method / PropertyDescription
IsAutoValidateEnabled When true, validation runs after every property change. Default: depends on OBL init.
BlockCommitOnValidationError When true (default), items with ValidationSeverity.Error failures are skipped during commit.
ValidateItem(item) Validates a single item. Returns a ValidationResult with IsValid, Errors, Warnings.
ValidateAll() Validates every item in Units. Returns an aggregate ValidationResult.
GetErrors(item) Returns List<ValidationError> for a specific item.
GetInvalidItems() Returns all items that currently have at least one validation error. 
// Enable auto-validate so grid cells show errors live
uow.IsAutoValidateEnabled = true;

// Manual validation before commit
var result = uow.ValidateAll();
if (!result.IsValid)
{
    foreach (var error in result.Errors)
        Console.WriteLine($"{error.FieldName}: {error.Message}");
    return;
}

// Get errors for a specific item
var errors = uow.GetErrors(customer);
foreach (var e in errors)
    Console.WriteLine($"[{e.Severity}] {e.FieldName}: {e.Message}");

// Find all broken items
var invalid = uow.GetInvalidItems();
Console.WriteLine($"{invalid.Count} items have validation errors");

Undo / Redo

Undo/Redo records every property change, Add, and Remove on the collection. It is off by default; enable it after construction.

MemberDescription
IsUndoEnabledEnable/disable the undo stack.
MaxUndoDepthMaximum number of actions to retain. Default 100. Older entries are evicted.
CanUndotrue when the undo stack is non-empty.
CanRedotrue when the redo stack is non-empty.
Undo()Reverts the last recorded action. Returns true if successful.
Redo()Re-applies the last undone action. Returns true if successful.
ClearUndoHistory()Empties both undo and redo stacks.
EnableUndo(bool, int maxDepth)Convenience: sets IsUndoEnabled and MaxUndoDepth in one call.
 
// Turn on undo at startup
uow.IsUndoEnabled = true;
uow.MaxUndoDepth  = 50;

// Or via the convenience method
uow.EnableUndo(true, maxDepth: 50);

// Edit + undo
uow.CurrentItem.Name = "Wrong Name";
Console.WriteLine($"CanUndo: {uow.CanUndo}"); // true
uow.Undo();
Console.WriteLine($"Name restored: {uow.CurrentItem.Name}");

// Redo the change
uow.Redo();

// Clear history (e.g. after a commit)
uow.ClearUndoHistory();

Computed Columns

Computed columns are evaluated on-demand per item. They are not persisted. Useful for display-only values derived from other fields.

MethodDescription

// Get all computed values in one call (for grid display)

}

Bookmarks

Bookmarks save a named cursor position in the collection so you can return quickly (similar to Oracle Forms or MS Access record navigation). 

// Save position before a search

// Navigate through results…
uow.GoToBookmark("SearchStart"); // jumps back

// Remove when done

// Clear all

Thread Safety, Freeze & Batch Update

FeatureAPIDescription
Thread-safe mode IsThreadSafe = true All OBL mutations are wrapped in a ReaderWriterLockSlim. Safe for concurrent read from UI thread and write from background thread.
Freeze Freeze() / Unfreeze(), IsFrozen Makes the collection read-only. Mutations throw InvalidOperationException. Useful during UI pagination rendering.
Batch Update BeginBatchUpdate() Returns an IDisposable. While open, CollectionChanged notifications are suppressed. On dispose, a single reset notification is fired. Supports nested scopes. 
// Batch insert 10 000 rows without 10 000 individual UI refreshes
using (uow.BeginBatchUpdate())
{
    foreach (var row in largeDataSet)
        uow.Add(row);
} // CollectionChanged fires once here

// Thread-safe cross-thread scenario
uow.IsThreadSafe = true;
Task.Run(() =>
{
    uow.Add(new Customer { Name = "Background Insert" });
    // Safe — writer lock acquired internally
});

Use the UnitOfWorkFactory when the entity type is only known at runtime (e.g. plugin-loaded entities or reflection-driven UI). For compile-time known types, use the generic constructors directly.

UnitOfWorkWrapper

UnitOfWorkWrapper wraps a dynamic UnitofWork instance and provides safe property access with fallback defaults (GetPropertySafely). It implements IUnitOfWorkWrapper which exposes all core operations through a non-generic contract — suitable for framework-level code that must work with heterogeneous entity types.

Advanced Features

DefaultsManager integration

When DefaultsManager is initialised, _defaultsHelper.ApplyInsertDefaults(entity) is called in New() and Add(), and _defaultsHelper.ApplyUpdateDefaults(entity) is called in Update() and inside the Commit insert/update lambdas. This automatically populates audit fields like CreatedDate, CreatedBy, ModifiedDate, etc. 

// Configure once at app startup
DefaultsManager.Initialize(dme);
DefaultsManager.SetColumnDefault(dme, "SalesDB", "Customers", "CreatedDate",  "NOW",      isRule: true);
DefaultsManager.SetColumnDefault(dme, "SalesDB", "Customers", "CreatedBy",    "USERNAME", isRule: true);
DefaultsManager.SetColumnDefault(dme, "SalesDB", "Customers", "ModifiedDate", "NOW",      isRule: true);
DefaultsManager.SetColumnDefault(dme, "SalesDB", "Customers", "IsActive",     "true",     isRule: false);

// Now New()/Add()/Update() automatically fill these fields
uow.New();
var c = uow.CurrentItem;
c.Name = "Alice";
// c.CreatedDate → DateTime.UtcNow, c.CreatedBy → Window user, c.IsActive → true

Change summary & query history

// Get a snapshot of all pending changes
ChangeSummary summary = uow.GetChangeSummary();
Console.WriteLine($"Inserts: {summary.InsertedCount}");
Console.WriteLine($"Updates: {summary.UpdatedCount}");
Console.WriteLine($"Deletes: {summary.DeletedCount}");

// Query history (audit trail of all Get() calls)
var history = uow.GetQueryHistory();
foreach (var entry in history)
    Console.WriteLine($"{entry.Timestamp:u} — {entry.RowCount} rows — filters: {entry.FilterSummary}");

// Clear history
uow.ClearQueryHistory();

Clone an item

// Shallow clone
var copy = (Customer)uow.CloneItem(customer, deep: false);
copy.Name = "Copy of " + customer.Name;
uow.Add(copy);

Soft delete workflow

uow.SoftDeleteFieldName = "IsDeleted";  // marks the record instead of removing it
uow.IncludeDeleted      = false;        // hide soft-deleted records in Get()

uow.Delete(customer);      // sets customer.IsDeleted = true, does NOT remove from collection

// To see all records including deleted:
uow.IncludeDeleted = true;
await uow.Get();

Optimistic concurrency

uow.ConcurrencyMode      = ConcurrencyMode.ThrowOnConflict;
uow.ConcurrencyFieldName = "RowVersion";

// If the server row has been modified since the item was loaded,
// Commit() will throw / return Errors.Failed with a concurrency violation message.

try
{
    var result = await uow.Commit();
    if (result.Flag != Errors.Ok)
        Console.WriteLine("Concurrency conflict: " + result.Message);
}
catch (Exception ex)
{
    Console.WriteLine("Concurrency: " + ex.Message);
    await uow.Rollback();
}

using var uow = new UnitofWork<Order>(dme, "OrdersDB", "Orders");

    {

};

Example 3 — Undo / Redo in a WinForms form

public class OrderForm : Form
{
    private UnitofWork<Order> _uow;

    public OrderForm(IDMEEditor dme)
    {
        _uow = new UnitofWork<Order>(dme, "OrdersDB", "Orders");
        _uow.IsUndoEnabled = true;
        _uow.MaxUndoDepth  = 30;
        
        _uow.PostEdit += (_, _) => UpdateUndoButtons();
    }

    private void UpdateUndoButtons()
    {
        btnUndo.Enabled = _uow.CanUndo;
        btnRedo.Enabled = _uow.CanRedo;
    }

    private void btnUndo_Click(object sender, EventArgs e)
    {
        if (_uow.Undo()) UpdateUndoButtons();
    }

    private void btnRedo_Click(object sender, EventArgs e)
    {
        if (_uow.Redo()) UpdateUndoButtons();
    }

    private async void btnSave_Click(object sender, EventArgs e)
    {
        var result = await _uow.Commit();
        if (result.Flag == Errors.Ok)
        {
            _uow.ClearUndoHistory();
            UpdateUndoButtons();
        }
    }
}

Example 6 — Validation + BlockCommit

using var uow = new UnitofWork<Invoice>(dme, "BillingDB", "Invoices");
uow.IsAutoValidateEnabled      = true;

await uow.Get();

uow.CurrentItem.Amount = -500; // violates [Range(0, double.MaxValue)]

var allResult = uow.ValidateAll();
if (!allResult.IsValid)
{
    Console.WriteLine("Errors before commit:");
    foreach (var err in allResult.Errors)
        Console.WriteLine($"  {err.FieldName}: {err.Message}");
}

// Commit will skip the invalid record but commit all valid ones
var result = await uow.Commit();
Console.WriteLine(result.Message); // e.g. "Committed 5 of 6 items (1 skipped — validation errors)"
