Beep Data Management Engine

Enterprise-grade data management solution for .NET 6, 7, 8, and 9 applications

Overview

The Beep Data Management Engine is a comprehensive, multi-target framework that provides powerful data access patterns, unit of work implementations, change tracking, and enterprise-level data management capabilities for modern .NET applications.

Built with extensibility in mind, the engine supports a wide variety of data sources from traditional relational databases to modern vector databases, file formats, and custom data providers through its plugin-based architecture.

Basic Usage

Here's a simple example to get you started:

Creating Your First UnitOfWork

using Autofac;
using TheTechIdea.Beep;
using TheTechIdea.Beep.ConfigUtil;
using TheTechIdea.Beep.Logger;
using TheTechIdea.Util;

// Initialize DME Editor using Autofac
var builder = new ContainerBuilder();
builder.RegisterType<DMEEditor>().As<IDMEEditor>().SingleInstance();
builder.RegisterType<ConfigEditor>().As<IConfigEditor>().SingleInstance();
builder.RegisterType<DMLogger>().As<IDMLogger>().SingleInstance();
builder.RegisterType<Util>().As<IUtil>().SingleInstance();
builder.RegisterType<ErrorsInfo>().As<IErrorsInfo>().SingleInstance();
builder.RegisterType<JsonLoader>().As<IJsonLoader>().SingleInstance();
builder.RegisterType<AssemblyHandler>().As<IAssemblyHandler>().SingleInstance();

var container = builder.Build();
var dmeEditor = container.Resolve<IDMEEditor>();

// Initialize required configurations for IDataSource support
InitializeBeepDefaults(dmeEditor);

// Create a data connection
ConnectionProperties connectionProps = new ConnectionProperties
{
    ConnectionName = "MyDatabase",
    DatabaseType = DataSourceType.SqlServer,
    ConnectionString = "Server=localhost;Database=MyDB;Integrated Security=true;"
};

// Add connection to configuration
dmeEditor.ConfigEditor.AddDataConnection(connectionProps);

// Get data source and open connection
IDataSource dataSource = dmeEditor.GetDataSource("MyDatabase");
dataSource.Openconnection();

// Create UnitOfWork for your entity
var unitOfWork = new UnitofWork<Customer>(
    dmeEditor, 
    "MyDatabase", 
    "Customers", 
    "CustomerId"
);

// Load data
await unitOfWork.Get();

// Work with your data
unitOfWork.New(); // Create new customer
var newCustomer = unitOfWork.CurrentItem;
newCustomer.Name = "John Doe";
newCustomer.Email = "john@example.com";

// Save changes
await unitOfWork.Commit();

Overview

The Beep Data Management Engine follows a modular, layered architecture designed for scalability, maintainability, and extensibility:

🎛️ Core Layer

DMEEditor, ConfigEditor, UnitOfWork Ecosystem
Central orchestration and configuration management

🔄 Processing Layer

ETL Engine, Workflow Engine, Sync Managers
Data transformation and business process automation

🗄️ Data Access Layer

Data Sources, Connection Management, Query Engine
Unified access to diverse data storage systems

🔧 Infrastructure Layer

Logging, Error Handling, Assembly Management
Cross-cutting concerns and system services

🏗️ Helper Layer

Utilities, Extensions, Type Mapping
Reusable components and development accelerators

📦 Container Layer

Dependency Injection, Service Management
Service composition and lifecycle management

🎯 Design Principles
  • Separation of Concerns: Each layer has distinct responsibilities
  • Dependency Injection: Loose coupling through container management
  • Extensibility: Plugin architecture for custom components
  • Configuration-Driven: Behavior controlled through configuration
  • Multi-Target Support: .NET 6, 7, 8, and 9 compatibility
  • Enterprise Ready: Scalable for production workloads
