Setup Framework

Comprehensive wizard-based system for automating application initialization: driver provisioning, connection configuration, database schema creation, and seed data injection across all platforms (WinForms, Blazor WASM, Blazor Server, MAUI, CLI, Web API).

Overview

The Setup Framework automates the end-to-end initialization of BeepDM applications. It:

  • Runs as a sequence of steps — each step handles one responsibility (driver load, connection, schema, seeding)
  • Is idempotent — steps detect when work is done and skip safely
  • Checkpoints state — supports resume after interruption or failure
  • Works cross-platform — dedicated adapters bridge to WinForms, Blazor, MAUI, CLI, Web API
  • Exposes progress — unified IProgress<PassedArgs> reporting per-step
  • Returns reports — SHA-256 checksummed outcome with per-step timings and messages

Architecture

Core Components

Component Responsibility Location

Step Implementations (Phase 2–5)

Step Purpose Dependencies
DriverProvisionStep Ensure connection driver is loaded (local or NuGet) None (first)
ConnectionConfigStep Build, validate, persist connection; optionally open DriverProvisionStep
SchemaSetupStep Plan, validate, and apply database schema via MigrationManager ConnectionConfigStep
SeedingStep Run all registered seeders in topological order SchemaSetupStep

8-Phase Implementation Plan

Phase 1: Contract & Wizard Architecture
Core interfaces (ISetupStep, ISetupWizard, ISetupWizardFactory, ISetupProgressReporter) and SetupWizard orchestrator with state sync.
Phase 2: Driver & Connection Steps
DriverProvisionStep (3-state loading) and ConnectionConfigStep (link, build, validate, persist, open).
Phase 3: Schema Setup & Dry-Run
SchemaSetupStep (9-step migration pipeline: plan → policy → dry-run → preflight → compensation → approve → execute) with DryRun mode support.
Phase 4: Seeding Subsystem
ISeeder/SeederBase, SeederRegistry (Kahn topological sort), ReferenceDataSeederBase, and SeedingStep with partial-resume support.
Phase 5: Platform Adapters
Desktop (WinForms), ConsoleSetupWizardAdapter, BlazorWasmSetupWizardAdapter, BlazorServerSetupWizardAdapter, WebApiSetupWizardAdapter, MauiSetupWizardAdapter.
Phase 6: Preflight Validation & Rollback
Pre-execution checks, compensation planning, rollback readiness assessment.
Phase 7: Observability & Reporting
Structured logging, audit trails, JSON export, tamper detection via SHA-256.
Phase 8: Advanced Features
Multi-tenant upgrades, CI/CD integration, orchestration with external systems.

Core API Contracts

ISetupStep

public interface ISetupStep
{
    string StepId { get; }
    string StepName { get; }
    string Description { get; }
    IReadOnlyList<string> DependsOn { get; }
    
    bool CanSkip(SetupContext context);
    IErrorsInfo Validate(SetupContext context);
    IErrorsInfo Execute(SetupContext context, IProgress<PassedArgs> progress = null);
}

public interface ISetupWizardAdapter

Usage Patterns

Desktop Application (WinForms)

var editor = new DMEEditor();
var connProps = new ConnectionProperties
{
    ConnectionName = "ProductDB",
    DatabaseType = DataSourceType.SqlLite,
    ConnectionString = "Data Source=./products.db"
};

var wizard = new SetupWizardBuilder()
    .WithId("product-app-setup")
    .WithEnvironment("Development")
    .AddStep(new DriverProvisionStep(new DriverProvisionStepOptions 
        { PackageName = "TheTechIdea.Beep.DataSources.SQLite" }))
    .AddStep(new ConnectionConfigStep(new ConnectionConfigStepOptions 
        { ConnectionProperties = connProps, OpenConnection = true }))
    .AddStep(new SchemaSetupStep(new SchemaSetupStepOptions 
        { EntityTypes = new[] { typeof(Product), typeof(Category) } }))
    .AddStep(new SeedingStep(new SeedingStepOptions { Registry = registry }))
    .Build();

var adapter = new DesktopSetupWizardAdapter(
    progressArgs => progressLabel.Text = progressArgs.Messege,
    report => MessageBox.Show(report.Succeeded ? "Setup OK" : "Setup failed"));

var context = new SetupContext { Editor = editor, Options = new SetupOptions() };
var report = await adapter.RunAsync(wizard, context);

Blazor Server with SignalR Progress

public class MyBlazorSetupAdapter : BlazorServerSetupWizardAdapter
{
    private readonly IHubContext<SetupHub> _hub;
    
    public MyBlazorSetupAdapter(IHubContext<SetupHub> hub) => _hub = hub;
    
    protected override void OnProgress(PassedArgs args)
        => _ = _hub.Clients.All.SendAsync("Progress", args.ParameterInt1, args.Messege);
    
    protected override void OnComplete(SetupReport report)
        => _ = _hub.Clients.All.SendAsync("Complete", report.Succeeded);
}

Resume from Checkpoint

var options = new SetupOptions { StateFilePath = "./setup-checkpoint.json" };
var wizard = builder.Build();

// First run
var report1 = await adapter.RunAsync(wizard, context);
// Context.State is persisted to StateFilePath after each step

// After interruption or restart, resume
var report2 = await wizard.Resume(context); // Loads checkpoint, skips completed steps

State Management

SetupState

public class SetupState
{
    public HashSet<string> CompletedStepIds { get; set; }
    public HashSet<string> SkippedStepIds { get; set; }
    public string FailedStepId { get; set; }
    public string SchemaHash { get; set; }
    public HashSet<string> CompletedSeederIds { get; set; }
    public DateTimeOffset? StartedAt { get; set; }
    public DateTimeOffset? LastUpdatedAt { get; set; }
    public Dictionary<string, string> Metadata { get; set; }
    
    public bool IsStepCompleted(string stepId) 
        => CompletedStepIds.Contains(stepId) || SkippedStepIds.Contains(stepId);
}

Sync Strategy

  • SyncFromContext: Merges fresh writes from a step (via context.State) back into the wizard's State (post-Execute on success path)
  • SyncToContext: Pushes wizard's State into context before Execute, so steps read current bookkeeping
  • PersistState: Atomic temp-file + rename to checkpoint file after each step
  • LoadPersistedState: Restores checkpoint on wizard init or Resume

Platform Adapters

DesktopSetupWizardAdapter

For WinForms/WPF. Runs wizard on thread-pool; callbacks marshal to UI thread.

ConsoleSetupWizardAdapter

For CLI/BeepShell. Writes formatted text progress; can be subclassed for Spectre.Console.

BlazorWasmSetupWizardAdapter

For Blazor WASM. Loads/saves state via platform store (localStorage); supports cancellation.

BlazorServerSetupWizardAdapter

For Blazor Server. Extensible for SignalR real-time progress push to connected clients.

WebApiSetupWizardAdapter

For ASP.NET Core. Exposes Status polling object; ideal for UI polling endpoints.

MauiSetupWizardAdapter

For .NET MAUI. Thread-safe main-thread dispatch; integrates with MAUI threading model.

Key Design Principles

  • Idempotency: Every step detects if its work is complete (via CanSkip) and safely skips
  • Resilience: Failed steps are recorded; Resume picks up from the next step
  • Progress Transparency: Unified IProgress<PassedArgs> per-step with percentage + message
  • State Isolation: Each SetupContext owns its state; wizard merges selectively
  • Extensibility: Builders, adapters, and custom step implementations supported
  • Error Reporting: Return IErrorsInfo instead of throwing; exceptions caught and wrapped
