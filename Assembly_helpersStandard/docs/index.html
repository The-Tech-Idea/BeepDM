Assembly Helpers Documentation

Dynamic assembly loading, scanning, and management for .NET 8 and 9 applications

Overview

TheTechIdea.Beep.Tools.AssemblyHandler is a comprehensive library for dynamic assembly loading, extension scanning, and driver management in .NET applications. Built for .NET 8 and 9, it provides sophisticated capabilities for plugin architectures, extensible applications, and modular system designs.

This library handles complex scenarios including assembly resolution, type caching, parallel processing for large assemblies, and sophisticated driver discovery mechanisms.

Key Features

Dynamic Assembly Loading

Load assemblies from file paths, scan directories, and handle assembly resolution with sophisticated caching mechanisms.

Extension Scanning

Automatically discover and instantiate extensions, addins, and plugins with parallel processing support.

Driver Management

Comprehensive ADO.NET and custom driver discovery, configuration, and management capabilities.

Performance Optimized

Type caching, assembly caching, and parallel processing for optimal performance in large applications.

Error Handling

Robust error handling with detailed logging and graceful degradation for problematic assemblies.

Hierarchical Organization

Organize loaded types and addins into hierarchical structures for easy navigation and management.

Quick Start

C# 
// Initialize AssemblyHandler
var configEditor = new ConfigEditor();
var errorObject = new ErrorsInfo();
var logger = new DMLogger();
var utilFunction = new Util();

var assemblyHandler = new AssemblyHandler(
    configEditor, 
    errorObject, 
    logger, 
    utilFunction
);

// Load all assemblies with progress reporting
var progress = new Progress<PassedArgs>(args => {
    Console.WriteLine($"Progress: {args.Messege}");
});

var result = assemblyHandler.LoadAllAssembly(progress, CancellationToken.None);

// Create instance from type name
var instance = assemblyHandler.CreateInstanceFromString(
    "MyNamespace.MyClass", 
    constructorArg1, 
    constructorArg2
);

// Get loaded data source classes
var dataSources = assemblyHandler.DataSourcesClasses;
foreach (var ds in dataSources)
{
    Console.WriteLine($"Found data source: {ds.className}");
}

Architecture

The Assembly Helpers library is built around several core components:

Core Components

  • AssemblyHandler - Main orchestrator for assembly operations
  • Type Cache - High-performance type resolution and caching
  • Assembly Cache - Thread-safe assembly loading and caching
  • Extension Scanners - Pluggable scanning mechanisms
  • Driver Discovery - ADO.NET and custom driver identification
  • Hierarchical Organization - Tree-based addin and extension organization

Common Usage Patterns

Plugin Architecture

Use AssemblyHandler to create extensible applications with plugin support:

// Scan for plugins implementing IPlugin interface
assemblyHandler.GetBuiltinClasses();
var plugins = assemblyHandler.ConfigEditor.Addins
    .Where(a => a.componentType == "IPlugin");

Data Source Discovery

Automatically discover and configure data source drivers:

// Load and scan data source assemblies
assemblyHandler.LoadAssembly(@"C:\DataSources", FolderFileTypes.DataSources);

// Get available data sources
var availableDataSources = assemblyHandler.DataSourcesClasses
    .Where(ds => ds.IsDataSource);

Extension Management

Manage loader extensions and scanning mechanisms:

// Load extension scanners
assemblyHandler.GetExtensionScanners(progress, cancellationToken);

// Get available loader extensions
var extensions = assemblyHandler.LoaderExtensionClasses;

Performance Considerations

Type Caching

The library automatically caches resolved types to avoid repeated reflection operations. Cache is case-insensitive for improved lookup performance.

Parallel Processing

Large assemblies (>100 types) are processed in parallel to improve scanning performance. Configure thresholds based on your application needs.

Assembly Caching

Loaded assemblies are cached using thread-safe collections to prevent duplicate loading and improve performance.
