AssemblyHandler

Core class for dynamic assembly loading, scanning, and management

Namespace: TheTechIdea.Beep.Tools
Assembly: Assembly_helpers.dll
Implements: IAssemblyHandler, IDisposable
Target Framework: .NET 8, .NET 9

Overview

The AssemblyHandler class is the core component for managing assembly operations in the Beep framework. It provides comprehensive functionality for loading assemblies, scanning for extensions, discovering data sources and drivers, and organizing components into hierarchical structures.

This class handles complex scenarios including assembly resolution, type caching, parallel processing for performance optimization, and sophisticated error handling for robust plugin architectures.

Core Methods

LoadAllAssembly

IErrorsInfo LoadAllAssembly(IProgress<PassedArgs> progress, CancellationToken token)

Comprehensive method that loads all assemblies from configured folders and scans them for various components.

Parameters

Name Type Description
progress IProgress<PassedArgs> Progress reporting mechanism for operation updates
token CancellationToken Token for canceling the operation

Returns

IErrorsInfo - Information about the operation success or failure

Example

var progress = new Progress<PassedArgs>(args => {
    Console.WriteLine($"Progress: {args.Messege}");

    progress, 
    cancellationToken.Token

    Console.WriteLine($"Error: {result.Message}");

Performance Features

Type Caching

The AssemblyHandler uses sophisticated caching mechanisms to improve performance:

  • Type Cache: Case-insensitive dictionary for fast type resolution
  • Assembly Cache: Thread-safe concurrent dictionary for loaded assemblies
  • Namespace Filtering: Configurable namespace exclusion for focused scanning

Parallel Processing

Large assemblies are processed in parallel for optimal performance:

// Automatic parallel processing for assemblies with >100 types
if (types.Length > 100)
{
    var typeInfoCollection = new ConcurrentBag<TypeInfo>();
    Parallel.ForEach(types, type => {
        // Process types in parallel
    });
}

Error Handling

The AssemblyHandler provides comprehensive error handling:

Assembly Loading Errors

try
{
    var result = assemblyHandler.LoadAssembly(path, fileTypes);
    if (assemblyHandler.ErrorObject.Flag == Errors.Failed)
    {
        Console.WriteLine($"Error: {assemblyHandler.ErrorObject.Message}");
    }
}
catch (FileLoadException ex)
{
    // Handle assembly already loaded
}
catch (BadImageFormatException ex)
{
    // Handle invalid assembly format
}

Type Resolution Errors

var type = assemblyHandler.GetType("MyNamespace.MyClass");
if (type == null)
{
    Console.WriteLine("Type not found in loaded assemblies");
}
else
{
    // Safe to use type
}

Best Practices

Use Progress Reporting

Always provide progress reporting for long-running operations to improve user experience.

Configure Namespace Filtering

Set up NamespacestoIgnore to exclude system assemblies and improve scanning performance.

Handle Cancellation

Use CancellationToken for operations that might need to be cancelled by the user.

Check Error States

Always check ErrorObject.Flag after operations to handle failures gracefully.
