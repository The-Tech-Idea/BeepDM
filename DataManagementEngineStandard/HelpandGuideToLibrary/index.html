Extending with New Data Sources

The Beep Data Management Engine is designed for extensibility. Adding support for new databases or file types is straightforward by implementing the IDataSource interface.

Why It's Easy to Extend

The engine uses a plugin-based architecture where each data source is a separate implementation of IDataSource. The framework handles driver loading, connection management, and type mapping automatically.

How to Add a New Data Source

Follow these steps to add support for a new database or file type:

1

Implement IDataSource Interface

Create a new class that implements the IDataSource interface. This interface defines all the methods needed for data operations.

public class MyCustomDataSource : IDataSource
{
    public string GuidID { get; set; } = Guid.NewGuid().ToString();
    public string DatasourceName { get; set; }
    public DataSourceType DatasourceType { get; set; }
    public DatasourceCategory Category { get; set; }
    public ConnectionState ConnectionStatus { get; set; }
    public IDataConnection Dataconnection { get; set; }
    public IErrorsInfo ErrorObject { get; set; }
    public IDMLogger Logger { get; set; }
    public IDMEEditor DMEEditor { get; set; }
    public List<EntityStructure> Entities { get; set; } = new List<EntityStructure>();
    public List<string> EntitiesNames { get; set; } = new List<string>();

    // Constructor
    public MyCustomDataSource(string datasourcename, IDMLogger logger, 
                             IDMEEditor dmeEditor, DataSourceType databasetype, 
                             IErrorsInfo errorObject)
    {
        DatasourceName = datasourcename;
        Logger = logger;
        DMEEditor = dmeEditor;
        DatasourceType = databasetype;
        ErrorObject = errorObject;
        
        // Initialize your custom connection
        Dataconnection = new MyCustomConnection(DMEEditor)
        {
            Logger = logger,
            ErrorObject = errorObject
        };
    }

    // Implement required methods...
    public virtual ConnectionState Openconnection() { /* Implementation */ }
    public virtual ConnectionState Closeconnection() { /* Implementation */ }
    public virtual object GetEntity(string EntityName, List<AppFilter> Filter) { /* Implementation */ }
    // ... other required methods
}
2

Create Custom Connection Class

Implement IDataConnection to handle the specific connection logic for your data source.

public class MyCustomConnection : IDataConnection
{
    public ConnectionState ConnectionStatus { get; set; }
    public ConnectionProperties ConnectionProp { get; set; }
    public ConnectionDriversConfig DataSourceDriver { get; set; }
    public IDMEEditor DMEEditor { get; set; }
    public IErrorsInfo ErrorObject { get; set; }
    public IDMLogger Logger { get; set; }

    public MyCustomConnection(IDMEEditor dmeEditor)
    {
        DMEEditor = dmeEditor;
    }

    public ConnectionState OpenConnection()
    {
        try
        {
            // Your custom connection logic here
            // For example, connecting to a proprietary database or API
            
            ConnectionStatus = ConnectionState.Open;
            Logger?.WriteLog("Successfully connected to custom data source");
            return ConnectionStatus;
        }
        catch (Exception ex)
        {
            ConnectionStatus = ConnectionState.Broken;
            ErrorObject.Flag = Errors.Failed;
            ErrorObject.Message = ex.Message;
            Logger?.WriteLog($"Failed to connect: {ex.Message}");
            return ConnectionStatus;
        }
    }

    public ConnectionState CloseConn()
    {
        // Clean up resources
        ConnectionStatus = ConnectionState.Closed;
        return ConnectionStatus;
    }
}
3

Register Your Data Source

Add your data source to the engine's configuration so it can be discovered and used.

// In your application startup or configuration
public void RegisterCustomDataSource()
{
    // Add to assemblies configuration
    AssemblyClassDefinition customDataSourceDef = new AssemblyClassDefinition
    {
        className = "MyCustomDataSource",
        PackageName = "MyCustom.DataSources",
        type = typeof(MyCustomDataSource)
    };

    DMEEditor.ConfigEditor.DataSourcesClasses.Add(customDataSourceDef);

    // Add driver configuration
    ConnectionDriversConfig driverConfig = new ConnectionDriversConfig
    {
        PackageName = "MyCustom.DataSources",
        classHandler = "MyCustomDataSource",
        ConnectionType = "MyCustomConnection",
        DatasourceCategory = DatasourceCategory.RDBMS, // or appropriate category
        DatasourceType = DataSourceType.SqlLite // Define your custom type
    };

    DMEEditor.ConfigEditor.DataDriversClasses.Add(driverConfig);
}
4

Use Your New Data Source

Once registered, your data source can be used like any other data source in the engine.

public class MyCustomConnection : IDataConnection { public ConnectionState ConnectionStatus { get; set; } public ConnectionProperties ConnectionProp { get; set; } public ConnectionDriversConfig DataSourceDriver { get; set; } public IDMEEditor DMEEditor { get; set; } public IErrorsInfo ErrorObject { get; set; } public IDMLogger Logger { get; set; } public MyCustomConnection(IDMEEditor dmeEditor) { DMEEditor = dmeEditor; } public ConnectionState OpenConnection() { try { // Your custom connection logic here // For example, connecting to a proprietary database or API ConnectionStatus = ConnectionState.Open; Logger?.WriteLog("Successfully connected to custom data source"); return ConnectionStatus; } catch (Exception ex) { ConnectionStatus = ConnectionState.Broken; ErrorObject.Flag = Errors.Failed; ErrorObject.Message = ex.Message; Logger?.WriteLog($"Failed to connect: {ex.Message}"); return ConnectionStatus; } } public ConnectionState CloseConn() { // Clean up resources ConnectionStatus = ConnectionState.Closed; return ConnectionStatus; } }
4

Use Your New Data Source

Once registered, your data source can be used like any other data source in the engine.

// Create connection properties
ConnectionProperties connectionProps = new ConnectionProperties
{
    ConnectionName = "MyCustomDB",
    DatabaseType = DataSourceType.SqlLite, // Your custom type
    ConnectionString = "Server=mycustomserver;Database=mydb;",
    Category = DatasourceCategory.RDBMS
};

// Add to configuration
DMEEditor.ConfigEditor.AddDataConnection(connectionProps);

// Use it like any other data source
IDataSource customDS = DMEEditor.GetDataSource("MyCustomDB");
customDS.Openconnection();

var entities = customDS.GetEntitesList();
var data = customDS.GetEntity("MyTable", null);

RDBDataSource Example

The RDBDataSource class is an excellent reference implementation that supports most RDBMS systems (SQL Server, Oracle, MySQL, PostgreSQL, etc.). It demonstrates how to:

  • Handle different SQL dialects and syntax variations
  • Implement connection management and pooling
  • Support transactions and bulk operations
  • Handle schema introspection and metadata
  • Implement CRUD operations with proper error handling

You can find the full implementation in the RDBSource.cs file in the RDBMSDataSource project.

Best Practices

  • Always implement proper connection lifecycle management
  • Use the provided logging infrastructure for debugging
  • Follow the existing naming conventions and patterns
  • Implement async methods when possible for better performance
  • Handle edge cases and provide meaningful error messages

Advanced Features

For more advanced data sources, you can also implement:

  • Custom Filtering: Override query building for specific filter syntax
  • Bulk Operations: Implement efficient bulk insert/update operations
  • Schema Creation: Support for creating new tables/entities
  • Relationship Mapping: Handle foreign keys and relationships
  • Custom Data Types: Map specialized data types to .NET types
  • Streaming Support: Handle large datasets efficiently

By following this pattern, you can quickly extend the Beep Data Management Engine to support any data source, from traditional databases to modern APIs, file formats, or even proprietary systems.
