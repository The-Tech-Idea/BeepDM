# Phase 11.3 — Session Affinity + Drain + In-Flight Tracking

**Priority:** P2  
**Depends on:** Phase 11.1, Phase 11.2  
**Goals satisfied:** G6 (session affinity), G7 (zero-downtime rolling restart)  
**Estimated effort:** ~0.5 day

---

## Objective

1. Implement sticky session affinity so repeated calls from the same "session key"
   (e.g., `UserId`, `TransactionId`, `CorrelationId`) always land on the same node
   as long as that node remains alive and the TTL hasn't expired.
2. Harden `DrainNode()` with proper async support and an in-flight wait loop.
3. Expose `InFlightCount` per-node so callers and load-balancing strategies have
   accurate concurrency data.

## 1. Affinity Map Design

The `_affinityMap` stores:

```
sessionKey (string) → (NodeId: string, LastUsed: DateTime)
```

**Session key selection order** (evaluated from the `ProxyExecutionContext`):

1. `ctx.SessionKey` (explicit override)
2. `ctx.UserId` (if set)
3. `ctx.CorrelationId` (fallback)

If none are set, affinity is skipped and the normal router is used.

```csharp
// In ProxyCluster.cs
private readonly ConcurrentDictionary<string, (string NodeId, DateTime LastUsed)>
    _affinityMap = new();
```

## 2. `SelectNode` with Affinity Branch

```csharp
private ProxyNode SelectNode(string operationName, string entityHint, ProxyExecutionContext ctx)
{
    var live = _nodes.Values
        .Where(n => n.IsAlive && !n.IsDraining)
        .ToList();

    if (live.Count == 0)
        throw new InvalidOperationException("ProxyCluster: no live nodes available.");

    // ── Affinity lookup ────────────────────────────────────────────────
    if (_clusterPolicy.EnableNodeAffinity)
    {
        string sessionKey = ResolveSessionKey(ctx);
        if (sessionKey is not null)
        {
            if (_affinityMap.TryGetValue(sessionKey, out var entry))
            {
                var affinityNode = live.FirstOrDefault(n => n.NodeId == entry.NodeId);
                if (affinityNode is not null)
                {
                    // Refresh TTL
                    _affinityMap[sessionKey] = (entry.NodeId, DateTime.UtcNow);
                    ApplySlowStartWeights(live);
                    return affinityNode;
                }
                // Affinity target is dead / draining — fall through to normal selection
                // and record new affinity below
            }

            // No existing affinity — select normally then pin
            ApplySlowStartWeights(live);
            var selected = _nodeRouter.SelectBest(live, ctx);
            _affinityMap[sessionKey] = (selected.NodeId, DateTime.UtcNow);
            return selected;
        }
    }

    // ── Normal routing path ────────────────────────────────────────────
    ApplySlowStartWeights(live);
    return _nodeRouter.SelectBest(live, ctx);
}

private static string ResolveSessionKey(ProxyExecutionContext ctx)
{
    if (ctx is null) return null;
    if (!string.IsNullOrEmpty(ctx.SessionKey))    return ctx.SessionKey;
    if (!string.IsNullOrEmpty(ctx.UserId))        return ctx.UserId;
    if (!string.IsNullOrEmpty(ctx.CorrelationId)) return ctx.CorrelationId;
    return null;
}
```

## 3. Affinity TTL Sweep

```csharp
private Timer _affinityTtlTimer;

private void StartAffinityTtlSweep()
{
    int ttlMs = (_clusterPolicy.NodeAffinityTtlSeconds > 0
        ? _clusterPolicy.NodeAffinityTtlSeconds
        : 300) * 1000;

    // Run the sweep at half-TTL intervals so no key lives more than 1.5× TTL
    _affinityTtlTimer = new Timer(
        state => SweepExpiredAffinity(),
        null,
        dueTime:  ttlMs / 2,
        period:   ttlMs / 2);
}

private void SweepExpiredAffinity()
{
    var ttl = TimeSpan.FromSeconds(
        _clusterPolicy.NodeAffinityTtlSeconds > 0
        ? _clusterPolicy.NodeAffinityTtlSeconds : 300);

    foreach (var key in _affinityMap.Keys)
    {
        if (_affinityMap.TryGetValue(key, out var entry)
            && DateTime.UtcNow - entry.LastUsed > ttl)
        {
            _affinityMap.TryRemove(key, out _);
        }
    }
}
```

## 4. Evict Affinity on Node Down

When `ProxyCluster.NodeProbing` marks a node dead it must evict all affinity entries
pointing to that node so subsequent calls can reroute transparently.

```csharp
// Inside ProxyCluster.NodeProbing.cs — in the method that marks a node as dead:
private void MarkNodeDead(ProxyNode node, string reason)
{
    if (!node.IsAlive) return;     // already marked

    node.IsAlive = false;
    node.ConsecutiveFailures = 0;
    node.ConsecutiveSuccesses = 0;

    // Evict all affinity entries pointing at this node
    foreach (var key in _affinityMap.Keys)
        if (_affinityMap.TryGetValue(key, out var e) && e.NodeId == node.NodeId)
            _affinityMap.TryRemove(key, out _);

    // Emit cluster event
    OnNodeDown?.Invoke(this, new NodeStatusEventArgs(node.NodeId, false, reason));

    _auditSink.Write(new ProxyAuditEntry
    {
        EntryKind     = AuditEntryKind.NodeStatus,
        NodeId        = node.NodeId,
        Success       = false,
        Message       = $"Node marked dead: {reason}",
        OccurredAtUtc = DateTime.UtcNow
    });
}
```

## 5. `DrainNode` (sync + async)

```csharp
// Sync — blocks calling thread
public void DrainNode(string nodeId, int timeoutMs = 30_000)
{
    if (!_nodes.TryGetValue(nodeId, out var node)) return;
    node.IsDraining = true;

    var deadline   = DateTime.UtcNow.AddMilliseconds(timeoutMs);
    var spinWait   = 50;   // ms

    while (node.InFlightCount > 0 && DateTime.UtcNow < deadline)
        Thread.Sleep(spinWait);

    // Even if InFlightCount > 0 at deadline, we allow forceful removal.
    _auditSink.Write(new ProxyAuditEntry
    {
        EntryKind     = AuditEntryKind.Drain,
        NodeId        = nodeId,
        Success       = node.InFlightCount == 0,
        Message       = node.InFlightCount == 0
            ? "Drain completed cleanly"
            : $"Drain timed out ({node.InFlightCount} in-flight operations remaining)",
        OccurredAtUtc = DateTime.UtcNow
    });
}

// Async — preferred for UI/async callers
public async Task DrainNodeAsync(string nodeId,
    int timeoutMs = 30_000,
    CancellationToken ct = default)
{
    if (!_nodes.TryGetValue(nodeId, out var node)) return;
    node.IsDraining = true;

    var cts      = CancellationTokenSource.CreateLinkedTokenSource(ct);
    cts.CancelAfter(timeoutMs);

    while (node.InFlightCount > 0 && !cts.IsCancellationRequested)
        await Task.Delay(50, cts.Token).ConfigureAwait(false);

    _auditSink.Write(new ProxyAuditEntry
    {
        EntryKind     = AuditEntryKind.Drain,
        NodeId        = nodeId,
        Success       = node.InFlightCount == 0,
        Message       = node.InFlightCount == 0
            ? "Drain completed cleanly"
            : $"Drain timed out ({node.InFlightCount} in-flight operations remaining)",
        OccurredAtUtc = DateTime.UtcNow
    });
}
```

## 6. Zero-Downtime Rolling Restart Protocol

Using `DrainNode` the operator can achieve a zero-downtime rolling restart:

```
For each node N in cluster:
  1. cluster.DrainNode(N.NodeId, timeoutMs: 30_000)
  2. cluster.RemoveNode(N.NodeId)         // disposes the proxy
  3. new_node = CreateFreshNode(N.Config)
  4. cluster.AddNode(new_node)            // new node enters slow-start if SlowStartDurationMs > 0
  5. Wait until new_node.InFlightCount > 0 (traffic flowing)
     OR wait SlowStartDurationMs
```

This protocol means at most one node is removed/restarting at any moment, so the
cluster always has N-1 healthy nodes serving traffic.

## 7. Sequence Diagram — Sticky Affinity

```
Request (userId="alice")
  │
  ▼
ProxyCluster.SelectNode(ctx={UserId="alice"})
  │
  ├─ EnableNodeAffinity = true → sessionKey = "alice"
  │
  ├─ _affinityMap["alice"] exists? ──YES──▶ node alive? ──YES──▶ refresh TTL, return node-2
  │                                                      NO ──▶ fall through
  │
  └─ No affinity (or dead target)
       │
       ▼
     _nodeRouter.SelectBest(liveNodes, ctx) → node-3
       │
       ▼
     _affinityMap["alice"] = (node-3, now)   ← new sticky assignment
       │
       ▼
     return node-3

------
TTL sweep (runs every NodeAffinityTtlSeconds/2)
  │
  ▼
  foreach key in _affinityMap
    if now - LastUsed > TTL → TryRemove(key)
```

## 8. Sequence Diagram — Node Down → Affinity Eviction

```
NodeProbing timer fires
  │
  ▼
Probe node-2 → FAIL (consecutive failures >= NodeUnhealthyThreshold)
  │
  ▼
MarkNodeDead(node-2)
  │── node.IsAlive = false
  │── foreach key in _affinityMap where entry.NodeId == "node-2": TryRemove(key)
  │── OnNodeDown.Invoke(...)
  └── _auditSink.Write(NodeStatus / dead)

Next request for session "alice" (was pinned to node-2)
  │
  ▼
SelectNode → _affinityMap["alice"] missing (evicted)
           → normal route → node-1 (only surviving node)
           → _affinityMap["alice"] = (node-1, now)
```

## Acceptance Criteria

- [ ] When `EnableNodeAffinity = true`, repeated calls with the same `UserId`/`SessionKey` route to the same node
- [ ] When the affinity-pinned node is marked dead, subsequent calls route to a different live node and re-pin there
- [ ] Affinity entries expire after `NodeAffinityTtlSeconds` of idle time
- [ ] `DrainNode(id)` returns only after `InFlightCount` reaches 0 or timeout elapses
- [ ] `DrainNodeAsync(id, ct)` respects `CancellationToken`
- [ ] `InFlightCount` is accurate under concurrent requests (no races)
- [ ] `IProxyCluster` interface exposes `DrainNodeAsync`
