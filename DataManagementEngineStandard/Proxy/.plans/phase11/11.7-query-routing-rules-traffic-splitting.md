# Phase 11.7 — Query Routing Rules + Traffic Splitting

**Priority:** P2  
**Depends on:** Phase 11.1  
**Goals satisfied:** G10 (query routing rules), G16 (traffic splitting / canary)  
**Inspiration:** ProxySQL `query_rules`, MaxScale routing hints; Istio `VirtualService` weights  
**Estimated effort:** ~0.5 day

---

## Objective

1. **Query routing rules** — let operators write declarative rules (match on operation
   name and/or entity name using regex) that override the default router for specific
   traffic patterns.  
   Example: route all SELECT-equivalent reads to replica nodes, all mutating ops to
   the primary.

2. **Traffic splitting** — distribute a percentage of traffic to a canary node
   independently of weight, so a deployment can be validated with 5 % of live load
   before full cut-over.

## 1. `ProxyRoutingRule`

```csharp
// In ProxyotherClasses.cs

/// <summary>
/// Declarative routing rule — evaluated before the normal router strategy.
/// Rules are sorted by descending <see cref="Priority"/> (higher = evaluated first).
/// </summary>
public sealed class ProxyRoutingRule
{
    /// <summary>
    /// Regex applied to the operation name (e.g. "GetEntity", "ExecuteQuery", "InsertDataRow").
    /// Null = any operation.
    /// </summary>
    public string? OperationPattern { get; init; }

    /// <summary>
    /// Regex applied to the entity/table name hint passed to route selection.
    /// Null = any entity.
    /// </summary>
    public string? EntityPattern { get; init; }

    /// <summary>Target node ID. Null = use the override strategy without a fixed target.</summary>
    public string? TargetNodeId { get; init; }

    /// <summary>
    /// When <see cref="TargetNodeId"/> is null or the target is unavailable,
    /// apply this strategy to select from the candidate pool.
    /// </summary>
    public ProxyRoutingOverrideStrategy OverrideStrategy { get; init; }
        = ProxyRoutingOverrideStrategy.RouteToReplica;

    /// <summary>Higher values are evaluated first.</summary>
    public int Priority { get; init; } = 0;
}

/// <summary>Strategy applied by a routing rule when no explicit TargetNodeId is given.</summary>
public enum ProxyRoutingOverrideStrategy
{
    /// <summary>Route to any replica node (role = Replica). Falls back to primary if none.</summary>
    RouteToReplica,
    /// <summary>Route to the primary node (role = Primary).</summary>
    RouteToPrimary,
    /// <summary>Route to any live node using the cluster's default router.</summary>
    RouteToAny
}
```

## 2. `TrafficSplitRule`

```csharp
/// <summary>
/// Splits a percentage of traffic to a specific node for canary testing.
/// Multiple split rules may be active simultaneously; their WeightPercent values
/// are consumed in priority order and the remainder goes to the default router.
/// </summary>
public sealed class TrafficSplitRule
{
    /// <summary>Node that receives the canary slice.</summary>
    public string TargetNodeId { get; init; } = default!;

    /// <summary>0–100. 5 = 5% of all matching requests go to TargetNodeId.</summary>
    public int WeightPercent { get; init; } = 5;

    /// <summary>Scope of operations this split applies to.</summary>
    public ProxySplitScope OperationScope { get; init; } = ProxySplitScope.All;
}

public enum ProxySplitScope
{
    All,
    ReadsOnly,
    WritesOnly
}
```

## 4. Rule Evaluation in `SelectNode`

**File:** `ProxyCluster.cs`

The rule evaluation happens **after** entity-affinity and **before** session affinity
so that per-entity ownership always wins:

```csharp
// ── Traffic split check (canary) ──────────────────────────────────────────
private static readonly ThreadLocal<Random> _rng
    = new(() => new Random(Guid.NewGuid().GetHashCode()));

private ProxyNode? TrySelectSplitNode(
    IReadOnlyList<ProxyNode> live,
    string operationName)
{
    foreach (var rule in _clusterPolicy.TrafficSplits)
    {
        if (rule.OperationScope == ProxySplitScope.ReadsOnly
            && !IsReadOperation(operationName)) continue;
        if (rule.OperationScope == ProxySplitScope.WritesOnly
            && IsReadOperation(operationName)) continue;

        if (_rng.Value!.Next(100) < rule.WeightPercent)
        {
            // This request is in the canary slice
            var canary = live.FirstOrDefault(n => n.NodeId == rule.TargetNodeId);
            if (canary is not null) return canary;
        }
    }
    return null;
}

// ── Routing rule evaluation  ──────────────────────────────────────────────
private ProxyNode? TrySelectFromRules(
    IReadOnlyList<ProxyNode> live,
    string operationName,
    string? entityHint)
{
    // Rules should already be sorted by descending Priority in ApplyClusterPolicy
    foreach (var rule in _clusterPolicy.RoutingRules)
    {
        bool opMatch = rule.OperationPattern is null
            || Regex.IsMatch(operationName, rule.OperationPattern, RegexOptions.IgnoreCase);

        bool entityMatch = rule.EntityPattern is null
            || (entityHint is not null
                && Regex.IsMatch(entityHint, rule.EntityPattern, RegexOptions.IgnoreCase));

        if (!opMatch || !entityMatch) continue;

        // Rule matches — apply
        if (rule.TargetNodeId is not null)
        {
            var target = live.FirstOrDefault(n => n.NodeId == rule.TargetNodeId);
            if (target is not null) return target;
            // Target dead — fall through to strategy
        }

        return rule.OverrideStrategy switch
        {
            ProxyRoutingOverrideStrategy.RouteToReplica
                => live.FirstOrDefault(n => n.NodeRole == ProxyDataSourceRole.Replica)
                   ?? live.FirstOrDefault(),   // fallback to any if no replicas

            ProxyRoutingOverrideStrategy.RouteToPrimary
                => live.FirstOrDefault(n => n.NodeRole == ProxyDataSourceRole.Primary)
                   ?? live.First(),

            _ => null   // RouteToAny: fall through to default router
        };
    }
    return null;
}

private static bool IsReadOperation(string operationName)
    => operationName.StartsWith("Get",    StringComparison.OrdinalIgnoreCase)
    || operationName.StartsWith("Select", StringComparison.OrdinalIgnoreCase)
    || operationName.StartsWith("Read",   StringComparison.OrdinalIgnoreCase)
    || operationName.StartsWith("Find",   StringComparison.OrdinalIgnoreCase)
    || operationName.StartsWith("Fetch",  StringComparison.OrdinalIgnoreCase)
    || operationName.StartsWith("List",   StringComparison.OrdinalIgnoreCase)
    || operationName.StartsWith("Search", StringComparison.OrdinalIgnoreCase);
```

## 5. Policy Sorting of Rules at Apply Time

```csharp
public void ApplyClusterPolicy(ProxyPolicy policy)
{
    // Sort rules by descending Priority so TrySelectFromRules can short-circuit
    var sortedRules = policy.RoutingRules
        .OrderByDescending(r => r.Priority)
        .ToList()
        .AsReadOnly();

    _clusterPolicy = policy with { RoutingRules = sortedRules };

    // ... rest of ApplyClusterPolicy
}
```

## 6. Configuration Examples

### Read/write split

```csharp
var policy = new ProxyPolicy
{
    RoutingRules = new[]
    {
        // Writes (Insert/Update/Delete) → primary
        new ProxyRoutingRule
        {
            OperationPattern  = "Insert|Update|Delete|ExecuteNonQuery",
            OverrideStrategy  = ProxyRoutingOverrideStrategy.RouteToPrimary,
            Priority          = 10
        },
        // Reads → any replica
        new ProxyRoutingRule
        {
            OperationPattern  = "Get|Select|Read|Find",
            OverrideStrategy  = ProxyRoutingOverrideStrategy.RouteToReplica,
            Priority          = 5
        }
    }
};
```

### Canary deployment (5 % to new node)

```csharp
var policy = new ProxyPolicy
{
    TrafficSplits = new[]
    {
        new TrafficSplitRule
        {
            TargetNodeId   = "node-canary",
            WeightPercent  = 5,
            OperationScope = ProxySplitScope.ReadsOnly
        }
    }
};
```

### Combined

```csharp
var policy = new ProxyPolicy
{
    NodeRoutingStrategy = ProxyNodeRoutingStrategy.LeastConnections,
    RoutingRules   = new[] { writeRule, readRule },
    TrafficSplits  = new[] { canaryRule }
};
```

## 7. Sequence Diagram — Rule Evaluation

```
SelectNode("GetEntity", "Orders", ctx)
  │
  ├─ TrySelectSplitNode(live, "GetEntity")
  │     ├─ TrafficSplits[0]: scope=Reads, weight=5%
  │     └─ rng.Next(100) = 3 < 5 → return "node-canary"
  │          (5% of reads go here)
  │
  │  (no split this time — rng = 72)
  │
  ├─ TrySelectFromRules(live, "GetEntity", "Orders")
  │     ├─ Rule priority=10: pattern="Insert|Update|Delete" → does NOT match "GetEntity"
  │     ├─ Rule priority=5:  pattern="Get|Select|Read"     → MATCHES
  │     │     OverrideStrategy = RouteToReplica
  │     └─ live replica node-2 → return node-2
  │
  │  (if routing rules return null)
  │
  ├─ EntityAffinity.Resolve("Orders") → "node-1" → return node-1
  │
  └─ _nodeRouter.SelectBest(live, ctx)
```

## Acceptance Criteria

- [ ] Rules are evaluated in descending `Priority` order
- [ ] `OperationPattern` and `EntityPattern` are optional; null matches everything
- [ ] `TargetNodeId` specified + node alive → that node is returned
- [ ] `TargetNodeId` specified + node dead → fall through to `OverrideStrategy`
- [ ] `RouteToReplica` picks a replica node; if none exist it picks any live node
- [ ] `RouteToPrimary` picks the primary; if none exist it picks `First()` live
- [ ] `TrafficSplitRule` with `WeightPercent = 5` routes approximately 5 % of calls to canary
- [ ] Split is statistically correct over 10 000 iterations (±2 %)
- [ ] `ApplyClusterPolicy` pre-sorts rules by descending priority
- [ ] Zero regex-compilation overhead at runtime (pre-compile `Regex` objects in `ApplyClusterPolicy`)
