# Phase 11.5 — Redis-Backed Circuit State Store

**Priority:** P3 (optional, in-process store is default)  
**Depends on:** Phase 11.1 (ICircuitStateStore interface used by ProxyCluster)  
**Goals satisfied:** G2 (cluster-wide shared circuit state)  
**Estimated effort:** ~0.5 day

---

## Objective

Replace the in-process `ConcurrentDictionary`-backed `ICircuitStateStore` with a
Redis-backed implementation.  
This ensures circuit state (open/closed/half-open counts) is shared across multiple
`ProxyCluster` instances running in different processes or machines.

## 2. `RedisCircuitStateStore`

**File:** `ProxyDataSource.RedisCircuitStateStore.cs`

```csharp
using StackExchange.Redis;

namespace TheTechIdea.Beep.Proxy
{
    /// <summary>
    /// Redis-backed circuit-state store.
    /// All keys live under the prefix "<paramref name="keyPrefix"/>:" so multiple
    /// clusters can share a single Redis instance without key collisions.
    /// </summary>
    public sealed class RedisCircuitStateStore : ICircuitStateStore, IDisposable
    {
        private readonly IConnectionMultiplexer _redis;
        private readonly IDatabase              _db;
        private readonly string                 _prefix;

        // ── Key format ────────────────────────────────────────────────
        // state:   <prefix>:state:<key>   → string "Open" | "Closed" | "HalfOpen"
        // fails:   <prefix>:fails:<key>   → integer (INCR)

        public RedisCircuitStateStore(
            string connectionString,
            string keyPrefix = "beepproxy")
        {
            ArgumentNullException.ThrowIfNullOrEmpty(connectionString);
            _prefix = keyPrefix;
            _redis  = ConnectionMultiplexer.Connect(connectionString);
            _db     = _redis.GetDatabase();
        }

        // ── Overload for injected multiplexer (testing / lifecycle mgmt) ──
        public RedisCircuitStateStore(IConnectionMultiplexer redis, string keyPrefix = "beepproxy")
        {
            _redis  = redis ?? throw new ArgumentNullException(nameof(redis));
            _db     = redis.GetDatabase();
            _prefix = keyPrefix;
        }

        // ── ICircuitStateStore ─────────────────────────────────────────

        public CircuitState GetState(string key)
        {
            var raw = _db.StringGet(StateKey(key));
            return raw.IsNullOrEmpty
                ? CircuitState.Closed
                : Enum.TryParse<CircuitState>(raw, out var cs) ? cs : CircuitState.Closed;
        }

        public void SetState(string key, CircuitState state, TimeSpan? ttl = null)
        {
            // Default TTL: 5 minutes so stale state doesn't accumulate if a cluster
            // node is removed without explicitly resetting state.
            _db.StringSet(StateKey(key), state.ToString(), ttl ?? TimeSpan.FromMinutes(5));
        }

        public int GetFailureCount(string key)
        {
            var val = _db.StringGet(FailKey(key));
            return val.IsNullOrEmpty ? 0 : (int)val;
        }

        public void IncrementFailureCount(string key, TimeSpan window)
        {
            var fk = FailKey(key);
            // INCR + EXPIRE in a single transaction so the window always starts fresh
            var tran = _db.CreateTransaction();
            tran.StringIncrementAsync(fk);
            tran.KeyExpireAsync(fk, window);
            tran.Execute();
        }

        public void ResetFailureCount(string key)
        {
            _db.KeyDelete(FailKey(key));
        }

        // ── Keyspace notifications (optional — enables push-based recovery) ─
        /// <summary>
        /// Subscribe to Redis keyspace events so that when another cluster instance
        /// resets a circuit key, <paramref name="onReset"/> is called immediately
        /// rather than waiting for the next poll cycle.
        /// </summary>
        /// <param name="onReset">Callback receiving the circuit key that was reset.</param>
        public void SubscribeToResetEvents(Action<string> onReset)
        {
            // Requires notify-keyspace-events to include "KExg" on the Redis server.
            // Command: CONFIG SET notify-keyspace-events KExg
            var sub = _redis.GetSubscriber();
            sub.Subscribe(
                new RedisChannel($"__keyevent@0__:del", RedisChannel.PatternMode.Literal),
                (channel, message) =>
                {
                    string msg = message.ToString();
                    string prefix = FailKey(string.Empty);   // "<prefix>:fails:"
                    if (msg.StartsWith(prefix, StringComparison.Ordinal))
                        onReset?.Invoke(msg.Substring(prefix.Length));
                });
        }

        // ── Helpers ───────────────────────────────────────────────────

        private string StateKey(string key) => $"{_prefix}:state:{key}";
        private string FailKey(string key)  => $"{_prefix}:fails:{key}";

        public void Dispose() => _redis?.Dispose();
    }
}
```

## 3. `ProxyCluster` Constructor Overload for Redis

No new file needed — add a convenience constructor to `ProxyCluster.cs`:

```csharp
/// <summary>
/// Creates a cluster backed by a Redis-shared circuit state store.
/// Suitable for multi-process / multi-machine deployments.
/// </summary>
public static ProxyCluster CreateWithRedis(
    ProxyPolicy   clusterPolicy,
    string        redisConnectionString,
    string        redisKeyPrefix  = "beepproxy",
    IProxyAuditSink auditSink     = null)
{
    var store = new RedisCircuitStateStore(redisConnectionString, redisKeyPrefix);
    return new ProxyCluster(clusterPolicy, store, auditSink);
}
```

## 4. Redis Key Schema

| Key Pattern | Type | TTL | Purpose |
|-------------|------|-----|---------|
| `{prefix}:state:{circuitKey}` | string | 5 min (auto-refresh on write) | Circuit open/closed/half-open |
| `{prefix}:fails:{circuitKey}` | integer | `window` passed to `IncrementFailureCount` | Failure count within the rolling window |

**Example keys** (prefix = `"orders-prod"`):

```
orders-prod:state:node-1/ordersDB      → "Open"
orders-prod:fails:node-1/ordersDB      → 7
orders-prod:state:node-2/ordersDB      → "Closed"
```

## 5. Deployment Notes

### Enable keyspace notifications on Redis (optional push recovery)

```
redis-cli CONFIG SET notify-keyspace-events KExg
```

This enables `onReset` callbacks so that when a circuit failure count is deleted
(reset), **all** cluster instances are notified within milliseconds, without polling.

### Redis minimum version

Redis ≥ 3.0 (keyspace notifications require 2.8+; MULTI/EXEC available since 1.2).
`StackExchange.Redis` ≥ 2.6.122.

### Security

- Use Redis AUTH (`password` in `connectionString`).
- Prefer TLS: `redis:6380,ssl=true,password=...`.
- `FLUSHALL` in prod is forbidden; key prefix keeps cluster data isolated.

## 6. Sequence Diagram — Cross-Process Circuit State

```
Process A (cluster-1)
  │
  │── node-1 fails 5 times ──► IncrementFailureCount("node1/ordersDB", window)
  │                                     │
  │                                     ▼
  │                              Redis: INCR  orders-prod:fails:node1/ordersDB  → 5
  │                                     │
  │── threshold reached                 │
  │── SetState("node1/ordersDB", Open)  │
  │                              Redis: SET orders-prod:state:node1/ordersDB "Open"

Process B (cluster-2, same Redis)
  │
  │── GetState("node1/ordersDB")
  │         │
  │         ▼
  │     Redis: GET orders-prod:state:node1/ordersDB  → "Open"
  │
  └── Skips node-1 immediately — no trial needed
```

## Acceptance Criteria

- [ ] `RedisCircuitStateStore.GetState` returns `CircuitState.Closed` when no key exists
- [ ] `IncrementFailureCount` uses INCR + EXPIRE atomically (no race window)
- [ ] `SetState(key, Open, ttl)` writes the correct Redis key with a TTL
- [ ] `ResetFailureCount` deletes the fail-count key
- [ ] Two in-process instances pointing at same Redis database share circuit state
- [ ] `SubscribeToResetEvents` callback fires when failure key is deleted in another process
- [ ] `Dispose` closes the Redis connection
- [ ] `CreateWithRedis` factory constructs cluster with Redis store correctly
