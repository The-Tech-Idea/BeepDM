# Phase 11.10 — Fault Injection + Per-Node Rate Limiting

**Priority:** P3 (testing / advanced traffic control)  
**Depends on:** Phase 11.1  
**Goals satisfied:** G14 (fault injection), G17 (per-node rate limiting)  
**Inspiration:** Netflix Chaos Monkey / Istio fault injection; Nginx `limit_req`, HAProxy `filter rate-limit`  
**Estimated effort:** ~0.5 day

---

## Objective

1. **Fault injection** — deliberately introduce errors or latency on a per-node or
   per-entity basis for chaos engineering and resilience testing in non-production
   environments.

2. **Per-node rate limiting** — cap the maximum requests-per-second (RPS) routed to
   any individual node, with configurable overflow actions (route elsewhere, queue,
   or reject).

## 1. `FaultInjectionPolicy`

```csharp
// In ProxyotherClasses.cs

/// <summary>
/// Enables deliberate error and latency injection for chaos engineering.
/// Should only be enabled in test / staging environments.
/// </summary>
public sealed class FaultInjectionPolicy
{
    /// <summary>
    /// Probability (0.0–1.0) that a request returns a fake error.
    /// 0.05 = 5 % of requests will fail with <see cref="FaultException"/>.
    /// </summary>
    public double ErrorRate { get; init; } = 0.0;

    /// <summary>
    /// Probability (0.0–1.0) that a request experiences an artificial delay.
    /// </summary>
    public double DelayRate { get; init; } = 0.0;

    /// <summary>Artificial delay in ms when DelayRate triggers.</summary>
    public int DelayMs { get; init; } = 500;

    /// <summary>
    /// When set, only requests routed to this node receive fault injection.
    /// Null = apply to all nodes.
    /// </summary>
    public string? TargetNodeId { get; init; }

    /// <summary>
    /// When set, only requests for this entity name receive fault injection.
    /// Null = apply to all entities.
    /// </summary>
    public string? TargetEntity { get; init; }
}
```

## 2. `NodeRateLimit` + `RateLimitAction`

```csharp
/// <summary>
/// Caps requests-per-second routed to a specific node.
/// </summary>
public sealed class NodeRateLimit
{
    /// <summary>The node this limit applies to.</summary>
    public string NodeId { get; init; } = default!;

    /// <summary>Maximum requests per second (token-bucket refill rate).</summary>
    public int MaxRps { get; init; } = 1000;

    /// <summary>What happens when the node's rate limit is exceeded.</summary>
    public RateLimitAction Action { get; init; } = RateLimitAction.RouteElsewhere;
}

public enum RateLimitAction
{
    /// <summary>Pick the next available node instead (transparent to caller).</summary>
    RouteElsewhere,
    /// <summary>Queue the request until a slot is available (up to cluster queue depth).</summary>
    Queue,
    /// <summary>Immediately reject with a rate-limit exception.</summary>
    Reject
}
```

## 3. Fault Injection Implementation

**File:** `ProxyCluster.cs`

```csharp
private static readonly ThreadLocal<Random> _faultRng
    = new(() => new Random(Guid.NewGuid().GetHashCode()));

/// <summary>
/// Applies fault injection policy if configured.
/// Call this at the START of every operation delegating to a node.
/// </summary>
private async Task ApplyFaultInjectionAsync(
    string    operationName,
    string?   entityHint,
    ProxyNode node,
    CancellationToken ct)
{
    var fi = _clusterPolicy.FaultInjection;
    if (fi is null) return;

    // Check scope filters
    if (fi.TargetNodeId   is not null && fi.TargetNodeId   != node.NodeId) return;
    if (fi.TargetEntity   is not null && fi.TargetEntity   != entityHint)  return;

    var rng = _faultRng.Value!;

    // ── Artificial delay ──────────────────────────────────────────────
    if (fi.DelayRate > 0 && rng.NextDouble() < fi.DelayRate)
        await Task.Delay(fi.DelayMs, ct).ConfigureAwait(false);

    // ── Artificial error ──────────────────────────────────────────────
    if (fi.ErrorRate > 0 && rng.NextDouble() < fi.ErrorRate)
        throw new ProxyFaultInjectionException(
            $"[FaultInjection] Artificial error on node '{node.NodeId}'" +
            (entityHint is not null ? $" entity '{entityHint}'" : string.Empty));
}
```

Usage inside a delegated operation:

```csharp
public IEnumerable<object> GetEntity(string entityName, List<AppFilter> filter)
{
    var node = SelectNode("GetEntity", entityName, null);

    // Fault injection (no-op when FaultInjection = null)
    ApplyFaultInjectionAsync("GetEntity", entityName, node, CancellationToken.None)
        .GetAwaiter().GetResult();   // sync path; use await in async overloads

    node.IncrementInFlight();
    try   { return node.Proxy.GetEntity(entityName, filter); }
    catch { RecordNodeOutcome(node, false); throw; }
    finally { node.DecrementInFlight(); }
}
```

## 4. Per-Node Rate Limiting (Token Bucket)

Each node gets a `SemaphoreSlim` acting as a token bucket — max `MaxRps` permits
are available; one permit is consumed per request.

```csharp
// In ProxyCluster.cs — field
private readonly ConcurrentDictionary<string, NodeRateLimiter> _rateLimiters = new();

private sealed class NodeRateLimiter : IDisposable
{
    private readonly SemaphoreSlim _semaphore;
    private readonly int           _maxRps;
    private readonly Timer         _refillTimer;

    public NodeRateLimiter(int maxRps)
    {
        _maxRps    = maxRps;
        _semaphore = new SemaphoreSlim(maxRps, maxRps);

        // Refill all permits every 1 second (simplistic fixed-window token bucket)
        _refillTimer = new Timer(_ =>
        {
            int deficit = _maxRps - _semaphore.CurrentCount;
            if (deficit > 0)
                _semaphore.Release(deficit);
        }, null, 1000, 1000);
    }

    /// <summary>
    /// Try to acquire a slot; returns false immediately if full.
    /// </summary>
    public bool TryAcquire() => _semaphore.Wait(0);

    /// <summary>
    /// Wait up to <paramref name="timeoutMs"/> ms for a slot.
    /// </summary>
    public Task<bool> TryAcquireAsync(int timeoutMs, CancellationToken ct)
        => _semaphore.WaitAsync(timeoutMs, ct);

    public void Dispose() { _refillTimer.Dispose(); _semaphore.Dispose(); }
}
```

Rate-limit check in `SelectNode`:

```csharp
private ProxyNode SelectNode(string operationName, string? entityHint, ProxyExecutionContext? ctx)
{
    var live = _nodes.Values
        .Where(n => n.IsAlive && !n.IsDraining)
        .ToList();

    if (live.Count == 0)
        throw new InvalidOperationException("ProxyCluster: no live nodes available.");

    // ... all prior selection logic (entity affinity, routing rules, etc.) ...

    // ── Rate limit check on selected node ─────────────────────────────
    var selected = _nodeRouter.SelectBest(live, ctx);
    selected = ApplyRateLimit(selected, live, operationName);

    return selected;
}

private ProxyNode ApplyRateLimit(
    ProxyNode         selected,
    List<ProxyNode>   live,
    string            operationName)
{
    var limitCfg = _clusterPolicy.NodeRateLimits
        .FirstOrDefault(l => l.NodeId == selected.NodeId);
    if (limitCfg is null) return selected;

    var limiter = _rateLimiters.GetOrAdd(
        selected.NodeId, _ => new NodeRateLimiter(limitCfg.MaxRps));

    if (limiter.TryAcquire())
        return selected;   // slot available

    // Slot unavailable — apply action
    return limitCfg.Action switch
    {
        RateLimitAction.RouteElsewhere
            => live.FirstOrDefault(n => n.NodeId != selected.NodeId)
               ?? selected,   // only one node → no choice

        RateLimitAction.Queue
            // Caller blocks until slot available (handled in async path)
            => selected,   // sync path: fall through, will wait in operation

        RateLimitAction.Reject
            => throw new ProxyRateLimitException(
                $"Rate limit exceeded for node '{selected.NodeId}' " +
                $"(max {limitCfg.MaxRps} rps)."),

        _ => selected
    };
}
```

## 6. Sequence Diagrams

### Fault injection

```
GetEntity("Orders", filter)
  │
  ├─ node = SelectNode(...)  → node-canary
  │
  ├─ ApplyFaultInjectionAsync(node=node-canary, entity="Orders")
  │     ├─ FaultInjection.TargetNodeId = "node-canary" → matches
  │     ├─ rng.NextDouble() = 0.07 < ErrorRate(0.10) → THROW ProxyFaultInjectionException
  │
  └─ Caller catches exception (test asserts resilience behaviour)
```

### Rate limit overflow — RouteElsewhere

```
SelectNode("GetEntity", "Orders", ctx)
  │
  ├─ _nodeRouter → node-1
  │
  ├─ ApplyRateLimit(node-1, live)
  │     ├─ limiter[node-1].TryAcquire() → false  (500 rps bucket empty)
  │     ├─ Action = RouteElsewhere
  │     └─ pick live node ≠ node-1 → node-2
  │
  └─ return node-2
```

## Acceptance Criteria

- [ ] `FaultInjection.ErrorRate = 0` never throws `ProxyFaultInjectionException`
- [ ] `FaultInjection.ErrorRate = 1.0` always throws
- [ ] `TargetNodeId` scope filter prevents injection on non-matching nodes
- [ ] `TargetEntity` scope filter prevents injection on non-matching entities
- [ ] Token bucket refills to `MaxRps` every second
- [ ] `RateLimitAction.RouteElsewhere` picks a different live node transparently
- [ ] `RateLimitAction.Reject` throws `ProxyRateLimitException` immediately
- [ ] `ApplyClusterPolicy` rebuilds rate limiters, disposing old ones without leaking
- [ ] `FaultInjectionPolicy = null` adds zero overhead (no rng calls, no await)
