# Phase 11.8 — Request Hedging + Outlier Detection + Slow-Start

**Priority:** P2  
**Depends on:** Phase 11.1, Phase 11.4  
**Goals satisfied:** G9 (request hedging), G12 (slow-start), G13 (outlier detection)  
**Inspiration:** Envoy hedged requests; HAProxy slow-start; Envoy outlier detection (RFC)  
**Estimated effort:** ~1 day

---

## Objective

1. **Request hedging** — if a node hasn't responded within `HedgingThresholdMs`,
   fire the same request in parallel to a second node; cancel the slower one
   when the first resolves.

2. **Slow-start** — newly-added nodes receive a linearly-ramped traffic share
   during `SlowStartDurationMs` so they warm up gradually.

3. **Outlier detection** — automatically eject nodes exhibiting a high error rate or
   consecutive failures from the routing pool for an exponentially-increasing
   back-off period.

/// <summary>
/// Envoy-style outlier detection: eject misbehaving nodes from the routing pool
/// for an exponentially increasing back-off period.
/// </summary>
public sealed class OutlierDetectionPolicy
{
    /// <summary>Consecutive errors needed to eject a node immediately.</summary>
    public int ConsecutiveErrorThreshold { get; init; } = 5;

    /// <summary>
    /// Error proportion (0.0–1.0) over the analysis interval.
    /// Nodes above this threshold are ejected.
    /// </summary>
    public double ErrorRateThreshold { get; init; } = 0.5;

    /// <summary>Interval in ms over which error rate is measured.</summary>
    public int IntervalMs { get; init; } = 10_000;

    /// <summary>Base ejection time in ms (first ejection).</summary>
    public int BaseEjectionTimeMs { get; init; } = 30_000;

    /// <summary>Maximum ejection time in ms (exponential cap).</summary>
    public int MaxEjectionTimeMs { get; init; } = 300_000;   // 5 min

    /// <summary>
    /// Maximum percentage of nodes that can be ejected simultaneously.
    /// Protects against ejecting the entire pool.
    /// </summary>
    public int MaxEjectionPercent { get; init; } = 50;
}

## 2. Request Hedging

**File:** `ProxyCluster.cs`

The hedging wrapper is a generic helper that wraps any async operation:

```csharp
/// <summary>
/// Fires the operation on <paramref name="primary"/>; if it doesn't complete
/// within <see cref="ProxyPolicy.HedgingThresholdMs"/>, fires the same operation
/// on <paramref name="hedge"/> in parallel then takes whichever resolves first.
/// </summary>
private static async Task<T> ExecuteWithHedging<T>(
    ProxyNode             primary,
    ProxyNode             hedge,
    Func<ProxyNode, Task<T>> operation,
    int                   thresholdMs,
    CancellationToken     ct = default)
{
    // Fire the primary request
    var primaryCts  = CancellationTokenSource.CreateLinkedTokenSource(ct);
    var primaryTask = operation(primary);

    // Set a delay timer for the hedge
    using var delayCts   = new CancellationTokenSource();
    var hedgeDelay = Task.Delay(thresholdMs, delayCts.Token);

    // Wait for either the primary to finish, or the hedge delay to expire
    var firstCompleted = await Task.WhenAny(primaryTask, hedgeDelay).ConfigureAwait(false);

    if (firstCompleted == primaryTask)
    {
        // Primary won — cancel hedge delay, return result
        delayCts.Cancel();
        return await primaryTask.ConfigureAwait(false);
    }

    // Hedge delay elapsed — fire hedge in parallel
    var hedgeCts  = CancellationTokenSource.CreateLinkedTokenSource(ct);
    var hedgeTask = operation(hedge);

    // Return whichever finishes first (success or failure)
    var winner = await Task.WhenAny(primaryTask, hedgeTask).ConfigureAwait(false);

    // Cancel the loser
    if (winner == primaryTask) hedgeCts.Cancel();
    else primaryCts.Cancel();

    return await winner.ConfigureAwait(false);
}
```

Integration inside the per-operation delegation pattern:

```csharp
// Example: GetEntity with hedging — matches IDataSource.GetEntityAsync signature
public Task<IEnumerable<object>> GetEntityAsync(string entityName, List<AppFilter> filter)
{
    var live = GetLiveNodes();

    if (_clusterPolicy.EnableHedging && live.Count >= 2)
    {
        var primary = SelectNode("GetEntity", entityName, ctx);
        var hedge   = SelectNode("GetEntity", entityName, ctx);   // picks 2nd-best
        // Ensure hedge ≠ primary when possible
        if (hedge.NodeId == primary.NodeId && live.Count > 1)
            hedge = live.First(n => n.NodeId != primary.NodeId);

        return ExecuteWithHedging(
            primary,
            hedge,
            node => node.Proxy.GetEntityAsync(entityName, filter),
            _clusterPolicy.HedgingThresholdMs,
            ct);
    }

    // No hedging — normal single-node path
    var node = SelectNode("GetEntity", entityName, ctx);
    node.IncrementInFlight();
    try   { return node.Proxy.GetEntityAsync(entityName, filter); }
    finally { node.DecrementInFlight(); }
}
```

## 3. Outlier Detection

**File:** `ProxyCluster.NodeProbing.cs`

Per-node error tracking (stored on `ProxyNode`):

```csharp
// Add to ProxyNode.cs (internal state)
internal int   OutlierConsecutiveErrors { get; set; }
internal int   OutlierWindowRequests    { get; set; }
internal int   OutlierWindowErrors      { get; set; }
internal DateTime OutlierWindowStart    { get; set; } = DateTime.UtcNow;
internal DateTime? OutlierEjectedUntil  { get; set; }
internal int   OutlierEjectionCount     { get; set; }   // for exponential back-off
```

Outlier detection analysis — runs on the same probe timer as node liveness:

```csharp
private void RunOutlierDetection()
{
    var policy = _clusterPolicy.OutlierDetection;
    if (policy is null) return;

    var now          = DateTime.UtcNow;
    var liveCount    = _nodes.Values.Count(n => n.IsAlive && !n.IsDraining);
    var maxEjectable = Math.Max(1,
        (int)Math.Floor(liveCount * policy.MaxEjectionPercent / 100.0));
    int ejectedThisCycle = 0;

    foreach (var node in _nodes.Values)
    {
        // ── Restore previously ejected node ──────────────────────────
        if (node.OutlierEjectedUntil.HasValue && now >= node.OutlierEjectedUntil.Value)
        {
            node.OutlierEjectedUntil        = null;
            node.IsDraining                 = false;   // re-enter routing pool
            node.OutlierConsecutiveErrors   = 0;
            node.OutlierWindowRequests      = 0;
            node.OutlierWindowErrors        = 0;
            node.OutlierWindowStart         = now;
            OnNodeRestored?.Invoke(this, new NodeStatusEventArgs(node.NodeId, true, "outlier re-admitted"));
            continue;
        }

        if (node.OutlierEjectedUntil.HasValue) continue;   // still ejected

        // ── Check consecutive error threshold ─────────────────────────
        bool shouldEject =
            node.OutlierConsecutiveErrors >= policy.ConsecutiveErrorThreshold;

        // ── Check error-rate threshold over the analysis window ───────
        if (!shouldEject
            && (now - node.OutlierWindowStart).TotalMilliseconds >= policy.IntervalMs)
        {
            double errorRate = node.OutlierWindowRequests == 0
                ? 0
                : (double)node.OutlierWindowErrors / node.OutlierWindowRequests;

            shouldEject = errorRate >= policy.ErrorRateThreshold;

            // Reset window
            node.OutlierWindowRequests = 0;
            node.OutlierWindowErrors   = 0;
            node.OutlierWindowStart    = now;
        }

        // ── Eject if needed ────────────────────────────────────────────
        if (shouldEject && ejectedThisCycle < maxEjectable)
        {
            node.OutlierEjectionCount++;
            int ejectionMs = Math.Min(
                policy.BaseEjectionTimeMs * node.OutlierEjectionCount,
                policy.MaxEjectionTimeMs);

            node.OutlierEjectedUntil = now.AddMilliseconds(ejectionMs);
            node.IsDraining          = true;   // exclude from routing

            ejectedThisCycle++;

            _auditSink.Write(new ProxyAuditEntry
            {
                EntryKind     = AuditEntryKind.OutlierEjection,
                NodeId        = node.NodeId,
                Success       = false,
                Message       = $"Outlier ejected for {ejectionMs}ms (ejection #{node.OutlierEjectionCount})",
                OccurredAtUtc = now
            });
        }
    }
}
```

## 4. Slow-Start Ramp Formula

As described in Phase 11.1, the weight ramp is linear:

```
rampFactor = min(1.0, elapsedSinceAddedMs / SlowStartDurationMs)
effectiveWeight = max(1, floor(baseWeight × rampFactor))
```

At 0 ms → weight ≈ 0 (floor sets it to 1 minimum).  
At `SlowStartDurationMs` → weight = baseWeight (fully ramped).

This is the same formula used by HAProxy slow-start:
https://www.haproxy.com/blog/whats-new-in-haproxy-1-9

```csharp
private void ApplySlowStartWeights(List<ProxyNode> candidates)
{
    if (_clusterPolicy.SlowStartDurationMs <= 0) return;

    foreach (var n in candidates)
    {
        double elapsedMs  = (DateTime.UtcNow - n.AddedAtUtc).TotalMilliseconds;
        double rampFactor = Math.Min(1.0, elapsedMs / _clusterPolicy.SlowStartDurationMs);
        // Guard: minimum weight 1 so the node is never completely starved
        n.Weight = Math.Max(1, (int)Math.Floor(n.Weight * rampFactor));
    }
}
```

**Important:** `ApplySlowStartWeights` only modifies the weight for the duration of
`SelectNode`. The node's persisted `Weight` (set by the operator) is not mutated;
the ramp calculation reads `AddedAtUtc` each time.

Actually the weight _must not_ be mutated on `ProxyNode.Weight` directly or the
system loses the operator-configured base weight. Refactor:

```csharp
// ProxyNode.cs — add:
internal int EffectiveWeight { get; set; }

// ProxyCluster.ApplySlowStartWeights — change to:
private void ApplySlowStartWeights(List<ProxyNode> candidates)
{
    if (_clusterPolicy.SlowStartDurationMs <= 0)
    {
        foreach (var n in candidates) n.EffectiveWeight = n.Weight;
        return;
    }
    foreach (var n in candidates)
    {
        double elapsedMs  = (DateTime.UtcNow - n.AddedAtUtc).TotalMilliseconds;
        double rampFactor = Math.Min(1.0, elapsedMs / _clusterPolicy.SlowStartDurationMs);
        n.EffectiveWeight = Math.Max(1, (int)Math.Floor(n.Weight * rampFactor));
    }
}
```

All routers that use `node.Weight` must be updated to use `node.EffectiveWeight`.

## Acceptance Criteria

- [ ] `ExecuteWithHedging` only sends hedge if primary exceeds `HedgingThresholdMs`
- [ ] Only one hedge at a time (not `MaxHedgeRequests` fans, just 1 extra)
- [ ] Hedge is cancelled cleanly when primary finishes first
- [ ] `OutlierDetectionPolicy.ConsecutiveErrorThreshold` triggers immediate eject
- [ ] `OutlierDetectionPolicy.ErrorRateThreshold` triggers eject at end of window
- [ ] Eject duration doubles on each subsequent ejection (capped at `MaxEjectionTimeMs`)
- [ ] At most `MaxEjectionPercent` % of nodes are ejected simultaneously
- [ ] Node is restored to routing pool when ejection period expires
- [ ] Slow-start uses `EffectiveWeight` (does **not** mutate `Weight`)
- [ ] Slow-start weight is ≥ 1 at all times
