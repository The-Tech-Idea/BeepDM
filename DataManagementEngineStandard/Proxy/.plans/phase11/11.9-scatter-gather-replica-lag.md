# Phase 11.9 — Scatter-Gather Reads + Replica Lag Awareness

**Priority:** P2  
**Depends on:** Phase 11.1, Phase 11.4  
**Goals satisfied:** G11 (scatter-gather), G15 (replica lag awareness)  
**Inspiration:** Vitess `ExecuteMulti`, AWS Aurora replica lag staleness guard  
**Estimated effort:** ~0.5 day

---

## Objective

1. **Scatter-gather reads** — broadcast a read to every live replica in parallel, then
   merge results.  Useful when data is sharded across nodes and a single node only
   holds a partition of the answer.

2. **Replica lag awareness** — exclude replicas whose replication lag exceeds
   `MaxReplicaLagMs` from the routing pool for read operations, ensuring stale data
   is never served when freshness is required.

## 2. `ReplicaLagMs` on `ProxyNode`

```csharp
// Add to ProxyNode.cs

/// <summary>
/// Current measured replication lag in milliseconds.
/// Updated by the probe timer via <see cref="ProxyCluster.NodeProbing"/>.
/// -1 = unknown (node has never been probed for lag).
/// </summary>
public long ReplicaLagMs { get; internal set; } = -1;
```

Lag can be updated two ways:

1. **Passive** — from the node's own metrics if the underlying datasource exposes it
   (`node.Proxy.GetMetrics()` may include a lag metric).
2. **Active** — by running a dedicated probe query (e.g., `SELECT
   TIMESTAMPDIFF(SECOND, NOW(), MAX(updated_at))`) on each replica and feeding the
   result back.

The probe hook is a no-op in the base implementation; operators can inject a custom
lag updater:

```csharp
// Optional delegate registered on ProxyCluster
public Func<IProxyNode, Task<long>>? ReplicaLagProbe { get; set; }
```

If not set, `ReplicaLagMs` remains `-1` and the lag guard is effectively disabled.

## 3. Lag-Aware Node Selection

```csharp
// In ProxyCluster.cs

/// <summary>
/// Returns live replica nodes with acceptable lag.
/// Falls back to primary nodes when no acceptable replica is available.
/// </summary>
private List<ProxyNode> GetReadCandidates()
{
    var live = _nodes.Values
        .Where(n => n.IsAlive && !n.IsDraining)
        .ToList();

    long maxLag = _clusterPolicy.MaxReplicaLagMs;

    if (maxLag <= 0)
        return live;   // lag guard disabled

    var acceptable = live
        .Where(n =>
            // Replicas with unknown lag (-1) or acceptable lag pass the filter
            n.NodeRole != ProxyDataSourceRole.Replica
            || n.ReplicaLagMs < 0
            || n.ReplicaLagMs <= maxLag)
        .ToList();

    // If the lag guard excluded all replicas, fall back to all live nodes
    return acceptable.Count > 0 ? acceptable : live;
}
```

## 4. `ExecuteScatterGatherAsync<T>`

Broadcasts the read to all candidates in parallel and applies a caller-supplied
merge function to combine results:

```csharp
/// <summary>
/// Execute <paramref name="operation"/> on every candidate node in parallel and
/// return the merged result via <paramref name="merge"/>.
/// </summary>
/// <typeparam name="T">Result type (e.g., IEnumerable&lt;object&gt;, List&lt;EntityStructure&gt;).</typeparam>
/// <param name="operation">The read operation to fan out.</param>
/// <param name="merge">
///   Receives all successful results and returns the final merged value.
///   Must handle an empty collection (all nodes failed).
/// </param>
private async Task<T> ExecuteScatterGatherAsync<T>(
    IReadOnlyList<ProxyNode>       candidates,
    Func<ProxyNode, Task<T>>       operation,
    Func<IReadOnlyList<T>, T>      merge,
    CancellationToken              ct = default)
{
    if (candidates.Count == 0)
        throw new InvalidOperationException("ScatterGather: no live nodes to scatter to.");

    var tasks = candidates
        .Select(node => ExecuteSingleScatterAsync(node, operation, ct))
        .ToList();

    var results = await Task.WhenAll(tasks).ConfigureAwait(false);

    var successful = results
        .Where(r => r.Success)
        .Select(r => r.Value!)
        .ToList();

    if (successful.Count == 0)
    {
        var firstFail = results.First(r => !r.Success);
        throw new AggregateException(
            "All scatter-gather nodes failed.",
            firstFail.Exception ?? new Exception("Unknown error"));
    }

    return merge(successful);
}

private sealed record ScatterResult<T>(bool Success, T? Value, Exception? Exception);

private async Task<ScatterResult<T>> ExecuteSingleScatterAsync<T>(
    ProxyNode             node,
    Func<ProxyNode, Task<T>> operation,
    CancellationToken     ct)
{
    node.IncrementInFlight();
    try
    {
        var result = await operation(node).ConfigureAwait(false);
        return new ScatterResult<T>(true, result, null);
    }
    catch (Exception ex)
    {
        RecordNodeOutcome(node, success: false);
        return new ScatterResult<T>(false, default, ex);
    }
    finally
    {
        node.DecrementInFlight();
    }
}
```

## 5. Default Merge Functions

Provided as static helpers for common result types matching `IDataSource` return types:

```csharp
public static class ScatterMerge
{
    /// <summary>
    /// Concatenate all shard results into one flat sequence.
    /// Matches IDataSource.GetEntity / GetEntityAsync return type IEnumerable&lt;object&gt;.
    /// </summary>
    public static IEnumerable<object> ConcatObjects(
        IReadOnlyList<IEnumerable<object>> shards)
        => shards.SelectMany(s => s);

    /// <summary>
    /// De-duplicate across shards on a key property.
    /// Useful when the same logical record might appear in more than one node
    /// (e.g., fanout writes were partially applied).
    /// </summary>
    public static IEnumerable<object> DeduplicateByKey(
        IReadOnlyList<IEnumerable<object>> shards,
        Func<object, object> keySelector)
        => shards.SelectMany(s => s)
                 .GroupBy(keySelector)
                 .Select(g => g.First());

    /// <summary>
    /// Take first non-null result (for operations where one shard owns the record,
    /// e.g. a GetEntity by primary key that only one node will return a non-empty result).
    /// Matches RunQuery / GetEntity semantics.
    /// </summary>
    public static IEnumerable<object> FirstNonEmpty(
        IReadOnlyList<IEnumerable<object>> shards)
        => shards.FirstOrDefault(s => s.Any()) ?? Enumerable.Empty<object>();
}
```

## 6. Integration into `GetEntityAsync`

Matches the `IDataSource` signature:
```csharp
// IDataSource: Task<IEnumerable<object>> GetEntityAsync(string EntityName, List<AppFilter> Filter)
public async Task<IEnumerable<object>> GetEntityAsync(
    string entityName, List<AppFilter> filter)
{
    if (_clusterPolicy.ReadMode == ProxyReadMode.ScatterGather)
    {
        var candidates = GetReadCandidates();
        return await ExecuteScatterGatherAsync(
            candidates,
            node => node.Proxy.GetEntityAsync(entityName, filter),
            ScatterMerge.ConcatObjects,   // or DeduplicateByKey / FirstNonEmpty
            ct).ConfigureAwait(false);
    }

    // SingleNode path
    var node = SelectNode("GetEntity", entityName, ctx);
    node.IncrementInFlight();
    try   { return await node.Proxy.GetEntityAsync(entityName, filter).ConfigureAwait(false); }
    finally { node.DecrementInFlight(); }
}
```

Same pattern applies to `RunQuery(string qrystr)` which returns `IEnumerable<object>`.

## 7. Lag Probe Integration

```csharp
// In ProxyCluster.NodeProbing.cs — add to the probe tick:

private async Task RunLagProbesAsync()
{
    if (ReplicaLagProbe is null) return;

    foreach (var node in _nodes.Values.Where(n => n.IsAlive))
    {
        try
        {
            long lagMs = await ReplicaLagProbe(node).ConfigureAwait(false);
            node.ReplicaLagMs = lagMs;
        }
        catch
        {
            // Probe failure is not counted as a node failure
            // — just leave the previous lag value intact.
        }
    }
}
```

## Acceptance Criteria

- [ ] `ReadMode.ScatterGather` fires operation on all `GetReadCandidates()` in parallel
- [ ] Scatter results from failed nodes are silently excluded (not thrown) if at least one succeeds
- [ ] All nodes fail → throws `AggregateException` with last failure
- [ ] Default merge `ConcatObjects` concatenates all shards into a flat `IEnumerable<object>`
- [ ] `MaxReplicaLagMs = 0` disables the lag guard (all replicas included)
- [ ] Replicas with `ReplicaLagMs > MaxReplicaLagMs` are excluded from `GetReadCandidates()`
- [ ] All replicas exceed lag threshold → falls back to all live nodes (never returns empty list)
- [ ] `ReplicaLagMs = -1` (unknown) passes the lag filter (conservative: trust unknown)
- [ ] `ReplicaLagProbe = null` leaves `ReplicaLagMs = -1` permanently
